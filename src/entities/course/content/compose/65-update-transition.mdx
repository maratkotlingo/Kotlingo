## Что такое `updateTransition`

`updateTransition` создает и запоминает `Transition`, который управляет несколькими animated values, зависящими от одного `targetState`. Если `animate*AsState` отвечает за одно значение, то `Transition` отвечает за группу значений, которые вместе описывают переход компонента из одного состояния в другое.

Пример: карточка может быть `Collapsed` или `Expanded`. При раскрытии меняются высота, цвет фона, alpha деталей и угол стрелки. Все эти значения — часть одного перехода. Их удобно держать в одном `Transition`.

```kotlin
import androidx.compose.animation.animateColor
import androidx.compose.animation.core.animateDp
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.updateTransition
import androidx.compose.foundation.layout.Column
import androidx.compose.foundation.layout.height
import androidx.compose.material3.Card
import androidx.compose.material3.MaterialTheme
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.draw.alpha
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

enum class ExpandableCardState {
    Collapsed,
    Expanded
}

@Composable
fun ExpandableCard(
    state: ExpandableCardState,
    title: String,
    details: String
) {
    val transition = updateTransition(
        targetState = state,
        label = "expandable card"
    )

    val height by transition.animateDp(
        label = "card height"
    ) { targetState ->
        when (targetState) {
            ExpandableCardState.Collapsed -> 96.dp
            ExpandableCardState.Expanded -> 180.dp
        }
    }

    val detailsAlpha by transition.animateFloat(
        label = "details alpha"
    ) { targetState ->
        when (targetState) {
            ExpandableCardState.Collapsed -> 0f
            ExpandableCardState.Expanded -> 1f
        }
    }

    val arrowRotation by transition.animateFloat(
        label = "arrow rotation"
    ) { targetState ->
        when (targetState) {
            ExpandableCardState.Collapsed -> 0f
            ExpandableCardState.Expanded -> 180f
        }
    }

    val containerColor by transition.animateColor(
        label = "container color"
    ) { targetState ->
        when (targetState) {
            ExpandableCardState.Collapsed -> MaterialTheme.colorScheme.surface
            ExpandableCardState.Expanded -> MaterialTheme.colorScheme.primaryContainer
        }
    }

    Card(
        modifier = Modifier.height(height),
        colors = androidx.compose.material3.CardDefaults.cardColors(
            containerColor = containerColor
        )
    ) {
        Column {
            Text(text = title)
            Text(
                text = "⌄",
                modifier = Modifier.graphicsLayer {
                    rotationZ = arrowRotation
                }
            )
            Text(
                text = details,
                modifier = Modifier.alpha(detailsAlpha)
            )
        }
    }
}
```

Все animated values читают один и тот же `targetState`. Когда state меняется, transition синхронно ведет все дочерние анимации.

<CatTip>
Если вы видите четыре `animate*AsState`, которые все зависят от одного enum, это хороший сигнал подумать об `updateTransition`. Он делает state machine явной.
</CatTip>

## Когда использовать `updateTransition`

Используйте `updateTransition`, когда:

- несколько animated values зависят от одного state;
- переходы образуют state machine;
- нужны разные specs для разных направлений;
- хочется централизовать motion logic компонента;
- animated values должны стартовать и завершаться согласованно;
- компонент имеет состояния вроде `Collapsed`, `Expanded`, `Selected`, `Disabled`;
- нужно переиспользовать transition logic отдельно от layout.

Не нужен `updateTransition`, когда:

- анимируется одно значение;
- values зависят от разных независимых states;
- нужна coroutine sequence;
- анимация управляется жестом;
- transition бесконечный;
- достаточно `AnimatedVisibility` или `AnimatedContent`.

## Общая форма API

Базовый паттерн:

```kotlin
val transition = updateTransition(
    targetState = state,
    label = "component transition"
)

val value by transition.animateFloat(
    label = "value"
) { targetState ->
    when (targetState) {
        ComponentState.First -> 0f
        ComponentState.Second -> 1f
    }
}
```

`updateTransition` принимает target state любого типа. Для читаемости чаще используют enum или sealed interface.

## Доступные animate-функции

У `Transition` есть функции для разных типов:

- `animateFloat`;
- `animateDp`;
- `animateColor`;
- `animateInt`;
- `animateIntOffset`;
- `animateOffset`;
- `animateRect`;
- `animateSize`;
- `animateValue`.

Каждая функция принимает:

- `transitionSpec`: optional spec, который может зависеть от segment;
- `label`: имя для tooling;
- target value lambda: mapping state -> target value.

```kotlin
import androidx.compose.animation.core.animateInt
import androidx.compose.animation.core.updateTransition
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue

enum class DownloadState {
    NotStarted,
    InProgress,
    Finished
}

@Composable
fun DownloadPercent(
    state: DownloadState
) {
    val transition = updateTransition(
        targetState = state,
        label = "download state"
    )

    val percent by transition.animateInt(
        label = "download percent"
    ) { targetState ->
        when (targetState) {
            DownloadState.NotStarted -> 0
            DownloadState.InProgress -> 62
            DownloadState.Finished -> 100
        }
    }

    Text(text = "$percent%")
}
```

Для реального download progress лучше использовать actual progress из state. Этот пример показывает тип `animateInt`.

## `transitionSpec`

`transitionSpec` позволяет задать spec для конкретного segment: от какого state к какому state происходит переход. Внутри доступны `initialState`, `targetState` и infix helper `isTransitioningTo`.

```kotlin
import androidx.compose.animation.core.Spring
import androidx.compose.animation.core.animateDp
import androidx.compose.animation.core.spring
import androidx.compose.animation.core.tween
import androidx.compose.animation.core.updateTransition
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.height
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.unit.dp

enum class SheetPreviewState {
    Hidden,
    Peek,
    Expanded
}

@Composable
fun SheetPreview(
    state: SheetPreviewState
) {
    val transition = updateTransition(
        targetState = state,
        label = "sheet preview"
    )

    val height by transition.animateDp(
        transitionSpec = {
            if (SheetPreviewState.Hidden isTransitioningTo SheetPreviewState.Expanded) {
                spring(
                    dampingRatio = Spring.DampingRatioNoBouncy,
                    stiffness = Spring.StiffnessMediumLow
                )
            } else {
                tween(durationMillis = 220)
            }
        },
        label = "sheet height"
    ) { targetState ->
        when (targetState) {
            SheetPreviewState.Hidden -> 0.dp
            SheetPreviewState.Peek -> 96.dp
            SheetPreviewState.Expanded -> 360.dp
        }
    }

    Box(
        modifier = Modifier.height(height)
    )
}
```

`isTransitioningTo` делает код выразительным: spec зависит не просто от target, а от пары initial -> target.

## Несколько specs в одном transition

Разные animated values могут иметь разные specs. Например, container меняет цвет быстрее, чем height.

```kotlin
import androidx.compose.animation.animateColor
import androidx.compose.animation.core.animateDp
import androidx.compose.animation.core.tween
import androidx.compose.animation.core.updateTransition
import androidx.compose.foundation.background
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.height
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun StateStrip(
    state: SheetPreviewState
) {
    val transition = updateTransition(
        targetState = state,
        label = "state strip"
    )

    val height by transition.animateDp(
        transitionSpec = {
            tween(durationMillis = 280)
        },
        label = "strip height"
    ) { targetState ->
        when (targetState) {
            SheetPreviewState.Hidden -> 0.dp
            SheetPreviewState.Peek -> 8.dp
            SheetPreviewState.Expanded -> 16.dp
        }
    }

    val color by transition.animateColor(
        transitionSpec = {
            tween(durationMillis = 160)
        },
        label = "strip color"
    ) { targetState ->
        when (targetState) {
            SheetPreviewState.Hidden -> Color.Transparent
            SheetPreviewState.Peek -> Color.Gray
            SheetPreviewState.Expanded -> Color.Blue
        }
    }

    Box(
        modifier = Modifier
            .fillMaxWidth()
            .height(height)
            .background(color)
    )
}
```

Один transition не означает одинаковый duration для всех значений. Он означает общий state и общую координацию.

## Enum state против booleans

Если у компонента больше двух режимов, enum обычно лучше нескольких booleans.

Плохо:

```kotlin
data class PanelFlags(
    val expanded: Boolean,
    val selected: Boolean,
    val disabled: Boolean
)
```

Такой state допускает странные комбинации: `expanded = true`, `disabled = true`, `selected = true`, хотя продукт может не поддерживать такой режим.

Лучше:

```kotlin
enum class PanelState {
    Compact,
    Expanded,
    Selected,
    Disabled
}
```

Теперь transition описывает допустимые состояния явно.

```kotlin
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.updateTransition
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue

@Composable
fun rememberPanelAlpha(
    state: PanelState
): Float {
    val transition = updateTransition(
        targetState = state,
        label = "panel alpha"
    )

    val alpha by transition.animateFloat(
        label = "panel alpha value"
    ) { targetState ->
        when (targetState) {
            PanelState.Compact -> 0.86f
            PanelState.Expanded -> 1f
            PanelState.Selected -> 1f
            PanelState.Disabled -> 0.38f
        }
    }

    return alpha
}
```

## Sealed state

Если state несет данные, используйте sealed interface.

```kotlin
import androidx.compose.runtime.Immutable

@Immutable
sealed interface PlayerState {
    data object Stopped : PlayerState
    data class Playing(val trackTitle: String) : PlayerState
    data class Paused(val trackTitle: String) : PlayerState
    data object Buffering : PlayerState
}
```

Для transition target values лучше не зависеть от часто меняющихся данных, если они не должны запускать motion. Можно перейти к motion state.

```kotlin
enum class PlayerMotionState {
    Inactive,
    Active,
    Loading
}

fun PlayerState.toMotionState(): PlayerMotionState {
    return when (this) {
        PlayerState.Stopped -> PlayerMotionState.Inactive
        is PlayerState.Playing -> PlayerMotionState.Active
        is PlayerState.Paused -> PlayerMotionState.Active
        PlayerState.Buffering -> PlayerMotionState.Loading
    }
}
```

```kotlin
import androidx.compose.animation.core.animateDp
import androidx.compose.animation.core.updateTransition
import androidx.compose.foundation.layout.height
import androidx.compose.material3.Surface
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.unit.dp

@Composable
fun PlayerMiniPanel(
    playerState: PlayerState
) {
    val motionState = playerState.toMotionState()
    val transition = updateTransition(
        targetState = motionState,
        label = "player motion"
    )

    val height by transition.animateDp(
        label = "player height"
    ) { targetState ->
        when (targetState) {
            PlayerMotionState.Inactive -> 0.dp
            PlayerMotionState.Active -> 72.dp
            PlayerMotionState.Loading -> 56.dp
        }
    }

    Surface(
        modifier = Modifier.height(height)
    ) {
        androidx.compose.material3.Text(text = "Плеер")
    }
}
```

Так motion не перезапускается при каждой смене `trackTitle`.

<CatTip>
Разделяйте business state и motion state. Иногда они совпадают, но часто motion должен реагировать только на крупные режимы, а не на каждое поле UI state.
</CatTip>

## Reusable transition data

Для сложных компонентов удобно вынести animated values в отдельный immutable holder.

```kotlin
import androidx.compose.runtime.Immutable
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.Dp

@Immutable
data class CardTransitionData(
    val height: Dp,
    val containerColor: Color,
    val detailsAlpha: Float,
    val arrowRotation: Float
)
```

```kotlin
import androidx.compose.animation.animateColor
import androidx.compose.animation.core.animateDp
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.updateTransition
import androidx.compose.material3.MaterialTheme
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.unit.dp

@Composable
fun updateCardTransitionData(
    state: ExpandableCardState
): CardTransitionData {
    val transition = updateTransition(
        targetState = state,
        label = "card transition data"
    )

    val height by transition.animateDp(
        label = "height"
    ) { targetState ->
        when (targetState) {
            ExpandableCardState.Collapsed -> 96.dp
            ExpandableCardState.Expanded -> 180.dp
        }
    }

    val color by transition.animateColor(
        label = "color"
    ) { targetState ->
        when (targetState) {
            ExpandableCardState.Collapsed -> MaterialTheme.colorScheme.surface
            ExpandableCardState.Expanded -> MaterialTheme.colorScheme.primaryContainer
        }
    }

    val alpha by transition.animateFloat(
        label = "alpha"
    ) { targetState ->
        when (targetState) {
            ExpandableCardState.Collapsed -> 0f
            ExpandableCardState.Expanded -> 1f
        }
    }

    val rotation by transition.animateFloat(
        label = "rotation"
    ) { targetState ->
        when (targetState) {
            ExpandableCardState.Collapsed -> 0f
            ExpandableCardState.Expanded -> 180f
        }
    }

    return CardTransitionData(
        height = height,
        containerColor = color,
        detailsAlpha = alpha,
        arrowRotation = rotation
    )
}
```

Теперь UI-компонент читает готовые values:

```kotlin
@Composable
fun ReusableTransitionCard(
    state: ExpandableCardState,
    title: String,
    details: String
) {
    val transitionData = updateCardTransitionData(state)

    Card(
        modifier = Modifier.height(transitionData.height),
        colors = androidx.compose.material3.CardDefaults.cardColors(
            containerColor = transitionData.containerColor
        )
    ) {
        Column {
            Text(text = title)
            Text(
                text = "⌄",
                modifier = Modifier.graphicsLayer {
                    rotationZ = transitionData.arrowRotation
                }
            )
            Text(
                text = details,
                modifier = Modifier.alpha(transitionData.detailsAlpha)
            )
        }
    }
}
```

Такой подход упрощает testing и preview сложного component motion.

## Child transitions

`Transition` может создавать child transitions для подкомпонентов. Это полезно, когда parent state сложный, а child нужен только boolean или маленький enum.

```kotlin
import androidx.compose.animation.core.createChildTransition
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.updateTransition
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.draw.alpha

enum class ProfileHeaderState {
    Compact,
    Expanded,
    Editing
}

@Composable
fun ProfileHeader(
    state: ProfileHeaderState
) {
    val transition = updateTransition(
        targetState = state,
        label = "profile header"
    )

    val actionsTransition = transition.createChildTransition(
        label = "actions visible"
    ) { targetState ->
        targetState != ProfileHeaderState.Compact
    }

    HeaderActions(
        transition = actionsTransition
    )
}

@Composable
fun HeaderActions(
    transition: androidx.compose.animation.core.Transition<Boolean>
) {
    val alpha by transition.animateFloat(
        label = "actions alpha"
    ) { visible ->
        if (visible) {
            1f
        } else {
            0f
        }
    }

    androidx.compose.material3.Text(
        text = "Действия",
        modifier = Modifier.alpha(alpha)
    )
}
```

Child transition сохраняет связь с parent transition, но дает child-компоненту более простой state.

## `AnimatedVisibility` внутри `Transition`

Когда visibility является частью transition state machine, можно использовать extension на `Transition`.

```kotlin
import androidx.compose.animation.AnimatedVisibility
import androidx.compose.animation.core.updateTransition
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable

@Composable
fun TransitionVisibilityExample(
    state: ExpandableCardState
) {
    val transition = updateTransition(
        targetState = state,
        label = "visibility transition"
    )

    transition.AnimatedVisibility(
        visible = { targetState ->
            targetState == ExpandableCardState.Expanded
        }
    ) {
        Text(text = "Дополнительный контент")
    }
}
```

Так `AnimatedVisibility` использует тот же transition state, что и другие animated values.

## Production-пример: download action button

Кнопка скачивания имеет несколько состояний: idle, downloading, completed, failed. У каждого состояния свой цвет, ширина, icon alpha и progress alpha.

```kotlin
import androidx.compose.runtime.Immutable

@Immutable
sealed interface DownloadButtonState {
    data object Idle : DownloadButtonState
    data class Downloading(val progress: Float) : DownloadButtonState
    data object Completed : DownloadButtonState
    data object Failed : DownloadButtonState
}

enum class DownloadButtonMotionState {
    Idle,
    Downloading,
    Completed,
    Failed
}

fun DownloadButtonState.toMotionState(): DownloadButtonMotionState {
    return when (this) {
        DownloadButtonState.Idle -> DownloadButtonMotionState.Idle
        is DownloadButtonState.Downloading -> DownloadButtonMotionState.Downloading
        DownloadButtonState.Completed -> DownloadButtonMotionState.Completed
        DownloadButtonState.Failed -> DownloadButtonMotionState.Failed
    }
}
```

```kotlin
import androidx.compose.animation.animateColor
import androidx.compose.animation.core.animateDp
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.tween
import androidx.compose.animation.core.updateTransition
import androidx.compose.foundation.layout.Row
import androidx.compose.foundation.layout.width
import androidx.compose.material3.Button
import androidx.compose.material3.CircularProgressIndicator
import androidx.compose.material3.MaterialTheme
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.draw.alpha
import androidx.compose.ui.unit.dp

@Composable
fun DownloadButton(
    state: DownloadButtonState,
    onClick: () -> Unit
) {
    val motionState = state.toMotionState()
    val transition = updateTransition(
        targetState = motionState,
        label = "download button"
    )

    val width by transition.animateDp(
        transitionSpec = {
            tween(durationMillis = 220)
        },
        label = "button width"
    ) { targetState ->
        when (targetState) {
            DownloadButtonMotionState.Idle -> 160.dp
            DownloadButtonMotionState.Downloading -> 190.dp
            DownloadButtonMotionState.Completed -> 170.dp
            DownloadButtonMotionState.Failed -> 180.dp
        }
    }

    val containerColor by transition.animateColor(
        label = "button color"
    ) { targetState ->
        when (targetState) {
            DownloadButtonMotionState.Idle -> MaterialTheme.colorScheme.primary
            DownloadButtonMotionState.Downloading -> MaterialTheme.colorScheme.secondary
            DownloadButtonMotionState.Completed -> MaterialTheme.colorScheme.tertiary
            DownloadButtonMotionState.Failed -> MaterialTheme.colorScheme.error
        }
    }

    val progressAlpha by transition.animateFloat(
        label = "progress alpha"
    ) { targetState ->
        if (targetState == DownloadButtonMotionState.Downloading) {
            1f
        } else {
            0f
        }
    }

    Button(
        onClick = onClick,
        modifier = Modifier.width(width),
        colors = androidx.compose.material3.ButtonDefaults.buttonColors(
            containerColor = containerColor
        ),
        enabled = motionState != DownloadButtonMotionState.Downloading
    ) {
        Row {
            Text(
                text = when (state) {
                    DownloadButtonState.Idle -> "Скачать"
                    is DownloadButtonState.Downloading -> "${(state.progress * 100).toInt()}%"
                    DownloadButtonState.Completed -> "Готово"
                    DownloadButtonState.Failed -> "Повторить"
                }
            )

            CircularProgressIndicator(
                modifier = Modifier.alpha(progressAlpha)
            )
        }
    }
}
```

Progress text обновляется из реального `state.progress`, но motion state не меняется при каждом проценте. Поэтому button width и color не перезапускают transition на каждый progress tick.

## Production-пример: adaptive navigation surface

Навигационная поверхность может иметь режимы bottom bar, rail и drawer. Motion values можно связать с одним state.

```kotlin
enum class NavigationSurfaceMode {
    Bar,
    Rail,
    Drawer
}
```

```kotlin
import androidx.compose.animation.core.animateDp
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.updateTransition
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.width
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.draw.alpha
import androidx.compose.ui.unit.dp

@Composable
fun AdaptiveNavigationContainer(
    mode: NavigationSurfaceMode
) {
    val transition = updateTransition(
        targetState = mode,
        label = "adaptive navigation"
    )

    val width by transition.animateDp(
        label = "navigation width"
    ) { targetMode ->
        when (targetMode) {
            NavigationSurfaceMode.Bar -> 0.dp
            NavigationSurfaceMode.Rail -> 80.dp
            NavigationSurfaceMode.Drawer -> 304.dp
        }
    }

    val labelAlpha by transition.animateFloat(
        label = "navigation label alpha"
    ) { targetMode ->
        when (targetMode) {
            NavigationSurfaceMode.Bar -> 0f
            NavigationSurfaceMode.Rail -> 0f
            NavigationSurfaceMode.Drawer -> 1f
        }
    }

    Box(
        modifier = Modifier.width(width)
    ) {
        Text(
            text = "Разделы",
            modifier = Modifier.alpha(labelAlpha)
        )
    }
}
```

Для реального adaptive UI переход между bar и rail может происходить при изменении window size class. Проверьте, нужна ли анимация при resize на больших экранах: иногда мгновенная адаптация выглядит лучше.

## Performance

`updateTransition` сам по себе не является дорогим. Стоимость зависит от того, что меняют animated values.

Практические правила:

- держите state маленьким и стабильным;
- используйте motion state, если business state меняется часто;
- не запускайте layout-heavy animations без необходимости;
- выносите сложные transitions в reusable holder;
- задавайте `label` каждому animated value;
- не создавайте expensive objects внутри target value lambda;
- не используйте transition как замену state management.

## Тестирование

`Transition` можно тестировать через UI tests и test clock. Обычно проверяют final state, а не каждый кадр.

```kotlin
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.updateTransition
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.draw.alpha
import androidx.compose.ui.platform.testTag

@Composable
fun TestableTransitionText(
    state: ExpandableCardState
) {
    val transition = updateTransition(
        targetState = state,
        label = "test transition"
    )

    val alpha by transition.animateFloat(
        label = "test alpha"
    ) { targetState ->
        when (targetState) {
            ExpandableCardState.Collapsed -> 0f
            ExpandableCardState.Expanded -> 1f
        }
    }

    Text(
        text = "Детали",
        modifier = Modifier
            .alpha(alpha)
            .testTag("details")
    )
}
```

В screenshot tests удобно фиксировать state без анимаций или продвигать clock к final position.

## Типичные ошибки

### Использовать несколько booleans вместо state machine

Если component mode имеет несколько взаимоисключающих состояний, enum или sealed interface почти всегда яснее.

### Анимировать от полного UI state

Если UI state часто меняется данными, transition может перезапускаться слишком часто. Выделите motion state.

### Использовать `updateTransition` для одного значения

Для одного значения проще `animate*AsState`. `updateTransition` нужен, когда есть координация.

### Прятать business logic в target lambda

Lambda должна mapping state -> visual value. Не ходите там в repository и не выполняйте тяжелые расчеты.

### Забывать labels

У transition и каждого child animation должен быть понятный label. Это облегчает debugging.

### Перегружать один transition всем экраном

Transition компонента должен описывать motion компонента. Не превращайте его в глобальный animation manager экрана.

## Итоги

`updateTransition` нужен для state-machine motion:

- создает `Transition` на основе `targetState`;
- управляет несколькими animated values;
- поддерживает `animateFloat`, `animateDp`, `animateColor`, `animateValue` и другие типы;
- `transitionSpec` может зависеть от segment через `isTransitioningTo`;
- enum и sealed interface делают motion state явным;
- child transitions помогают раздавать подкомпонентам упрощенный state;
- reusable transition data отделяет motion logic от layout;
- для часто меняющегося business state лучше выделять motion state;
- для одного значения чаще достаточно `animate*AsState`;
- для coroutine sequence и gestures нужен `Animatable`.
