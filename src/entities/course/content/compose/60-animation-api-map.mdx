## Почему в Compose так много animation API

Compose дает много animation API, потому что анимации бывают разными по природе. Иногда нужно плавно изменить один цвет. Иногда нужно показать или скрыть целый блок. Иногда состояние экрана меняется между несколькими режимами, и сразу несколько параметров должны анимироваться согласованно. Иногда анимация управляется жестом или coroutine. Иногда она бесконечно повторяется как loading indicator.

Главный навык — выбрать самый простой API, который точно выражает задачу:

- одно значение зависит от target state: `animate*AsState`;
- нужно показать или убрать composable: `AnimatedVisibility`;
- нужно заменить один content другим: `AnimatedContent` или `Crossfade`;
- нужно анимировать изменение размера layout: `Modifier.animateContentSize`;
- несколько значений зависят от одного state: `updateTransition`;
- нужна coroutine-driven анимация: `Animatable`;
- нужна бесконечная анимация: `rememberInfiniteTransition`;
- нужна анимация между navigation destinations: transitions в `composable`;
- нужна asset-based анимация: animated vector или Lottie-like library.

```kotlin
import androidx.compose.animation.animateColorAsState
import androidx.compose.foundation.background
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.material3.MaterialTheme
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun SelectedColorBox(
    selected: Boolean
) {
    val targetColor = if (selected) {
        MaterialTheme.colorScheme.primary
    } else {
        Color.LightGray
    }

    val color by animateColorAsState(
        targetValue = targetColor,
        label = "selected box color"
    )

    Box(
        modifier = Modifier
            .size(64.dp)
            .background(color)
    )
}
```

Этот пример использует самый простой подход: есть target color, Compose сам создает и ведет animation state.

<CatTip>
Если можно решить задачу `animate*AsState`, начните с него. Более низкоуровневые API нужны не потому, что они "профессиональнее", а потому что у них больше ручного управления.
</CatTip>

## Ментальная модель анимации

В Compose анимация обычно связывает текущее значение и target value. Когда target value меняется, animation API плавно интерполирует значение во времени. UI читает animated value как обычный state, поэтому recomposition или draw обновляют картинку.

Ключевые понятия:

- `targetValue`: значение, к которому нужно прийти;
- animated value: текущее промежуточное значение;
- `AnimationSpec`: правило движения;
- `label`: имя анимации для tooling;
- interruption: что происходит, если target меняется до завершения;
- visibility transition: анимация входа и выхода content;
- transition state: состояние, вокруг которого синхронизируются несколько animated values.

Compose animation APIs стараются быть declarative: вы описываете target, а runtime заботится о кадрах.

## animate*AsState

`animate*AsState` — семейство API для анимации одного значения. Оно подходит, когда значение полностью определяется текущим state.

Частые варианты:

- `animateFloatAsState`;
- `animateDpAsState`;
- `animateColorAsState`;
- `animateIntAsState`;
- `animateIntOffsetAsState`;
- `animateOffsetAsState`;
- `animateRectAsState`;
- `animateSizeAsState`;
- `animateValueAsState`.

```kotlin
import androidx.compose.animation.core.animateDpAsState
import androidx.compose.animation.core.spring
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.height
import androidx.compose.foundation.layout.width
import androidx.compose.material3.Card
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.unit.dp

@Composable
fun ExpandableMetricCard(
    expanded: Boolean,
    title: String
) {
    val width by animateDpAsState(
        targetValue = if (expanded) {
            240.dp
        } else {
            140.dp
        },
        animationSpec = spring(),
        label = "metric card width"
    )

    Card(
        modifier = Modifier
            .width(width)
            .height(96.dp)
    ) {
        Box {
            Text(text = title)
        }
    }
}
```

`animate*AsState` сам запоминает animation state. Когда `expanded` меняется, target width меняется, и анимация стартует автоматически.

## animationSpec

`AnimationSpec` описывает, как значение движется к target.

Основные specs:

- `tween`: анимация заданной длительности;
- `spring`: физическая пружина;
- `keyframes`: ключевые кадры;
- `keyframesWithSpline`: spline-based keyframes;
- `snap`: мгновенная смена без плавной анимации;
- `repeatable`: конечное повторение;
- `infiniteRepeatable`: бесконечное повторение;
- `decay`: затухающая анимация, часто для fling.

```kotlin
import androidx.compose.animation.core.FastOutSlowInEasing
import androidx.compose.animation.core.animateFloatAsState
import androidx.compose.animation.core.tween
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.draw.alpha
import androidx.compose.ui.unit.dp

@Composable
fun FadingContent(
    enabled: Boolean
) {
    val alpha by animateFloatAsState(
        targetValue = if (enabled) {
            1f
        } else {
            0.38f
        },
        animationSpec = tween(
            durationMillis = 220,
            easing = FastOutSlowInEasing
        ),
        label = "content alpha"
    )

    Box(
        modifier = Modifier
            .size(80.dp)
            .alpha(alpha)
    )
}
```

`tween` хорош для predictable UI motion: fade, color, small position changes. `spring` хорош, когда движение должно реагировать живее и естественнее.

## AnimatedVisibility

`AnimatedVisibility` используется, когда composable должен появиться или исчезнуть. Важно: когда exit animation завершается, content удаляется из composition.

```kotlin
import androidx.compose.animation.AnimatedVisibility
import androidx.compose.animation.expandVertically
import androidx.compose.animation.fadeIn
import androidx.compose.animation.fadeOut
import androidx.compose.animation.shrinkVertically
import androidx.compose.foundation.layout.Column
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable

@Composable
fun ErrorDetails(
    visible: Boolean,
    message: String,
    details: String
) {
    Column {
        Text(text = message)

        AnimatedVisibility(
            visible = visible,
            enter = fadeIn() + expandVertically(),
            exit = shrinkVertically() + fadeOut(),
            label = "error details visibility"
        ) {
            Text(text = details)
        }
    }
}
```

Используйте `AnimatedVisibility`, когда presence content меняется. Если content всегда должен оставаться в composition, но быть прозрачным, используйте alpha animation.

<MouseTrap>
Не используйте `AnimatedVisibility` для скрытия важного state holder, если удаление из composition сбрасывает нужный remembered state. В таком случае либо hoist state выше, либо используйте другой способ визуального скрытия.
</MouseTrap>

## Modifier.animateEnterExit

Дочерние элементы внутри `AnimatedVisibility` могут иметь собственные enter/exit transitions.

```kotlin
import androidx.compose.animation.AnimatedVisibility
import androidx.compose.animation.animateEnterExit
import androidx.compose.animation.fadeIn
import androidx.compose.animation.fadeOut
import androidx.compose.animation.slideInVertically
import androidx.compose.animation.slideOutVertically
import androidx.compose.foundation.layout.Column
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier

@Composable
fun NotificationBanner(
    visible: Boolean,
    title: String,
    body: String
) {
    AnimatedVisibility(
        visible = visible,
        enter = fadeIn(),
        exit = fadeOut(),
        label = "notification banner"
    ) {
        Column {
            Text(text = title)
            Text(
                text = body,
                modifier = Modifier.animateEnterExit(
                    enter = slideInVertically { fullHeight ->
                        fullHeight / 2
                    },
                    exit = slideOutVertically { fullHeight ->
                        fullHeight / 2
                    }
                )
            )
        }
    }
}
```

Так можно сделать общий fade для контейнера и отдельный slide для текста.

## AnimatedContent

`AnimatedContent` подходит, когда один content заменяется другим на основании `targetState`. Например, loading/content/error, stepper, счетчик, режим карточки.

```kotlin
import androidx.compose.animation.AnimatedContent
import androidx.compose.animation.SizeTransform
import androidx.compose.animation.fadeIn
import androidx.compose.animation.fadeOut
import androidx.compose.animation.togetherWith
import androidx.compose.material3.CircularProgressIndicator
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable

sealed interface LoadState {
    data object Loading : LoadState
    data class Content(val title: String) : LoadState
    data class Error(val message: String) : LoadState
}

@Composable
fun LoadStateContent(
    state: LoadState
) {
    AnimatedContent(
        targetState = state,
        transitionSpec = {
            fadeIn() togetherWith fadeOut() using SizeTransform(
                clip = false
            )
        },
        label = "load state content"
    ) { targetState ->
        when (targetState) {
            LoadState.Loading -> {
                CircularProgressIndicator()
            }

            is LoadState.Content -> {
                Text(text = targetState.title)
            }

            is LoadState.Error -> {
                Text(text = targetState.message)
            }
        }
    }
}
```

`AnimatedContent` дает доступ к initial и target states внутри `transitionSpec`, поэтому можно делать direction-aware transitions.

## Crossfade

`Crossfade` — более простой API для смены content через fade. Он хорош, когда не нужны сложные enter/exit transitions.

```kotlin
import androidx.compose.animation.Crossfade
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable

@Composable
fun UserStatusText(
    online: Boolean
) {
    Crossfade(
        targetState = online,
        label = "user status crossfade"
    ) { isOnline ->
        Text(
            text = if (isOnline) {
                "Пользователь онлайн"
            } else {
                "Пользователь офлайн"
            }
        )
    }
}
```

Если нужно контролировать size transform, slide или разные transitions для разных направлений, выбирайте `AnimatedContent`.

## Modifier.animateContentSize

`Modifier.animateContentSize` анимирует изменение размера content. Он полезен для раскрывающихся карточек, текстовых блоков и layout, где size меняется из-за content.

```kotlin
import androidx.compose.animation.animateContentSize
import androidx.compose.foundation.clickable
import androidx.compose.foundation.layout.Column
import androidx.compose.foundation.layout.padding
import androidx.compose.material3.Card
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.unit.dp

@Composable
fun ExpandableTextCard(
    title: String,
    body: String,
    expanded: Boolean,
    onClick: () -> Unit
) {
    Card(
        modifier = Modifier
            .clickable(onClick = onClick)
            .animateContentSize()
    ) {
        Column(
            modifier = Modifier.padding(16.dp)
        ) {
            Text(text = title)

            if (expanded) {
                Text(text = body)
            }
        }
    }
}
```

Размещайте `animateContentSize` перед size modifiers, если хотите анимировать изменения, вызванные content. Порядок modifiers влияет на результат.

## updateTransition

`updateTransition` нужен, когда несколько animated values зависят от одного state и должны быть согласованы. Например, карточка имеет состояния `Collapsed`, `Expanded`, `Selected`.

```kotlin
import androidx.compose.animation.core.updateTransition
import androidx.compose.animation.core.animateDp
import androidx.compose.animation.core.animateFloat
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.height
import androidx.compose.foundation.layout.width
import androidx.compose.material3.Card
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.draw.alpha
import androidx.compose.ui.unit.dp

enum class CardMode {
    Collapsed,
    Expanded
}

@Composable
fun TransitionCard(
    mode: CardMode,
    title: String
) {
    val transition = updateTransition(
        targetState = mode,
        label = "card mode transition"
    )

    val width by transition.animateDp(
        label = "card width"
    ) { targetMode ->
        when (targetMode) {
            CardMode.Collapsed -> 160.dp
            CardMode.Expanded -> 280.dp
        }
    }

    val detailsAlpha by transition.animateFloat(
        label = "details alpha"
    ) { targetMode ->
        when (targetMode) {
            CardMode.Collapsed -> 0f
            CardMode.Expanded -> 1f
        }
    }

    Card(
        modifier = Modifier
            .width(width)
            .height(120.dp)
    ) {
        Box {
            Text(text = title)
            Text(
                text = "Подробности",
                modifier = Modifier.alpha(detailsAlpha)
            )
        }
    }
}
```

Если сделать это двумя независимыми `animate*AsState`, код тоже может работать. Но `updateTransition` лучше выражает идею: несколько значений являются частью одного перехода состояния.

## Transition с sealed state

State machine можно описывать sealed interface. Это удобно для сложных компонентов.

```kotlin
sealed interface PlayerPanelState {
    data object Minimized : PlayerPanelState
    data object Compact : PlayerPanelState
    data object Expanded : PlayerPanelState
}
```

```kotlin
import androidx.compose.animation.core.animateDp
import androidx.compose.animation.core.updateTransition
import androidx.compose.foundation.layout.height
import androidx.compose.material3.Surface
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.unit.dp

@Composable
fun PlayerPanel(
    state: PlayerPanelState,
    title: String
) {
    val transition = updateTransition(
        targetState = state,
        label = "player panel transition"
    )

    val height by transition.animateDp(
        label = "player panel height"
    ) { targetState ->
        when (targetState) {
            PlayerPanelState.Minimized -> 56.dp
            PlayerPanelState.Compact -> 96.dp
            PlayerPanelState.Expanded -> 320.dp
        }
    }

    Surface(
        modifier = Modifier.height(height)
    ) {
        Text(text = title)
    }
}
```

Для component state machine такой код читается лучше, чем набор отдельных booleans.

## Animatable

`Animatable` — coroutine-driven анимация. Она нужна, когда анимацию нужно запускать вручную, отменять, связывать с gestures, делать sequence или вызывать `snapTo`.

```kotlin
import androidx.compose.animation.core.Animatable
import androidx.compose.animation.core.spring
import androidx.compose.foundation.gestures.detectTapGestures
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.material3.Surface
import androidx.compose.runtime.Composable
import androidx.compose.runtime.remember
import androidx.compose.runtime.rememberCoroutineScope
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.input.pointer.pointerInput
import androidx.compose.ui.unit.dp
import kotlinx.coroutines.launch

@Composable
fun PressScaleSurface() {
    val scale = remember {
        Animatable(1f)
    }
    val scope = rememberCoroutineScope()

    Surface(
        modifier = Modifier
            .size(96.dp)
            .graphicsLayer {
                scaleX = scale.value
                scaleY = scale.value
            }
            .pointerInput(Unit) {
                detectTapGestures(
                    onPress = {
                        scope.launch {
                            scale.animateTo(
                                targetValue = 0.94f,
                                animationSpec = spring()
                            )
                        }

                        tryAwaitRelease()

                        scope.launch {
                            scale.animateTo(
                                targetValue = 1f,
                                animationSpec = spring()
                            )
                        }
                    }
                )
            }
    ) {
        Box(
            modifier = Modifier.size(1.dp)
        )
    }
}
```

`Animatable` хранит текущее значение и управляет анимацией через suspend functions. Если новая анимация стартует до завершения старой, старая отменяется.

<MouseTrap>
Не используйте `Animatable` там, где достаточно `animate*AsState`. Ручная coroutine-анимация дает контроль, но требует аккуратно думать о scope, cancellation и restart.
</MouseTrap>

## rememberInfiniteTransition

`rememberInfiniteTransition` используется для бесконечных анимаций: shimmer, pulsing indicator, breathing highlight. Такие анимации должны быть экономными и не мешать пользователю.

```kotlin
import androidx.compose.animation.core.RepeatMode
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.infiniteRepeatable
import androidx.compose.animation.core.rememberInfiniteTransition
import androidx.compose.animation.core.tween
import androidx.compose.foundation.background
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.height
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.draw.alpha
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun PulsingLoadingBar() {
    val transition = rememberInfiniteTransition(
        label = "loading pulse transition"
    )

    val alpha by transition.animateFloat(
        initialValue = 0.35f,
        targetValue = 1f,
        animationSpec = infiniteRepeatable(
            animation = tween(durationMillis = 700),
            repeatMode = RepeatMode.Reverse
        ),
        label = "loading pulse alpha"
    )

    Box(
        modifier = Modifier
            .fillMaxWidth()
            .height(12.dp)
            .alpha(alpha)
            .background(Color.LightGray)
    )
}
```

Бесконечная анимация живет, пока composable находится в composition. Если indicator больше не нужен, уберите его из composition.

## Navigation transitions

Navigation Compose поддерживает transitions между destinations. Их используют, когда motion помогает понять пространственную связь экранов.

```kotlin
import androidx.compose.animation.EnterTransition
import androidx.compose.animation.ExitTransition
import androidx.compose.animation.fadeIn
import androidx.compose.animation.fadeOut
import androidx.compose.animation.slideInHorizontally
import androidx.compose.animation.slideOutHorizontally
import androidx.navigation.compose.NavHost
import androidx.navigation.compose.composable
import androidx.navigation.compose.rememberNavController

@Composable
fun AnimatedNavigationExample() {
    val navController = rememberNavController()

    NavHost(
        navController = navController,
        startDestination = FeedRoute
    ) {
        composable<FeedRoute>(
            enterTransition = {
                fadeIn()
            },
            exitTransition = {
                fadeOut()
            }
        ) {
            androidx.compose.material3.Text(text = "Лента")
        }

        composable<ArticleRoute>(
            enterTransition = {
                slideInHorizontally { fullWidth ->
                    fullWidth
                } + fadeIn()
            },
            exitTransition = {
                slideOutHorizontally { fullWidth ->
                    -fullWidth / 3
                } + fadeOut()
            },
            popEnterTransition = {
                EnterTransition.None
            },
            popExitTransition = {
                ExitTransition.None
            }
        ) {
            androidx.compose.material3.Text(text = "Статья")
        }
    }
}
```

Не анимируйте все переходы одинаково только потому, что API это позволяет. Motion должен поддерживать навигационную модель, а не отвлекать.

## AnimationVector и animateValueAsState

Если нужно анимировать собственный тип, используйте `animateValueAsState` и `TwoWayConverter`.

```kotlin
import androidx.compose.animation.core.TwoWayConverter
import androidx.compose.animation.core.animateValueAsState
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.geometry.Offset

data class ChartPoint(
    val x: Float,
    val y: Float
)

val ChartPointConverter = TwoWayConverter<ChartPoint, androidx.compose.animation.core.AnimationVector2D>(
    convertToVector = { point ->
        androidx.compose.animation.core.AnimationVector2D(
            v1 = point.x,
            v2 = point.y
        )
    },
    convertFromVector = { vector ->
        ChartPoint(
            x = vector.v1,
            y = vector.v2
        )
    }
)

@Composable
fun rememberAnimatedChartPoint(
    target: ChartPoint
): ChartPoint {
    val point by animateValueAsState(
        targetValue = target,
        typeConverter = ChartPointConverter,
        label = "chart point"
    )

    return point
}
```

Для простых типов используйте готовые converters. Для собственных value types определите, как тип превращается в animation vector и обратно.

## Performance considerations

Анимации могут быть дешевыми или дорогими. Дешевые обычно меняют draw-параметры: alpha, color, scale, translation. Более дорогие могут менять layout size, вызывать repeated measurement или перерисовывать сложный Canvas.

Практические правила:

- не анимируйте layout без необходимости;
- предпочитайте `graphicsLayer` для scale/translation/alpha;
- не создавайте тяжелые objects на каждый frame;
- используйте `drawWithCache` для дорогих brushes и paths;
- задавайте `label`, чтобы отлаживать animation tooling;
- проверяйте jank на реальном устройстве;
- уважайте системные настройки accessibility, если анимация интенсивная.

```kotlin
import androidx.compose.animation.core.animateFloatAsState
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.graphicsLayer

@Composable
fun Modifier.animatedScale(
    selected: Boolean
): Modifier {
    val scale by animateFloatAsState(
        targetValue = if (selected) {
            1.08f
        } else {
            1f
        },
        label = "selected scale"
    )

    return graphicsLayer {
        scaleX = scale
        scaleY = scale
    }
}
```

`graphicsLayer` часто дешевле, чем изменение layout size, потому что может применяться на draw/compositing уровне.

## Как выбрать API

Короткая карта выбора:

- "Плавно поменять цвет, размер, alpha, offset": `animate*AsState`;
- "Показать или скрыть блок": `AnimatedVisibility`;
- "Сменить один content на другой": `AnimatedContent`;
- "Просто fade между состояниями": `Crossfade`;
- "Анимировать размер после изменения content": `animateContentSize`;
- "Несколько значений зависят от одного state": `updateTransition`;
- "Запустить sequence, gesture animation или snap": `Animatable`;
- "Бесконечно повторять shimmer/pulse": `rememberInfiniteTransition`;
- "Анимировать route transitions": transitions в Navigation Compose;
- "Анимировать vector asset": animated vector;
- "Сложная готовая иллюстративная анимация": специализированная animation library.

## Типичные ошибки

### Выбирать слишком низкоуровневый API

Если задача решается `animateDpAsState`, не нужен `Animatable`. Чем ниже уровень, тем больше ручного lifecycle и cancellation.

### Анимировать все подряд

Анимация должна помогать восприятию. Слишком много motion делает интерфейс утомительным.

### Забывать label

`label` помогает в tooling и отладке. Для production-кода это маленькая привычка с большим выигрышем.

### Хранить animation state не там

Если анимация зависит от screen state, target должен приходить из state. Если локальный interaction state достаточно локален, не поднимайте его в ViewModel.

### Менять layout на каждом кадре без необходимости

Layout animation может быть дороже draw animation. Если можно выразить движение через `graphicsLayer`, часто это лучше.

### Использовать infinite animation без условия

Бесконечные анимации должны исчезать из composition, когда больше не нужны. Не держите shimmer под реальным content.

## Итоги

Animation API в Compose выбирается по задаче:

- `animate*AsState` — один target value;
- `AnimatedVisibility` — появление и исчезновение content;
- `AnimatedContent` — смена content по target state;
- `Crossfade` — простая fade-смена;
- `animateContentSize` — изменение размера content;
- `updateTransition` — несколько значений от одного state;
- `Animatable` — coroutine-driven и gesture-driven animation;
- `rememberInfiniteTransition` — бесконечные loading/pulse циклы;
- `AnimationSpec` задает характер движения;
- анимация должна быть частью UX-смысла, а не декоративным шумом.
