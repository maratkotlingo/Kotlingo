## Что такое semantics

Semantics — это слой описания UI для accessibility services, тестов, autofill, AI agents и других инструментов, которым недостаточно пикселей. Compose UI может выглядеть понятно визуально, но без semantics screen reader не узнает, что custom Canvas является progress bar, card является button, а swipe action можно выполнить альтернативным способом.

Semantics отвечает на вопросы:

- что это за элемент;
- как он называется;
- можно ли с ним взаимодействовать;
- какая у него роль;
- какое у него состояние;
- какие actions доступны;
- как элементы объединяются в accessibility tree;
- в каком порядке их обходить;
- какие элементы декоративные;
- что проверять в UI tests.

Многие Material components уже задают semantics. Custom UI требует ручного внимания.

```kotlin
import androidx.compose.foundation.clickable
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.padding
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.Role
import androidx.compose.ui.semantics.contentDescription
import androidx.compose.ui.semantics.role
import androidx.compose.ui.semantics.semantics
import androidx.compose.ui.unit.dp

@Composable
fun CustomActionCard(
    title: String,
    onOpen: () -> Unit
) {
    Box(
        modifier = Modifier
            .semantics {
                contentDescription = "Карточка: $title"
                role = Role.Button
            }
            .clickable(onClick = onOpen)
            .padding(16.dp)
    ) {
        Text(text = title)
    }
}
```

В большинстве случаев role лучше задавать через interaction modifier, например `clickable(role = Role.Button)`. Явный `semantics` нужен для custom cases.

<CatTip>
Semantics — это контракт UI с пользователем, который не видит пиксели так же, как вы. Если состояние или действие важно, оно должно существовать не только визуально.
</CatTip>

## `contentDescription`

`contentDescription` описывает изображение или non-text UI. Для декоративных изображений используйте `null` в `Image`, а не пустую строку.

```kotlin
import androidx.compose.foundation.Image
import androidx.compose.runtime.Composable
import androidx.compose.ui.res.painterResource

@Composable
fun MeaningfulImage() {
    Image(
        painter = painterResource(id = R.drawable.course_cover),
        contentDescription = "Обложка курса по Jetpack Compose"
    )
}
```

```kotlin
import androidx.compose.foundation.Image
import androidx.compose.runtime.Composable
import androidx.compose.ui.res.painterResource

@Composable
fun DecorativeTexture() {
    Image(
        painter = painterResource(id = R.drawable.card_texture),
        contentDescription = null
    )
}
```

Не дублируйте text content. Если рядом уже есть `Text("Настройки")`, icon рядом с ним часто должен иметь `contentDescription = null`.

<MouseTrap>
Пустая строка и `null` — разные сигналы. `null` говорит, что элемент декоративный. Пустая строка создает странный accessibility node без полезного имени.
</MouseTrap>

## Roles

`Role` сообщает тип interaction:

- `Role.Button`;
- `Role.Checkbox`;
- `Role.Switch`;
- `Role.RadioButton`;
- `Role.Tab`;
- `Role.Image`;
- `Role.DropdownList`;
- `Role.Carousel`;
- `Role.ValuePicker`.

```kotlin
import androidx.compose.foundation.clickable
import androidx.compose.foundation.layout.Row
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.foundation.layout.padding
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.Role
import androidx.compose.ui.unit.dp

@Composable
fun SettingsRow(
    title: String,
    onOpen: () -> Unit
) {
    Row(
        modifier = Modifier
            .fillMaxWidth()
            .clickable(
                role = Role.Button,
                onClickLabel = "Открыть",
                onClick = onOpen
            )
            .padding(16.dp)
    ) {
        Text(text = title)
    }
}
```

Role помогает service произносить правильную подсказку: activate, toggle, select и другие действия.

## `stateDescription`

`stateDescription` описывает состояние на человеческом языке. Это полезно для custom toggles, progress states, connection status и domain-specific values.

```kotlin
import androidx.compose.foundation.layout.Row
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.foundation.layout.padding
import androidx.compose.foundation.selection.toggleable
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.Role
import androidx.compose.ui.semantics.semantics
import androidx.compose.ui.semantics.stateDescription
import androidx.compose.ui.unit.dp

@Composable
fun SubscriptionToggleRow(
    subscribed: Boolean,
    onSubscribedChange: (Boolean) -> Unit
) {
    Row(
        modifier = Modifier
            .semantics {
                stateDescription = if (subscribed) {
                    "Подписка включена"
                } else {
                    "Подписка выключена"
                }
            }
            .toggleable(
                value = subscribed,
                role = Role.Switch,
                onValueChange = onSubscribedChange
            )
            .fillMaxWidth()
            .padding(16.dp)
    ) {
        Text(text = "Уведомления курса")
    }
}
```

Сначала задавайте explicit semantics, затем interaction modifier, если нужно переопределить default state labels.

## Progress semantics

Custom progress bar на `Canvas` должен задавать `progressBarRangeInfo`.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.foundation.layout.height
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.semantics.ProgressBarRangeInfo
import androidx.compose.ui.semantics.progressBarRangeInfo
import androidx.compose.ui.semantics.semantics
import androidx.compose.ui.unit.dp

@Composable
fun AccessibleLinearProgress(
    progress: Float
) {
    val clamped = progress.coerceIn(0f, 1f)

    Canvas(
        modifier = Modifier
            .fillMaxWidth()
            .height(12.dp)
            .semantics {
                progressBarRangeInfo = ProgressBarRangeInfo(
                    current = clamped,
                    range = 0f..1f,
                    steps = 0
                )
            }
    ) {
        drawRect(color = Color(0xFFE0E0E0))
        drawRect(
            color = Color(0xFF1976D2),
            size = size.copy(
                width = size.width * clamped
            )
        )
    }
}
```

Screen reader не распознает нарисованный прямоугольник как progress bar без semantics.

## Error semantics

`error` позволяет добавить расширенное сообщение об ошибке.

```kotlin
import androidx.compose.material3.OutlinedTextField
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.error
import androidx.compose.ui.semantics.semantics

@Composable
fun EmailFieldWithError(
    email: String,
    onEmailChange: (String) -> Unit,
    invalid: Boolean
) {
    OutlinedTextField(
        value = email,
        onValueChange = onEmailChange,
        isError = invalid,
        label = {
            Text(text = "Email")
        },
        supportingText = {
            if (invalid) {
                Text(text = "Введите адрес в формате name@example.com")
            }
        },
        modifier = Modifier.semantics {
            if (invalid) {
                error("Email должен содержать имя, символ @ и домен")
            }
        }
    )
}
```

Visible error text и semantic error должны дополнять друг друга.

## Custom accessibility actions

Custom action делает сложный gesture доступным через accessibility service.

```kotlin
import androidx.compose.material3.ListItem
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.CustomAccessibilityAction
import androidx.compose.ui.semantics.customActions
import androidx.compose.ui.semantics.semantics

@Composable
fun DismissibleItemSemantics(
    title: String,
    onDismiss: () -> Unit
) {
    ListItem(
        headlineContent = {
            Text(text = title)
        },
        modifier = Modifier.semantics {
            customActions = listOf(
                CustomAccessibilityAction(
                    label = "Удалить",
                    action = {
                        onDismiss()
                        true
                    }
                )
            )
        }
    )
}
```

Используйте custom actions для swipe, drag-and-drop, reorder и hidden item actions.

## Collection semantics

Custom lists и grids могут сообщать количество элементов и позицию item.

```kotlin
import androidx.compose.foundation.layout.Column
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.CollectionInfo
import androidx.compose.ui.semantics.CollectionItemInfo
import androidx.compose.ui.semantics.collectionInfo
import androidx.compose.ui.semantics.collectionItemInfo
import androidx.compose.ui.semantics.semantics

@Composable
fun AccessibleCustomList(
    items: List<String>
) {
    Column(
        modifier = Modifier.semantics {
            collectionInfo = CollectionInfo(
                rowCount = items.size,
                columnCount = 1
            )
        }
    ) {
        items.forEachIndexed { index, item ->
            Text(
                text = item,
                modifier = Modifier.semantics {
                    collectionItemInfo = CollectionItemInfo(
                        rowIndex = index,
                        rowSpan = 1,
                        columnIndex = 0,
                        columnSpan = 1
                    )
                }
            )
        }
    }
}
```

`LazyColumn` и Material components часто уже дают хорошую baseline semantics. Custom layout требует ручной настройки.

## Merge descendants

`mergeDescendants = true` объединяет semantics descendants в один node.

```kotlin
import androidx.compose.foundation.clickable
import androidx.compose.foundation.layout.Column
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.foundation.layout.padding
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.Role
import androidx.compose.ui.semantics.semantics
import androidx.compose.ui.unit.dp

@Composable
fun MergedArticleRow(
    title: String,
    subtitle: String,
    onOpen: () -> Unit
) {
    Column(
        modifier = Modifier
            .fillMaxWidth()
            .semantics(mergeDescendants = true) { }
            .clickable(
                role = Role.Button,
                onClick = onOpen
            )
            .padding(16.dp)
    ) {
        Text(text = title)
        Text(text = subtitle)
    }
}
```

Merging полезен для строки, которая воспринимается как один item. Не объединяйте элементы, если каждый child должен быть отдельной action target.

## `clearAndSetSemantics`

`clearAndSetSemantics` удаляет semantics descendants и задает новые. Используйте его осторожно.

```kotlin
import androidx.compose.foundation.clickable
import androidx.compose.foundation.layout.Row
import androidx.compose.foundation.layout.padding
import androidx.compose.material.icons.Icons
import androidx.compose.material.icons.filled.Star
import androidx.compose.material.icons.outlined.StarBorder
import androidx.compose.material3.Icon
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.Role
import androidx.compose.ui.semantics.clearAndSetSemantics
import androidx.compose.ui.semantics.contentDescription
import androidx.compose.ui.semantics.role
import androidx.compose.ui.semantics.stateDescription
import androidx.compose.ui.unit.dp

@Composable
fun FavoriteRowSemantics(
    title: String,
    favorite: Boolean,
    onToggle: () -> Unit
) {
    Row(
        modifier = Modifier
            .clearAndSetSemantics {
                contentDescription = title
                stateDescription = if (favorite) {
                    "В избранном"
                } else {
                    "Не в избранном"
                }
                role = Role.Checkbox
            }
            .clickable(onClick = onToggle)
            .padding(16.dp)
    ) {
        Icon(
            imageVector = if (favorite) {
                Icons.Default.Star
            } else {
                Icons.Outlined.StarBorder
            },
            contentDescription = null
        )
        Text(text = title)
    }
}
```

Если очистить semantics без воссоздания actions и state, компонент станет менее доступным.

## `hideFromAccessibility`

`hideFromAccessibility` скрывает decorative или redundant element от accessibility services, но сохраняет semantics для некоторых других use cases, например тестов.

```kotlin
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.hideFromAccessibility
import androidx.compose.ui.semantics.semantics

@Composable
fun DecorativeSeparator() {
    Text(
        text = "•",
        modifier = Modifier.semantics {
            hideFromAccessibility()
        }
    )
}
```

Для `Image` с декоративной картинкой обычно проще использовать `contentDescription = null`.

## Traversal order

Traversal order можно настроить через `isTraversalGroup` и `traversalIndex`.

```kotlin
import androidx.compose.foundation.layout.Column
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.isTraversalGroup
import androidx.compose.ui.semantics.semantics
import androidx.compose.ui.semantics.traversalIndex

@Composable
fun CustomTraversalPanel() {
    Column(
        modifier = Modifier.semantics {
            isTraversalGroup = true
        }
    ) {
        Text(
            text = "Сначала читается это",
            modifier = Modifier.semantics {
                traversalIndex = 0f
            }
        )
        Text(
            text = "Затем читается это",
            modifier = Modifier.semantics {
                traversalIndex = 1f
            }
        )
    }
}
```

Меняйте traversal order только когда визуальная структура не соответствует желаемому чтению и layout нельзя исправить проще.

## Pane title и live regions

`paneTitle` помогает обозначить новую область, например bottom sheet. `liveRegion` помогает объявлять важные изменения без перемещения focus.

```kotlin
import androidx.compose.foundation.layout.Column
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.LiveRegionMode
import androidx.compose.ui.semantics.liveRegion
import androidx.compose.ui.semantics.paneTitle
import androidx.compose.ui.semantics.semantics

@Composable
fun StatusPane(
    message: String
) {
    Column(
        modifier = Modifier.semantics {
            paneTitle = "Панель состояния"
            liveRegion = LiveRegionMode.Polite
        }
    ) {
        Text(text = message)
    }
}
```

Live region не должна использоваться для каждого мелкого изменения. Иначе screen reader станет слишком шумным.

## Testing semantics tree

Compose UI tests работают с semantics tree.

```kotlin
import androidx.compose.ui.semantics.Role
import androidx.compose.ui.semantics.SemanticsProperties
import androidx.compose.ui.test.SemanticsMatcher
import androidx.compose.ui.test.junit4.createComposeRule
import androidx.compose.ui.test.onNodeWithContentDescription
import androidx.compose.ui.test.performClick
import com.google.common.truth.Truth.assertThat
import org.junit.Rule
import org.junit.Test

class SemanticsTest {

    @get:Rule
    val composeRule = createComposeRule()

    @Test
    fun actionCard_hasButtonRole() {
        var opened = false

        composeRule.setContent {
            CustomActionCard(
                title = "Compose",
                onOpen = {
                    opened = true
                }
            )
        }

        val semanticsNode = composeRule
            .onNodeWithContentDescription("Карточка: Compose")
            .fetchSemanticsNode()

        val hasButtonRole = SemanticsMatcher
            .expectValue(
                SemanticsProperties.Role,
                Role.Button
            )
            .matches(semanticsNode)

        assertThat(hasButtonRole).isTrue()

        composeRule
            .onNodeWithContentDescription("Карточка: Compose")
            .performClick()

        assertThat(opened).isTrue()
    }
}
```

Если тест не находит node, проверьте merged tree и unmerged tree. Некоторые semantics объединяются.

## Типичные ошибки

### Дублировать текст в `contentDescription`

Screen reader может прочитать одно и то же дважды.

### Делать custom clickable без role

Пользователь слышит generic activation вместо понятного button, checkbox или switch.

### Очищать semantics слишком широко

`clearAndSetSemantics` может удалить важные child actions.

### Скрывать interactive element

`hideFromAccessibility` не должен применяться к элементам, с которыми нужно взаимодействовать.

### Надеяться на Canvas

Canvas не создает semantics для нарисованных объектов.

### Передавать состояние только цветом

Цвет должен иметь текстовую или semantic альтернативу.

## Итоги

Semantics делает UI понятным за пределами пикселей:

- Material components дают хороший baseline, custom UI требует ручного описания;
- `contentDescription` описывает meaningful non-text content;
- decorative content лучше скрывать через `null` или `hideFromAccessibility`;
- roles помогают accessibility service выбрать правильную подсказку;
- `stateDescription` описывает состояние;
- `progressBarRangeInfo`, `collectionInfo` и `collectionItemInfo` описывают custom controls;
- `customActions` дают альтернативу сложным gestures;
- `mergeDescendants` объединяет related children;
- `clearAndSetSemantics` нужно использовать осторожно;
- traversal order можно настроить, но лучше сначала исправить layout;
- semantics tree является основой Compose UI tests.
