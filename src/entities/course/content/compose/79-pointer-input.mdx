## Что такое pointer input

Pointer input — это слой Compose, который получает события указателя: касание пальцем, движение мыши, перо, stylus, trackpad и другие pointer devices. На этом уровне вы работаете не с готовым "кликом", "перетаскиванием" или "swipe", а с сырыми изменениями указателей во времени.

В Compose есть несколько уровней input API:

- готовые компоненты: `Button`, `Slider`, `Switch`, `TextField`;
- высокоуровневые modifiers: `clickable`, `combinedClickable`, `scrollable`, `draggable`;
- gesture detectors: `detectTapGestures`, `detectDragGestures`, `detectTransformGestures`;
- низкоуровневый `pointerInput`;
- platform interop через View, если экран мигрирует постепенно.

`pointerInput` нужен, когда жест невозможно выразить более высоким API:

- пользовательский gesture с несколькими фазами;
- drawing canvas с собственным touch protocol;
- selection rectangle;
- game-like interaction;
- пользовательский scrubber;
- input, зависящий от нескольких pointer-ов;
- диагностика event passes и consumption;
- сложная координация родителя и дочерних gestures.

```kotlin
import androidx.compose.foundation.background
import androidx.compose.foundation.gestures.awaitEachGesture
import androidx.compose.foundation.gestures.awaitFirstDown
import androidx.compose.foundation.gestures.waitForUpOrCancellation
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.runtime.rememberUpdatedState
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.input.pointer.pointerInput
import androidx.compose.ui.unit.dp

@Composable
fun RawTapSurface(
    onTap: (Offset) -> Unit
) {
    val currentOnTap by rememberUpdatedState(onTap)

    Box(
        modifier = Modifier
            .size(160.dp)
            .background(Color(0xFFE3F2FD))
            .pointerInput(Unit) {
                awaitEachGesture {
                    val down = awaitFirstDown()
                    val up = waitForUpOrCancellation()

                    if (up != null) {
                        currentOnTap(down.position)
                    }
                }
            }
    )
}
```

Этот пример показывает механику. Для обычного клика используйте `clickable`, потому что он добавляет semantics, keyboard activation, focus, hover и indication.

<CatTip>
`pointerInput` — это корутинный gesture engine внутри modifier-а. Вы описываете последовательность ожиданий: дождаться down, читать движения, дождаться up или cancellation, решить, что было потреблено.
</CatTip>

## `pointerInput` как coroutine boundary

`Modifier.pointerInput` принимает ключи. При изменении ключа Compose отменяет старую pointer input coroutine и запускает новую. Это похоже на `LaunchedEffect`, но coroutine привязана к input modifier.

```kotlin
Modifier.pointerInput(key1 = enabled) {
    if (!enabled) {
        return@pointerInput
    }

    awaitEachGesture {
        val down = awaitFirstDown()
        val up = waitForUpOrCancellation()

        if (up != null) {
            onGestureFinished(down.position)
        }
    }
}
```

Что важно:

- при recomposition coroutine не перезапускается сама по себе;
- при изменении ключей coroutine отменяется;
- при удалении modifier-а из composition coroutine отменяется;
- suspend-функции внутри pointer input должны быть готовы к cancellation;
- long-running loop должен жить внутри `awaitEachGesture`, а не блокировать UI thread;
- обратные вызовы лучше читать через `rememberUpdatedState`, если ключи не должны перезапускать gesture detector.

<MouseTrap>
Не кладите изменяющийся обратный вызов напрямую в ключ `pointerInput`, если не хотите отменять gesture на каждой смене обратный вызов instance. Для актуального обратный вызов используйте `rememberUpdatedState`.
</MouseTrap>

## `PointerInputScope`

Блок `pointerInput` выполняется в `PointerInputScope`. Он дает доступ к:

- `size`: размер области pointer input в пикселях;
- `density`: conversion между dp и px;
- coroutine context;
- suspend-функциям для ожидания pointer events;
- helper APIs из foundation gestures.

```kotlin
import androidx.compose.foundation.gestures.awaitEachGesture
import androidx.compose.foundation.gestures.awaitFirstDown
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.input.pointer.pointerInput
import androidx.compose.ui.unit.dp

@Composable
fun BoundsAwarePointerArea(
    onInsideStart: () -> Unit
) {
    Box(
        modifier = Modifier
            .size(
                width = 240.dp,
                height = 120.dp
            )
            .pointerInput(Unit) {
                awaitEachGesture {
                    val down = awaitFirstDown()

                    val insideBounds =
                        down.position.x in 0f..size.width.toFloat() &&
                            down.position.y in 0f..size.height.toFloat()

                    if (insideBounds) {
                        onInsideStart()
                    }
                }
            }
    )
}
```

Pointer coordinates относятся к bounds того composable, на котором стоит modifier.

## `awaitPointerEventScope`

`awaitPointerEventScope` открывает цикл получения `PointerEvent`. Внутри него можно читать каждое событие, его changes и pass.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.mutableStateListOf
import androidx.compose.runtime.remember
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.input.pointer.pointerInput
import androidx.compose.ui.unit.dp

@Composable
fun TouchTrail() {
    val points = remember {
        mutableStateListOf<Offset>()
    }

    Canvas(
        modifier = Modifier
            .size(220.dp)
            .pointerInput(Unit) {
                awaitPointerEventScope {
                    while (true) {
                        val event = awaitPointerEvent()
                        val pressedChange = event.changes.firstOrNull { change ->
                            change.pressed
                        }

                        if (pressedChange != null) {
                            points.add(pressedChange.position)

                            if (points.size > 80) {
                                points.removeAt(0)
                            }
                        }
                    }
                }
            }
    ) {
        points.forEach { point ->
            drawCircle(
                color = Color(0xFF1976D2),
                radius = 3.dp.toPx(),
                center = point
            )
        }
    }
}
```

Такой код полезен для drawing и диагностики. Для обычного drag лучше использовать `draggable` или `detectDragGestures`.

## `awaitEachGesture`

`awaitEachGesture` оборачивает один gesture cycle. Когда все pointers отпущены или gesture отменен, цикл начинается заново.

```kotlin
import androidx.compose.foundation.gestures.awaitEachGesture
import androidx.compose.foundation.gestures.awaitFirstDown
import androidx.compose.foundation.gestures.drag
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.offset
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.runtime.mutableStateOf
import androidx.compose.runtime.remember
import androidx.compose.runtime.setValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.input.pointer.pointerInput
import androidx.compose.ui.unit.IntOffset
import androidx.compose.ui.unit.dp
import kotlin.math.roundToInt

@Composable
fun RawDragBox() {
    var offset by remember {
        mutableStateOf(Offset.Zero)
    }

    Box(
        modifier = Modifier
            .offset {
                IntOffset(
                    x = offset.x.roundToInt(),
                    y = offset.y.roundToInt()
                )
            }
            .size(72.dp)
            .pointerInput(Unit) {
                awaitEachGesture {
                    val down = awaitFirstDown()

                    drag(down.id) { change ->
                        offset += change.positionChange()
                        change.consume()
                    }
                }
            }
    )
}
```

Здесь `consume()` сообщает другим участникам input pipeline, что change уже обработан. Без consumption родитель или соседние modifiers могут интерпретировать тот же drag.

## `PointerEvent`

`PointerEvent` описывает событие на конкретном шаге input pipeline.

Полезные поля:

- `changes`: список изменений всех активных pointers;
- `buttons`: состояние кнопок мыши;
- `keyboardModifiers`: состояние modifier keys;
- `type`: тип события, например press, move, release, scroll;
- `scrollDelta`: колесо мыши или trackpad scroll, если событие scroll-related.

`PointerInputChange` внутри `changes` содержит:

- `id`: identity конкретного pointer;
- `position`: текущая позиция;
- `previousPosition`: позиция на предыдущем событии;
- `pressed`: сейчас pointer down или нет;
- `previousPressed`: был ли down на предыдущем событии;
- `positionChange()`: delta между позициями;
- `changedToDown()`, `changedToUp()`: helper-проверки переходов;
- `isConsumed`: было ли изменение потреблено;
- `consume()`: пометить изменение как потребленное.

## Pointer passes

Compose прогоняет pointer event через несколько passes:

- `PointerEventPass.Initial`: ранний проход от parent к child;
- `PointerEventPass.Main`: основной проход от child к parent;
- `PointerEventPass.Final`: финальный проход от parent к child.

Это позволяет parent и child договориться, кто обработал gesture. В большинстве кодов pass указывать не нужно. Но для advanced gestures pass помогает наблюдать, что уже потреблено.

```kotlin
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.mutableStateOf
import androidx.compose.runtime.remember
import androidx.compose.runtime.setValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.input.pointer.PointerEventPass
import androidx.compose.ui.input.pointer.pointerInput
import androidx.compose.ui.unit.dp

@Composable
fun ConsumptionDebugArea() {
    var lastMessage by remember {
        mutableStateOf("Событий пока нет")
    }

    Box(
        modifier = Modifier
            .size(180.dp)
            .pointerInput(Unit) {
                awaitPointerEventScope {
                    while (true) {
                        val event = awaitPointerEvent(
                            pass = PointerEventPass.Final
                        )
                        val consumedCount = event.changes.count { change ->
                            change.isConsumed
                        }
                        lastMessage = "Потреблено изменений: $consumedCount"
                    }
                }
            }
    )
}
```

В реальном UI вместо строки для debug-сообщения обычно используют logging, наложение или тестовую диагностику. Здесь state нужен только для наглядности.

## Consumption

Consumption не останавливает физическое событие. Оно помечает change как обработанный, чтобы другие recognizers могли принять решение.

Когда consume уместен:

- пользовательский drag уже забрал движение;
- собственное рисование уже обработал stroke;
- дочерний элемент должен победить scroll родителя;
- gesture detector перешел в активное состояние;
- изменение позиции не должно попасть в another detector.

Когда consume не нужен:

- вы только наблюдаете pointer;
- вы показываете hover highlight;
- вы собираете аналитика без влияния на gesture arbitration;
- вы не уверены, что должны блокировать parent.

```kotlin
import androidx.compose.foundation.gestures.awaitEachGesture
import androidx.compose.foundation.gestures.awaitFirstDown
import androidx.compose.foundation.gestures.drag
import androidx.compose.ui.Modifier
import androidx.compose.ui.input.pointer.pointerInput

fun Modifier.consumeHorizontalDrag(
    onDelta: (Float) -> Unit
): Modifier {
    return pointerInput(Unit) {
        awaitEachGesture {
            val down = awaitFirstDown()

            drag(down.id) { change ->
                val deltaX = change.positionChange().x

                if (deltaX != 0f) {
                    onDelta(deltaX)
                    change.consume()
                }
            }
        }
    }
}
```

Если gesture должен сосуществовать со scroll, сначала подумайте о `nestedScroll`, `draggable` или готовом компоненте.

## Cancellation

Pointer gesture может быть отменен:

- pointer вышел из допустимого сценария gesture;
- другой gesture detector потребил изменение;
- component покинул composition;
- изменился ключ `pointerInput`;
- parent перехватил поведение;
- system gesture или window focus interrupt поменял input flow.

```kotlin
import androidx.compose.foundation.gestures.awaitEachGesture
import androidx.compose.foundation.gestures.awaitFirstDown
import androidx.compose.foundation.gestures.waitForUpOrCancellation
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.runtime.rememberUpdatedState
import androidx.compose.ui.Modifier
import androidx.compose.ui.input.pointer.pointerInput
import androidx.compose.ui.unit.dp

@Composable
fun CancellablePressArea(
    onPressStarted: () -> Unit,
    onPressReleased: () -> Unit,
    onPressCanceled: () -> Unit
) {
    val currentOnPressStarted by rememberUpdatedState(onPressStarted)
    val currentOnPressReleased by rememberUpdatedState(onPressReleased)
    val currentOnPressCanceled by rememberUpdatedState(onPressCanceled)

    Box(
        modifier = Modifier
            .size(120.dp)
            .pointerInput(Unit) {
                awaitEachGesture {
                    awaitFirstDown()
                    currentOnPressStarted()

                    val up = waitForUpOrCancellation()

                    if (up == null) {
                        currentOnPressCanceled()
                    } else {
                        currentOnPressReleased()
                    }
                }
            }
    )
}
```

Всегда обрабатывайте cancellation. Если забыть cancel branch, pressed state может визуально "залипнуть".

## Несколько `pointerInput` modifiers

Можно повесить несколько `pointerInput` в цепочку. Порядок modifiers влияет на то, как events проходят через hierarchy и passes.

```kotlin
Modifier
    .pointerInput(Unit) {
        awaitPointerEventScope {
            while (true) {
                awaitPointerEvent(pass = PointerEventPass.Initial)
            }
        }
    }
    .pointerInput(Unit) {
        awaitPointerEventScope {
            while (true) {
                awaitPointerEvent(pass = PointerEventPass.Main)
            }
        }
    }
```

На практике лучше не плодить независимые raw detectors без причины. Если жесты связаны, один `pointerInput` с явной state machine обычно понятнее.

## Производственный паттерн: координатный scrubber

Scrubber — хороший пример, где raw pointer input оправдан: значение зависит от X-координаты внутри компонента.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.gestures.awaitEachGesture
import androidx.compose.foundation.gestures.awaitFirstDown
import androidx.compose.foundation.gestures.drag
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.foundation.layout.height
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.runtime.rememberUpdatedState
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.input.pointer.pointerInput
import androidx.compose.ui.semantics.ProgressBarRangeInfo
import androidx.compose.ui.semantics.progressBarRangeInfo
import androidx.compose.ui.semantics.semantics
import androidx.compose.ui.unit.dp

@Composable
fun Scrubber(
    progress: Float,
    onProgressChange: (Float) -> Unit,
    modifier: Modifier = Modifier
) {
    val currentOnProgressChange by rememberUpdatedState(onProgressChange)

    Canvas(
        modifier = modifier
            .fillMaxWidth()
            .height(40.dp)
            .semantics {
                progressBarRangeInfo = ProgressBarRangeInfo(
                    current = progress.coerceIn(0f, 1f),
                    range = 0f..1f,
                    steps = 0
                )
            }
            .pointerInput(Unit) {
                fun positionToProgress(x: Float): Float {
                    return (x / size.width.toFloat()).coerceIn(0f, 1f)
                }

                awaitEachGesture {
                    val down = awaitFirstDown()
                    currentOnProgressChange(positionToProgress(down.position.x))

                    drag(down.id) { change ->
                        currentOnProgressChange(positionToProgress(change.position.x))
                        change.consume()
                    }
                }
            }
    ) {
        val trackY = size.height / 2f
        val clampedProgress = progress.coerceIn(0f, 1f)
        val activeEndX = size.width * clampedProgress

        drawLine(
            color = Color(0xFFB0BEC5),
            start = Offset(x = 0f, y = trackY),
            end = Offset(x = size.width, y = trackY),
            strokeWidth = 4.dp.toPx()
        )
        drawLine(
            color = Color(0xFF1976D2),
            start = Offset(x = 0f, y = trackY),
            end = Offset(x = activeEndX, y = trackY),
            strokeWidth = 4.dp.toPx()
        )
        drawCircle(
            color = Color(0xFF1976D2),
            radius = 8.dp.toPx(),
            center = Offset(
                x = activeEndX,
                y = trackY
            )
        )
    }
}
```

Даже у пользовательский pointer input нужен semantics. Программа чтения с экрана не видит Canvas-линии без `progressBarRangeInfo`.

## Типичные ошибки

### Использовать `pointerInput` для обычного клика

`clickable` дает accessibility, keyboard, hover, focus и ripple. Raw pointer input этого не делает.

### Не учитывать cancellation

Если gesture отменен, UI-состояние нужно вернуть в консистентное состояние.

### Перезапускать detector из-за нестабильного ключа

Слишком часто меняющиеся keys отменяют active gesture.

### Забывать `consume`

Если пользовательский drag забрал движение, но не consume-ит его, parent scroll может получить тот же delta.

### Писать несколько бесконечных loops подряд

Внутри одного `pointerInput` первый бесконечный loop не даст выполниться коду после себя.

### Ожидать semantics автоматически

Raw pointer input не добавляет роль, label, actions или state descriptions.

## Итоги

Pointer input — это низкоуровневый инструмент:

- сначала выбирайте готовый компонент или high-level modifier;
- `pointerInput` запускает coroutine, связанную с modifier;
- keys управляют cancellation и restart;
- `awaitEachGesture` описывает один gesture cycle;
- `awaitPointerEventScope` дает доступ к `PointerEvent`;
- `PointerInputChange` содержит позиции, pressed state и consumption;
- passes помогают разруливать parent-child обработку;
- cancellation — нормальная часть gesture lifecycle;
- raw gestures требуют ручной accessibility semantics;
- сложный pointer input нужно тестировать на touch, mouse, stylus и large screen devices.
