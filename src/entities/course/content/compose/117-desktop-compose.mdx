## Чем Desktop Compose отличается от Android

Desktop Compose запускается на JVM и позволяет создавать приложения для Windows, macOS и Linux. Mental model Compose тот же, но desktop environment другой:

- приложение имеет одно или несколько окон;
- есть menu bar;
- есть system tray;
- есть mouse hover, right click, context menus;
- keyboard shortcuts обязательны;
- window resize является нормой;
- scrollbars должны быть видимыми;
- диалогах не всегда full-screen;
- packaging требует native distributions;
- app lifecycle не похож на Android Activity lifecycle.

```kotlin
data class DesktopAppState(
    val title: String,
    val currentDocumentPath: String?,
    val unsavedChanges: Boolean
)
```

Desktop UI часто строится вокруг documents, windows, menus и keyboard-first workflows.

<CatTip>
Desktop Compose — это не tablet UI в большом окне. Пользователь ожидает меню, shortcuts, resize, hover, context menus и работу с файлами.
</CatTip>

## `application`

Desktop точка входа:

```kotlin
import androidx.compose.ui.window.Window
import androidx.compose.ui.window.application

fun main() = application {
    Window(
        onCloseRequest = ::exitApplication,
        title = "Kotlingo Desktop"
    ) {
        AppTheme {
            DesktopAppRoot()
        }
    }
}
```

`application` управляет desktop app lifecycle. `Window` создает окно. Внутри окна обычная Compose composition.

## Window state

Window size/position/state можно хранить через `rememberWindowState`.

```kotlin
import androidx.compose.ui.unit.dp
import androidx.compose.ui.window.Window
import androidx.compose.ui.window.WindowPosition
import androidx.compose.ui.window.rememberWindowState

fun main() = application {
    val windowState = rememberWindowState(
        width = 1100.dp,
        height = 760.dp,
        position = WindowPosition.PlatformDefault
    )

    Window(
        state = windowState,
        onCloseRequest = ::exitApplication,
        title = "Kotlingo"
    ) {
        DesktopAppRoot()
    }
}
```

Не хардкодьте layout под initial size. Пользователь может изменить окно в любую сторону.

## Multiple windows

Desktop app может иметь несколько windows:

```kotlin
@Composable
fun DesktopApplication() {
    var openDocuments by remember {
        mutableStateOf(listOf("compose.mdx"))
    }

    openDocuments.forEach { document ->
        Window(
            onCloseRequest = {
                openDocuments = openDocuments - document
            },
            title = document
        ) {
            DocumentWindow(
                documentPath = document
            )
        }
    }
}
```

Window state должен быть привязан к document/window identity, а не к позиции в list.

## Menu bar

Desktop пользователи ожидают меню:

```kotlin
import androidx.compose.ui.window.MenuBar

@Composable
fun AppMenuBar(
    onOpen: () -> Unit,
    onSave: () -> Unit,
    onQuit: () -> Unit
) {
    MenuBar {
        Menu(
            text = "Файл",
            mnemonic = 'Ф'
        ) {
            Item(
                text = "Открыть",
                onClick = onOpen,
                shortcut = KeyShortcut(Key.O, ctrl = true)
            )
            Item(
                text = "Сохранить",
                onClick = onSave,
                shortcut = KeyShortcut(Key.S, ctrl = true)
            )
            Separator()
            Item(
                text = "Выйти",
                onClick = onQuit
            )
        }
    }
}
```

Menu actions и keyboard shortcuts должны вызывать один и тот же command layer.

## Command model

Для desktop удобно иметь command model:

```kotlin
sealed interface DesktopCommand {
    data object OpenFile : DesktopCommand
    data object Save : DesktopCommand
    data object SaveAs : DesktopCommand
    data object CloseWindow : DesktopCommand
    data object Quit : DesktopCommand
}

class DesktopCommandHandler(
    private val appState: DesktopStateHolder
) {
    fun handle(command: DesktopCommand) {
        when (command) {
            DesktopCommand.OpenFile -> appState.openFile()
            DesktopCommand.Save -> appState.save()
            DesktopCommand.SaveAs -> appState.saveAs()
            DesktopCommand.CloseWindow -> appState.closeCurrentWindow()
            DesktopCommand.Quit -> appState.quit()
        }
    }
}
```

Menu, toolbar, context menu и shortcuts вызывают `handle(command)`.

## Keyboard shortcuts

```kotlin
import androidx.compose.foundation.focusable
import androidx.compose.foundation.layout.Box
import androidx.compose.ui.Modifier
import androidx.compose.ui.input.key.Key
import androidx.compose.ui.input.key.KeyEventType
import androidx.compose.ui.input.key.isCtrlPressed
import androidx.compose.ui.input.key.key
import androidx.compose.ui.input.key.onPreviewKeyEvent
import androidx.compose.ui.input.key.type

@Composable
fun DesktopShortcutRoot(
    onCommand: (DesktopCommand) -> Unit,
    content: @Composable () -> Unit
) {
    Box(
        modifier = Modifier
            .focusable()
            .onPreviewKeyEvent { event ->
                if (event.type != KeyEventType.KeyDown) {
                    return@onPreviewKeyEvent false
                }

                when {
                    event.isCtrlPressed && event.key == Key.S -> {
                        onCommand(DesktopCommand.Save)
                        true
                    }
                    event.isCtrlPressed && event.key == Key.O -> {
                        onCommand(DesktopCommand.OpenFile)
                        true
                    }
                    else -> false
                }
            }
    ) {
        content()
    }
}
```

Shortcut root должен получать focus или быть размещен так, чтобы events доходили до него.

## Указатель и наведение

Desktop требует hover states и right click.

```kotlin
@Composable
fun HoverableToolbarButton(
    text: String,
    onClick: () -> Unit
) {
    val interactionSource = remember {
        MutableInteractionSource()
    }
    val hovered by interactionSource.collectIsHoveredAsState()

    Button(
        onClick = onClick,
        interactionSource = interactionSource,
        colors = ButtonDefaults.buttonColors(
            containerColor = if (hovered) {
                MaterialTheme.colorScheme.primaryContainer
            } else {
                MaterialTheme.colorScheme.surface
            }
        )
    ) {
        Text(text = text)
    }
}
```

Hover не заменяет focus. Keyboard user должен видеть focus state.

## Context menus

```kotlin
import androidx.compose.foundation.ContextMenuArea
import androidx.compose.foundation.ContextMenuItem

@Composable
fun DocumentTab(
    title: String,
    onClose: () -> Unit,
    onRevealInFiles: () -> Unit
) {
    ContextMenuArea(
        items = {
            listOf(
                ContextMenuItem(
                    label = "Закрыть",
                    onClick = onClose
                ),
                ContextMenuItem(
                    label = "Показать в файлах",
                    onClick = onRevealInFiles
                )
            )
        }
    ) {
        Text(text = title)
    }
}
```

Context menu actions должны иметь альтернативу в menu/toolbar, если действие важное.

## Scrollbars

На desktop scrollbars часто ожидаемы:

```kotlin
import androidx.compose.foundation.VerticalScrollbar
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.fillMaxSize
import androidx.compose.foundation.rememberScrollbarAdapter
import androidx.compose.foundation.lazy.rememberLazyListState

@Composable
fun DesktopLessonList(
    lessons: List<LessonUi>
) {
    val listState = rememberLazyListState()

    Box(modifier = Modifier.fillMaxSize()) {
        LazyColumn(
            state = listState
        ) {
            items(
                items = lessons,
                key = { lesson ->
                    lesson.id
                }
            ) { lesson ->
                LessonRow(lesson = lesson)
            }
        }

        VerticalScrollbar(
            adapter = rememberScrollbarAdapter(listState),
            modifier = Modifier.align(Alignment.CenterEnd)
        )
    }
}
```

Touch-first Android UI может скрывать scrollbars; desktop UI часто должен их показывать.

## Tooltips

```kotlin
import androidx.compose.foundation.ExperimentalFoundationApi
import androidx.compose.foundation.TooltipArea

@OptIn(ExperimentalFoundationApi::class)
@Composable
fun ToolbarSaveButton(
    onSave: () -> Unit
) {
    TooltipArea(
        tooltip = {
            Surface {
                Text(text = "Сохранить Ctrl+S")
            }
        }
    ) {
        IconButton(onClick = onSave) {
            Icon(
                imageVector = Icons.Default.Save,
                contentDescription = "Сохранить"
            )
        }
    }
}
```

Tooltip помогает pointer users, но icon button все равно требует `contentDescription`.

## Dialogs

Desktop диалогах не должны быть mobile full-screen.

```kotlin
import androidx.compose.ui.window.Dialog

@Composable
fun UnsavedChangesDialog(
    onSave: () -> Unit,
    onDiscard: () -> Unit,
    onCancel: () -> Unit
) {
    Dialog(
        onCloseRequest = onCancel,
        title = "Несохраненные изменения"
    ) {
        Column(
            modifier = Modifier.padding(24.dp)
        ) {
            Text(text = "Сохранить изменения перед закрытием?")
            Row {
                Button(onClick = onSave) {
                    Text(text = "Сохранить")
                }
                Button(onClick = onDiscard) {
                    Text(text = "Не сохранять")
                }
                TextButton(onClick = onCancel) {
                    Text(text = "Отмена")
                }
            }
        }
    }
}
```

Dialog должен иметь predictable keyboard behavior: Enter/Escape, focus order, default action.

## File диалогах

Desktop file диалогах обычно platform-specific. Держите их за adapter boundary.

```kotlin
interface FileDialogController {
    fun openMarkdownFile(): String?
    fun saveMarkdownFile(defaultName: String): String?
}

class DesktopFileDialogController : FileDialogController {

    override fun openMarkdownFile(): String? {
        return openFileDialog(
            title = "Открыть MDX",
            allowedExtensions = listOf("mdx", "md")
        )
    }

    override fun saveMarkdownFile(defaultName: String): String? {
        return saveFileDialog(
            title = "Сохранить",
            defaultName = defaultName
        )
    }
}
```

Общий UI вызывает event, desktop shell открывает dialog.

## System tray

Для apps, которые работают в фоне, desktop tray может быть важен.

```kotlin
import androidx.compose.ui.window.Tray

@Composable
fun AppTray(
    onShow: () -> Unit,
    onQuit: () -> Unit
) {
    Tray(
        icon = painterResource("tray_icon.png"),
        menu = {
            Item(
                text = "Показать",
                onClick = onShow
            )
            Item(
                text = "Выйти",
                onClick = onQuit
            )
        }
    )
}
```

Tray behavior должен соответствовать expectations OS: закрытие окна не всегда равно завершению приложения.

## Packaging

Compose Multiplatform Gradle plugin поддерживает native distributions.

```kotlin
compose.desktop {
    application {
        mainClass = "com.example.MainKt"

        nativeDistributions {
            targetFormats(
                TargetFormat.Dmg,
                TargetFormat.Msi,
                TargetFormat.Deb
            )
            packageName = "Kotlingo"
            packageVersion = "1.0.0"
        }
    }
}
```

Боевой packaging включает:

- icons;
- app metadata;
- signing;
- нотариальное заверение для macOS;
- update channel;
- installer testing;
- file associations, если нужны.

## Desktop QA проверочный список

Проверяйте:

- resize narrow/wide/short;
- multiple monitors;
- HiDPI scaling;
- keyboard shortcuts;
- menu actions;
- context menus;
- focus traversal;
- scrollbars;
- file диалогах;
- закрытие окна с несохраненным состоянием;
- упаковка на каждую ОС;
- dark/light theme.

## Типичные ошибки

### Растягивать mobile screen

Desktop app требует panes, menus, shortcuts и resize-friendly layout.

### Игнорировать keyboard

Desktop users ожидают shortcuts и focus navigation.

### Не тестировать packaging

App может работать из IDE и ломаться в installer.

### Хранить window state по index

При закрытии/перестановке окон state перепутается.

### Делать platform диалогах в common UI

Общий UI должен отправлять event, desktop shell открывает dialog.

## Итоги

Desktop Compose требует desktop-first shell:

- `application` и `Window` создают app/window lifecycle;
- window state должен поддерживать resize;
- menus, shortcuts, context menus и tray являются важными desktop surfaces;
- pointer hover и keyboard focus должны быть видимыми;
- scrollbars и tooltips повышают desktop usability;
- file диалогах и OS integrations держите за platform adapter;
- packaging native distributions требует отдельной настройки и QA;
- общий UI полезен, но desktop shell должен уважать desktop conventions.
