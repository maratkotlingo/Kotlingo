## Что такое общий element transition

Общий element transition — это анимация, которая визуально связывает один и тот же объект между двумя UI-состояниями или экранами. Например, пользователь нажимает карточку товара в списке, и изображение товара плавно превращается в большое изображение на экране деталей. Вместо ощущения "экран исчез, другой появился" пользователь видит непрерывность объекта.

В Compose общий element API строится вокруг нескольких частей:

- `SharedTransitionLayout`: верхний контейнер, который предоставляет `SharedTransitionScope`;
- `SharedTransitionScope`: scope, внутри которого работают общий modifiers;
- `rememberSharedContentState(key)`: состояние общий content с уникальным key;
- `Modifier.sharedElement`: связывает одинаковый visual element между состояниями;
- `Modifier.sharedBounds`: связывает bounds контейнеров, даже если content внутри отличается;
- `AnimatedVisibilityScope` или `AnimatedContentScope`: animation scope, который сообщает, между какими content происходит transition.

```kotlin
import androidx.compose.animation.AnimatedContent
import androidx.compose.animation.ExperimentalSharedTransitionApi
import androidx.compose.animation.SharedTransitionLayout
import androidx.compose.animation.SharedTransitionScope
import androidx.compose.animation.AnimatedContentScope
import androidx.compose.animation.sharedElement
import androidx.compose.animation.rememberSharedContentState
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@OptIn(ExperimentalSharedTransitionApi::class)
@Composable
fun BasicSharedElementExample(
    expanded: Boolean
) {
    SharedTransitionLayout {
        AnimatedContent(
            targetState = expanded,
            label = "shared element content"
        ) { targetExpanded ->
            if (targetExpanded) {
                LargeCard(
                    sharedTransitionScope = this@SharedTransitionLayout,
                    animatedContentScope = this@AnimatedContent
                )
            } else {
                SmallCard(
                    sharedTransitionScope = this@SharedTransitionLayout,
                    animatedContentScope = this@AnimatedContent
                )
            }
        }
    }
}

@OptIn(ExperimentalSharedTransitionApi::class)
@Composable
fun SmallCard(
    sharedTransitionScope: SharedTransitionScope,
    animatedContentScope: AnimatedContentScope
) {
    with(sharedTransitionScope) {
        Box(
            modifier = Modifier
                .sharedElement(
                    sharedContentState = rememberSharedContentState(
                        key = "card"
                    ),
                    animatedVisibilityScope = animatedContentScope
                )
                .size(96.dp)
        ) {
            Text(text = "Small")
        }
    }
}

@OptIn(ExperimentalSharedTransitionApi::class)
@Composable
fun LargeCard(
    sharedTransitionScope: SharedTransitionScope,
    animatedContentScope: AnimatedContentScope
) {
    with(sharedTransitionScope) {
        Box(
            modifier = Modifier
                .sharedElement(
                    sharedContentState = rememberSharedContentState(
                        key = "card"
                    ),
                    animatedVisibilityScope = animatedContentScope
                )
                .size(240.dp)
        ) {
            Text(text = "Large")
        }
    }
}
```

Ключ `"card"` одинаковый в small и large состояниях. Именно key говорит Compose, что это один общий element.

<CatTip>
Общий element transition имеет смысл только для объекта, который пользователь воспринимает как один и тот же: фото товара, аватар, карточка, обложка, заголовок. Не делайте общий transition для случайных декоративных блоков.
</CatTip>

## Experimental API

Общий element APIs в Compose помечены experimental. Поэтому примеры используют:

```kotlin
@OptIn(ExperimentalSharedTransitionApi::class)
```

Это значит:

- API может меняться;
- нужно читать release notes при обновлении Compose;
- желательно изолировать общий transitions в небольших composable wrappers;
- не стоит размазывать experimental modifiers по всему codebase без правил.

<MouseTrap>
Не прячьте `@OptIn(ExperimentalSharedTransitionApi::class)` на уровне всего приложения без обсуждения. Лучше локализовать experimental usage рядом с конкретными общий transition компонентами.
</MouseTrap>

## `SharedTransitionLayout`

`SharedTransitionLayout` должен охватывать оба composable, между которыми происходит transition. Он предоставляет `SharedTransitionScope`.

```kotlin
import androidx.compose.animation.ExperimentalSharedTransitionApi
import androidx.compose.animation.SharedTransitionLayout
import androidx.compose.runtime.Composable

@OptIn(ExperimentalSharedTransitionApi::class)
@Composable
fun SharedRoot(
    content: @Composable SharedTransitionScopeContent.() -> Unit
) {
    SharedTransitionLayout {
        SharedTransitionScopeContent(
            scope = this
        ).content()
    }
}

@OptIn(ExperimentalSharedTransitionApi::class)
class SharedTransitionScopeContent(
    val scope: androidx.compose.animation.SharedTransitionScope
)
```

На практике часто не создают такой wrapper, а размещают `SharedTransitionLayout` вокруг `AnimatedContent` или `NavHost`.

## `rememberSharedContentState`

`rememberSharedContentState(key)` хранит уникальный key общий element. Key должен быть:

- стабильным;
- уникальным среди общий elements одного transition scope;
- одинаковым для matching elements;
- завязанным на identity объекта, а не на позицию в списке;
- достаточно конкретным, если один item имеет несколько общий parts.

```kotlin
data class ArticleSharedElementKey(
    val articleId: String,
    val type: ArticleSharedElementType
)

enum class ArticleSharedElementType {
    Image,
    Title,
    Card
}
```

```kotlin
val imageKey = ArticleSharedElementKey(
    articleId = article.id,
    type = ArticleSharedElementType.Image
)
```

Не используйте index в списке как общий key, если item может переехать. При изменении сортировки index перестанет означать тот же объект.

## `sharedElement`

`Modifier.sharedElement` используется, когда content считается одним и тем же visual element в обоих состояниях.

Подходящие элементы:

- image;
- avatar;
- thumbnail;
- poster;
- product photo;
- card surface, если shape и content сохраняют идентичность;
- logo.

```kotlin
import androidx.compose.animation.ExperimentalSharedTransitionApi
import androidx.compose.animation.SharedTransitionScope
import androidx.compose.animation.AnimatedVisibilityScope
import androidx.compose.animation.rememberSharedContentState
import androidx.compose.animation.sharedElement
import androidx.compose.foundation.Image
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.layout.ContentScale
import androidx.compose.ui.res.painterResource
import androidx.compose.ui.unit.dp

@OptIn(ExperimentalSharedTransitionApi::class)
@Composable
fun SharedArticleImage(
    articleId: String,
    imageResId: Int,
    sharedTransitionScope: SharedTransitionScope,
    animatedVisibilityScope: AnimatedVisibilityScope,
    large: Boolean
) {
    with(sharedTransitionScope) {
        Image(
            painter = painterResource(id = imageResId),
            contentDescription = null,
            contentScale = ContentScale.Crop,
            modifier = Modifier
                .sharedElement(
                    sharedContentState = rememberSharedContentState(
                        key = ArticleSharedElementKey(
                            articleId = articleId,
                            type = ArticleSharedElementType.Image
                        )
                    ),
                    animatedVisibilityScope = animatedVisibilityScope
                )
                .size(
                    if (large) {
                        240.dp
                    } else {
                        96.dp
                    }
                )
        )
    }
}
```

`animatedVisibilityScope` сообщает общий transition системе, какой visibility/content transition управляет появлением и исчезновением элементов.

## `sharedBounds`

`sharedBounds` похож на `sharedElement`, но используется, когда нужно связать границы контейнера, а content внутри может отличаться.

Подходящие случаи:

- карточка списка превращается в header деталей;
- search bar раскрывается в search screen;
- compact toolbar превращается в expanded toolbar;
- container меняет content, но сохраняет воспринимаемую identity.

```kotlin
import androidx.compose.animation.ExperimentalSharedTransitionApi
import androidx.compose.animation.SharedTransitionScope
import androidx.compose.animation.AnimatedVisibilityScope
import androidx.compose.animation.rememberSharedContentState
import androidx.compose.animation.sharedBounds
import androidx.compose.foundation.layout.Column
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.foundation.layout.padding
import androidx.compose.material3.Card
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.unit.dp

@OptIn(ExperimentalSharedTransitionApi::class)
@Composable
fun SharedArticleBounds(
    articleId: String,
    title: String,
    sharedTransitionScope: SharedTransitionScope,
    animatedVisibilityScope: AnimatedVisibilityScope,
    expanded: Boolean
) {
    with(sharedTransitionScope) {
        Card(
            modifier = Modifier
                .sharedBounds(
                    sharedContentState = rememberSharedContentState(
                        key = ArticleSharedElementKey(
                            articleId = articleId,
                            type = ArticleSharedElementType.Card
                        )
                    ),
                    animatedVisibilityScope = animatedVisibilityScope
                )
                .fillMaxWidth()
        ) {
            Column(
                modifier = Modifier.padding(16.dp)
            ) {
                Text(text = title)
                if (expanded) {
                    Text(text = "Подробное описание статьи")
                }
            }
        }
    }
}
```

Если visual content одинаковый, часто лучше `sharedElement`. Если меняется контейнер и layout, чаще подходит `sharedBounds`.

## Modifier ordering

Порядок modifiers влияет на matching bounds. Size, padding, clip и общий modifier должны стоять осознанно.

```kotlin
Modifier
    .sharedElement(
        sharedContentState = rememberSharedContentState(key = imageKey),
        animatedVisibilityScope = animatedVisibilityScope
    )
    .size(96.dp)
```

и

```kotlin
Modifier
    .size(96.dp)
    .sharedElement(
        sharedContentState = rememberSharedContentState(key = imageKey),
        animatedVisibilityScope = animatedVisibilityScope
    )
```

могут давать разное поведение, потому что общий bounds будут измеряться на разных этапах modifier chain. Если transition прыгает, первым делом проверьте порядок size, padding, clip и общий modifier на обеих сторонах.

## Navigation integration

С Navigation Compose `SharedTransitionLayout` обычно размещают вокруг `NavHost`. Затем `SharedTransitionScope` передают screen routes, а `AnimatedContentScope` берут из `composable` content.

```kotlin
import androidx.compose.animation.ExperimentalSharedTransitionApi
import androidx.compose.animation.SharedTransitionLayout
import androidx.compose.runtime.Composable
import androidx.navigation.compose.NavHost
import androidx.navigation.compose.composable
import androidx.navigation.compose.rememberNavController
import androidx.navigation.toRoute
import kotlinx.serialization.Serializable

@Serializable
data object FeedRoute

@Serializable
data class DetailsRoute(
    val articleId: String
)

@OptIn(ExperimentalSharedTransitionApi::class)
@Composable
fun SharedNavigationHost(
    articles: List<ArticleUi>
) {
    val navController = rememberNavController()

    SharedTransitionLayout {
        NavHost(
            navController = navController,
            startDestination = FeedRoute
        ) {
            composable<FeedRoute> {
                FeedScreen(
                    articles = articles,
                    sharedTransitionScope = this@SharedTransitionLayout,
                    animatedVisibilityScope = this@composable,
                    onArticleClick = { articleId ->
                        navController.navigate(
                            DetailsRoute(articleId = articleId)
                        )
                    }
                )
            }

            composable<DetailsRoute> { backStackEntry ->
                val route = backStackEntry.toRoute<DetailsRoute>()

                DetailsScreen(
                    articleId = route.articleId,
                    sharedTransitionScope = this@SharedTransitionLayout,
                    animatedVisibilityScope = this@composable,
                    onNavigateBack = {
                        navController.popBackStack()
                    }
                )
            }
        }
    }
}

data class ArticleUi(
    val id: String,
    val title: String,
    val imageResId: Int
)
```

В type-safe Navigation route читается через `toRoute<DetailsRoute>()`, поэтому ключ общий element остается связанным с тем же `articleId`, который пришел из typed route.

## Feed и Details screens

```kotlin
import androidx.compose.animation.AnimatedVisibilityScope
import androidx.compose.animation.ExperimentalSharedTransitionApi
import androidx.compose.animation.SharedTransitionScope
import androidx.compose.foundation.clickable
import androidx.compose.foundation.lazy.LazyColumn
import androidx.compose.foundation.lazy.items
import androidx.compose.material3.ListItem
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier

@OptIn(ExperimentalSharedTransitionApi::class)
@Composable
fun FeedScreen(
    articles: List<ArticleUi>,
    sharedTransitionScope: SharedTransitionScope,
    animatedVisibilityScope: AnimatedVisibilityScope,
    onArticleClick: (String) -> Unit
) {
    LazyColumn {
        items(
            items = articles,
            key = { article ->
                article.id
            }
        ) { article ->
            ListItem(
                headlineContent = {
                    Text(text = article.title)
                },
                leadingContent = {
                    SharedArticleImage(
                        articleId = article.id,
                        imageResId = article.imageResId,
                        sharedTransitionScope = sharedTransitionScope,
                        animatedVisibilityScope = animatedVisibilityScope,
                        large = false
                    )
                },
                modifier = Modifier.clickable {
                    onArticleClick(article.id)
                }
            )
        }
    }
}
```

```kotlin
import androidx.compose.animation.AnimatedVisibilityScope
import androidx.compose.animation.ExperimentalSharedTransitionApi
import androidx.compose.animation.SharedTransitionScope
import androidx.compose.foundation.layout.Column
import androidx.compose.material3.Button
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable

@OptIn(ExperimentalSharedTransitionApi::class)
@Composable
fun DetailsScreen(
    articleId: String,
    sharedTransitionScope: SharedTransitionScope,
    animatedVisibilityScope: AnimatedVisibilityScope,
    onNavigateBack: () -> Unit
) {
    Column {
        SharedArticleImage(
            articleId = articleId,
            imageResId = android.R.drawable.ic_menu_gallery,
            sharedTransitionScope = sharedTransitionScope,
            animatedVisibilityScope = animatedVisibilityScope,
            large = true
        )

        Text(text = "Статья $articleId")

        Button(
            onClick = onNavigateBack
        ) {
            Text(text = "Назад")
        }
    }
}
```

В реальном коде image resource на details screen должен приходить из репозиторий/state. В примере важна одинаковая общий key identity.

<MouseTrap>
Общий transition не исправит плохую идентичность данных. Если list item и details screen используют разные IDs или разные keys, элемент не будет надежно matched.
</MouseTrap>

## Resize mode

Общий bounds может менять размер двумя основными стратегиями:

- remeasure target content под изменяющиеся bounds;
- scale стабильный layout к target bounds.

Смысл выбора:

- remeasure лучше, когда content должен реально перераскладываться;
- scale может быть дешевле и визуально стабильнее для изображений;
- текст при scale может выглядеть неидеально;
- сложный layout при remeasure может быть дороже.

API общий elements дает параметры resize mode в общий bounds configuration. Выбор нужно проверять глазами на реальном контенте, особенно с длинными текстами и разными aspect ratio изображений.

## Наложение во время transition

Общий elements часто рисуются в наложение во время transition. Это помогает элементу двигаться поверх остального UI, не ломая layout. Но иногда нужно, чтобы bottom bar, FAB или app bar оставались поверх общий element.

Для таких случаев используется наложение-related modifier:

```kotlin
import androidx.compose.animation.ExperimentalSharedTransitionApi
import androidx.compose.animation.SharedTransitionScope
import androidx.compose.animation.renderInSharedTransitionScopeOverlay
import androidx.compose.material3.FloatingActionButton
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier

@OptIn(ExperimentalSharedTransitionApi::class)
@Composable
fun OverlayFab(
    sharedTransitionScope: SharedTransitionScope,
    onClick: () -> Unit
) {
    with(sharedTransitionScope) {
        FloatingActionButton(
            onClick = onClick,
            modifier = Modifier.renderInSharedTransitionScopeOverlay()
        ) {
            Text(text = "+")
        }
    }
}
```

Наложение нужен не всегда. Используйте его, когда layering во время transition реально ломает визуальную иерархию.

## Debugging

Если общий element не работает:

- проверьте, что обе стороны находятся внутри одного `SharedTransitionLayout`;
- проверьте одинаковый key;
- проверьте, что key уникален;
- проверьте `AnimatedVisibilityScope` или `AnimatedContentScope`;
- проверьте modifier ordering;
- проверьте, что оба элемента одновременно участвуют в transition;
- проверьте lazy list keys;
- проверьте, что элемент details не получает другой ID;
- временно упростите modifier chain до `sharedElement().size()`;
- проверьте, не отключен ли transition config.

## Limitations

Общий element transitions не должны использоваться везде.

Ограничения:

- API experimental;
- сложнее тестировать;
- требует аккуратной идентичности;
- может конфликтовать с lazy recycling и async image загрузка;
- modifier ordering важен;
- text scaling может выглядеть хуже, чем обычный fade;
- сложные общий bounds могут быть дорогими;
- navigation scopes требуют внимательности.

## Боевой-паттерн: ключи

```kotlin
import androidx.compose.runtime.Immutable

@Immutable
data class ProductSharedKey(
    val productId: String,
    val element: ProductSharedElement
)

enum class ProductSharedElement {
    Photo,
    Title,
    Card
}
```

```kotlin
fun productPhotoSharedKey(
    productId: String
): ProductSharedKey {
    return ProductSharedKey(
        productId = productId,
        element = ProductSharedElement.Photo
    )
}
```

Typed key уменьшает риск случайно использовать одну строку для разных элементов.

## Типичные ошибки

### Использовать одинаковый key для разных элементов

Если image и title имеют key `"item-$id"`, они конфликтуют. Добавьте type элемента в key.

### Использовать index списка

Index меняется при сортировке, фильтрации и вставке. Используйте stable ID.

### Забывать scope

`sharedElement` должен быть внутри `SharedTransitionScope`, а также иметь `AnimatedVisibilityScope` или compatible scope.

### Ставить modifiers в разном порядке

Разный порядок `padding`, `clip`, `size` и `sharedElement` на двух сторонах может создать jumps.

### Анимировать слишком много общий elements

Один-два общий elements помогают восприятию. Десять элементов одновременно создают шум.

### Использовать общий transition для unrelated content

Если пользователь не воспринимает элементы как один объект, общий transition будет выглядеть странно.

## Итоги

Общий element transitions связывают один visual object между состояниями:

- `SharedTransitionLayout` предоставляет общий scope;
- `Modifier.sharedElement` связывает одинаковый visual element;
- `Modifier.sharedBounds` связывает bounds контейнеров;
- `rememberSharedContentState(key)` задает identity;
- key должен быть стабильным и уникальным;
- общий APIs требуют `AnimatedVisibilityScope` или `AnimatedContentScope`;
- с Navigation `SharedTransitionLayout` обычно оборачивает `NavHost`;
- modifier ordering влияет на результат;
- наложение помогает управлять layering;
- API experimental, поэтому usage лучше локализовать;
- общий transition должен поддерживать смысл навигации, а не быть декоративным трюком.
