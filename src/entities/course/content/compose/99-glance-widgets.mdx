## Что такое Glance

Glance позволяет описывать Android app widgets в Compose-похожем стиле. Но это не обычный Jetpack Compose UI. Glance генерирует `RemoteViews`, которые хостятся launcher-ом или другой системной поверхностью.

Из этого следуют важные ограничения:

- widget не живет внутри вашего Activity;
- нет полноценного Compose runtime UI tree как на экране приложения;
- composable набор ограничен;
- нет произвольного Canvas;
- нет обычных gestures;
- state должен быть сохранен отдельно;
- обновления нужно запрашивать явно;
- actions выполняются через PendingIntent/обратный вызов-подобную модель;
- launcher может изменить размер widget;
- widget должен быть быстрым, понятным и battery-friendly.

```kotlin
data class LessonWidgetState(
    val title: String,
    val progressText: String,
    val nextLessonId: String?,
    val lastUpdatedText: String
)
```

Widget должен показывать snapshot state. Он не должен пытаться быть мини-версией всего приложения.

<CatTip>
Glance выглядит как Compose, но живет по правилам RemoteViews. Если помнить это с первой строки, API перестает казаться странным.
</CatTip>

## Dependencies

```kotlin
dependencies {
    implementation("androidx.glance:glance-appwidget:1.2.0")
    implementation("androidx.glance:glance-material3:1.2.0")
    implementation("androidx.work:work-runtime-ktx:2.11.0")
}
```

Версии держите в version catalog. Glance развивается отдельно от обычных Compose Material APIs, поэтому не предполагайте, что каждый familiar composable доступен в widget.

## Минимальный widget

`GlanceAppWidget` описывает UI.

```kotlin
import androidx.compose.runtime.Composable
import androidx.glance.GlanceId
import androidx.glance.GlanceModifier
import androidx.glance.appwidget.GlanceAppWidget
import androidx.glance.appwidget.provideContent
import androidx.glance.layout.Column
import androidx.glance.layout.fillMaxSize
import androidx.glance.text.Text

class LessonProgressWidget : GlanceAppWidget() {

    override suspend fun provideGlance(
        context: Context,
        id: GlanceId
    ) {
        provideContent {
            LessonProgressContent(
                state = LessonWidgetState(
                    title = "Jetpack Compose",
                    progressText = "90 из 128 уроков",
                    nextLessonId = "compose-91",
                    lastUpdatedText = "Обновлено недавно"
                )
            )
        }
    }
}

@Composable
fun LessonProgressContent(
    state: LessonWidgetState
) {
    Column(
        modifier = GlanceModifier.fillMaxSize()
    ) {
        Text(text = state.title)
        Text(text = state.progressText)
        Text(text = state.lastUpdatedText)
    }
}
```

Код похож на Compose, но imports из `androidx.glance.*`, а не из обычного `androidx.compose.foundation.*`.

## Receiver

App widget подключается через `GlanceAppWidgetReceiver`.

```kotlin
import androidx.glance.appwidget.GlanceAppWidget
import androidx.glance.appwidget.GlanceAppWidgetReceiver

class LessonProgressWidgetReceiver : GlanceAppWidgetReceiver() {

    override val glanceAppWidget: GlanceAppWidget =
        LessonProgressWidget()
}
```

Manifest:

```xml
<!-- Получатель для виджета прогресса уроков -->
<receiver
    android:name=".LessonProgressWidgetReceiver"
    android:exported="true">
    <intent-filter>
        <action android:name="android.appwidget.action.APPWIDGET_UPDATE" />
    </intent-filter>

    <meta-data
        android:name="android.appwidget.provider"
        android:resource="@xml/lesson_progress_widget_info" />
</receiver>
```

App widget provider info:

```xml
<!-- Файл res/xml/lesson_progress_widget_info.xml -->
<appwidget-provider
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:minWidth="180dp"
    android:minHeight="80dp"
    android:targetCellWidth="3"
    android:targetCellHeight="2"
    android:resizeMode="horizontal|vertical"
    android:updatePeriodMillis="0"
    android:widgetCategory="home_screen" />
```

`updatePeriodMillis="0"` означает, что приложение само управляет обновлениями. Для большинства боевой widgets это лучше, чем частые системные polling updates.

## RemoteViews limitations

Glance ограничен тем, что можно выразить через RemoteViews:

- нет arbitrary composables;
- нет обычного `Modifier`;
- нет Canvas;
- нет pointer input;
- нет lazy layouts как в обычном Compose;
- нет бесконечных animations;
- нет прямого доступа к ViewModel;
- layout должен быть простым;
- сложные shapes/effects недоступны или ограничены;
- image загрузка требует аккуратной подготовки.

Плохой mental model:

```kotlin
// Неверная идея: виджет не является маленькой Activity с обычным интерфейсом Compose.
```

Правильный mental model:

```kotlin
// Виджет получает сохраненное состояние и строит снимок RemoteViews.
```

<MouseTrap>
Если вы пытаетесь перенести экран приложения в Glance один к одному, вы почти наверняка делаете слишком сложный widget.
</MouseTrap>

## State definition

Glance widget state хранится через state definition. Для простых cases можно использовать Preferences.

```kotlin
import androidx.datastore.preferences.core.Preferences
import androidx.glance.appwidget.GlanceAppWidget
import androidx.glance.state.GlanceStateDefinition
import androidx.glance.state.PreferencesGlanceStateDefinition

class LessonProgressWidget : GlanceAppWidget() {

    override val stateDefinition: GlanceStateDefinition<*> =
        PreferencesGlanceStateDefinition
}
```

Чтение state:

```kotlin
import androidx.datastore.preferences.core.stringPreferencesKey
import androidx.glance.currentState

val titleKey = stringPreferencesKey("title")
val progressKey = stringPreferencesKey("progress")
val updatedKey = stringPreferencesKey("updated")

@Composable
fun LessonProgressFromPreferences() {
    val prefs = currentState<Preferences>()

    LessonProgressContent(
        state = LessonWidgetState(
            title = prefs[titleKey] ?: "Kotlingo",
            progressText = prefs[progressKey] ?: "Нет данных",
            nextLessonId = null,
            lastUpdatedText = prefs[updatedKey] ?: "Еще не обновлялось"
        )
    )
}
```

Для сложных state используйте отдельную сериализацию и храните только компактный widget snapshot.

## Updating state

State обновляется из приложения, worker-а или action обратный вызов, затем widget просит redraw.

```kotlin
import android.content.Context
import androidx.datastore.preferences.core.edit
import androidx.datastore.preferences.core.stringPreferencesKey
import androidx.glance.appwidget.GlanceAppWidgetManager

suspend fun updateLessonWidgetState(
    context: Context,
    snapshot: LessonWidgetState
) {
    val manager = GlanceAppWidgetManager(context)
    val ids = manager.getGlanceIds(
        LessonProgressWidget::class.java
    )

    ids.forEach { id ->
        updateAppWidgetState(
            context = context,
            definition = PreferencesGlanceStateDefinition,
            glanceId = id
        ) { prefs ->
            prefs[titleKey] = snapshot.title
            prefs[progressKey] = snapshot.progressText
            prefs[updatedKey] = snapshot.lastUpdatedText
        }
    }

    LessonProgressWidget().updateAll(context)
}
```

Widget update должен быть редким и осмысленным. Не обновляйте каждую секунду, если user value меняется раз в час.

## `updateAll`

`updateAll(context)` обновляет все instances конкретного widget.

```kotlin
import android.content.Context
import androidx.glance.appwidget.updateAll

suspend fun refreshAllLessonWidgets(
    context: Context
) {
    LessonProgressWidget().updateAll(context)
}
```

Если у пользователя несколько instances widget с разными настройками, храните state per `GlanceId` и не думайте, что "виджет один".

## Actions

Glance actions могут открывать Activity или запускать обратный вызов.

Открыть screen:

```kotlin
import androidx.glance.action.actionStartActivity
import androidx.glance.appwidget.action.actionRunCallback
import androidx.glance.layout.Column
import androidx.glance.text.Text
import androidx.glance.unit.ColorProvider

@Composable
fun LessonWidgetActions(
    state: LessonWidgetState
) {
    Column {
        Text(text = state.title)

        GlanceButton(
            text = "Открыть урок",
            onClick = actionStartActivity<MainActivity>()
        )
    }
}
```

Запустить обратный вызов:

```kotlin
import android.content.Context
import androidx.glance.GlanceId
import androidx.glance.action.ActionParameters
import androidx.glance.appwidget.action.ActionCallback

class MarkLessonDoneAction : ActionCallback {

    override suspend fun onAction(
        context: Context,
        glanceId: GlanceId,
        parameters: ActionParameters
    ) {
        lessonRepository(context).markCurrentLessonDone()
        LessonProgressWidget().update(context, glanceId)
    }
}
```

UI:

```kotlin
GlanceButton(
    text = "Готово",
    onClick = actionRunCallback<MarkLessonDoneAction>()
)
```

Обратный вызов должен быть коротким. Для тяжелой синхронизации запускайте WorkManager.

## Responsive sizes

Widget может менять размер. Делайте layout, который деградирует.

```kotlin
import androidx.glance.appwidget.LocalSize
import androidx.glance.layout.Column
import androidx.glance.layout.Row
import androidx.glance.text.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.unit.dp

@Composable
fun ResponsiveLessonWidget(
    state: LessonWidgetState
) {
    val size = LocalSize.current

    if (size.width < 220.dp) {
        Column {
            Text(text = state.title)
            Text(text = state.progressText)
        }
    } else {
        Row {
            Column {
                Text(text = state.title)
                Text(text = state.progressText)
            }
            Text(text = state.lastUpdatedText)
        }
    }
}
```

Не пытайтесь показать всю информацию в `2x1`. Маленький widget должен показывать только самое важное.

## WorkManager updates

Для периодического background refresh используйте WorkManager с разумной частотой и constraints.

```kotlin
import android.content.Context
import androidx.work.CoroutineWorker
import androidx.work.WorkerParameters

class LessonWidgetRefreshWorker(
    appContext: Context,
    workerParams: WorkerParameters
) : CoroutineWorker(appContext, workerParams) {

    override suspend fun doWork(): Result {
        val snapshot = lessonWidgetRepository(applicationContext)
            .loadSnapshot()

        updateLessonWidgetState(
            context = applicationContext,
            snapshot = snapshot
        )

        return Result.success()
    }
}
```

Планирование:

```kotlin
import androidx.work.ExistingPeriodicWorkPolicy
import androidx.work.PeriodicWorkRequestBuilder
import androidx.work.WorkManager
import java.util.concurrent.TimeUnit

fun scheduleLessonWidgetUpdates(
    context: Context
) {
    val request = PeriodicWorkRequestBuilder<LessonWidgetRefreshWorker>(
        repeatInterval = 6,
        repeatIntervalTimeUnit = TimeUnit.HOURS
    ).build()

    WorkManager
        .getInstance(context)
        .enqueueUniquePeriodicWork(
            "lesson-widget-refresh",
            ExistingPeriodicWorkPolicy.UPDATE,
            request
        )
}
```

Не используйте виджет как таймер с частыми updates. Launcher, battery и system scheduler этому не рады.

## Images

Для widget изображений лучше использовать resource, content URI или заранее подготовленный bitmap. Remote image загрузка прямо из widget UI — плохая идея.

```kotlin
import androidx.glance.Image
import androidx.glance.ImageProvider

@Composable
fun LessonIcon() {
    Image(
        provider = ImageProvider(R.drawable.ic_lesson),
        contentDescription = "Урок"
    )
}
```

Если изображение приходит из сети, скачайте и закэшируйте его заранее, затем обновите widget state.

## Theming

Glance Material 3 дает ограниченную theme model. Учитывайте:

- launcher background может отличаться;
- user может включить dark theme;
- dynamic color доступен не всегда одинаково;
- text contrast должен быть достаточным;
- widget backgrounds должны быть simple.

```kotlin
import androidx.glance.GlanceTheme
import androidx.glance.appwidget.material3.ColorProviders
import androidx.glance.material3.ColorProviders

class LessonProgressWidget : GlanceAppWidget() {

    @Composable
    override fun Content() {
        GlanceTheme(
            colors = ColorProviders(
                light = lightColorScheme(),
                dark = darkColorScheme()
            )
        ) {
            LessonProgressFromPreferences()
        }
    }
}
```

API names могут отличаться между версиями Glance Material. Архитектурно важно: theme задается внутри widget, а не наследуется от Activity.

## Testing widgets

Что проверять:

- small/medium/large sizes;
- empty state;
- stale data state;
- action opens correct screen;
- обратный вызов updates state;
- WorkManager обновляет snapshot;
- multiple widget instances;
- dark/light theme;
- launcher предпросмотр;
- в интерфейсе нет работы сети;
- update frequency не слишком высокая.

Unit-test mapper:

```kotlin
import com.google.common.truth.Truth.assertThat
import org.junit.Test

class LessonWidgetSnapshotTest {

    @Test
    fun progressText_includesCompletedAndTotal() {
        val snapshot = LessonWidgetSnapshot(
            completed = 90,
            total = 128
        ).toWidgetState()

        assertThat(snapshot.progressText)
            .isEqualTo("90 из 128 уроков")
    }
}
```

Glance rendering тестируется сложнее, поэтому business mapping и update scheduling лучше держать отдельно от UI.

## Типичные ошибки

### Думать, что Glance — обычный Compose

RemoteViews ограничения быстро ломают ожидания.

### Обновлять widget слишком часто

Battery и scheduler важнее желания показать live animation.

### Делать widget слишком интерактивным

Widget должен открывать app или выполнять короткие actions.

### Хранить state только в памяти

Widget process может не жить, когда launcher запрашивает UI.

### Не учитывать несколько instances

Пользователь может добавить один widget для Kotlin, другой для Compose.

### Загружать сеть в UI

Подготовьте snapshot заранее через репозиторий/worker.

## Итоги

Glance widgets — Compose-like API поверх RemoteViews:

- `GlanceAppWidget` описывает widget UI;
- `GlanceAppWidgetReceiver` подключает widget к системе;
- UI ограничен RemoteViews model;
- state хранится через `stateDefinition`, часто Preferences;
- `updateAll` и `update(context, glanceId)` явно обновляют widgets;
- actions открывают Activity или запускают короткий обратный вызов;
- WorkManager подходит для фонового refresh;
- responsive sizes обязательны;
- несколько widget instances требуют per-id state;
- widget должен быть glanceable, battery-friendly и не похожим на мини-Activity.
