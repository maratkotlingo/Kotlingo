## Что такое `Painter`

`Painter` — это объект, который умеет рисовать внутри `DrawScope` и может сообщать preferred intrinsic size. Он похож на Android `Drawable` по роли, но встроен в Compose drawing model.

`Painter` используется в:

- `Image(painter = painter)`;
- `Icon(painter = painter)`;
- `Modifier.paint(painter = painter)`;
- `painterResource(id = imageResId)`;
- `rememberVectorPainter(image = imageVector)`;
- `BitmapPainter`;
- `ColorPainter`;
- `BrushPainter`;
- custom painter-классах.

```kotlin
import androidx.compose.foundation.Image
import androidx.compose.runtime.Composable
import androidx.compose.ui.res.painterResource

@Composable
fun ResourcePainterImage() {
    Image(
        painter = painterResource(id = R.drawable.cover),
        contentDescription = "Обложка курса"
    )
}
```

`painterResource` возвращает подходящий painter для ресурса: bitmap asset станет bitmap painter, vector drawable станет vector painter.

<CatTip>
`Painter` — это reusable drawing object. Если вы хотите переиспользовать одно и то же рисование в `Image`, `Icon` и `Modifier.paint`, painter часто выразительнее, чем отдельный composable.
</CatTip>

## `Painter` против `Canvas` и draw modifiers

`Canvas` — это composable с собственным layout node. Он получает размер через modifier и рисует внутри него.

Drawing modifiers рисуют вокруг уже существующего composable: behind content, вместо content или поверх content.

`Painter` — это переносимая draw-логика. Он может быть передан в `Image`, применен через `paint`, сохранен через `remember` и переиспользован в разных местах.

Разница важна:

- `Canvas` подходит для отдельного custom-drawn UI element;
- `drawBehind` и `drawWithContent` подходят для decoration существующего composable;
- `Painter` подходит для reusable graphics asset или программно рисуемого изображения;
- custom `Painter` может влиять на измерение через `intrinsicSize`;
- draw modifier строго рисует в уже выделенных bounds.

<MouseTrap>
Не превращайте каждый custom drawing block в `Painter`. Если рисование тесно связано с конкретным composable и не переиспользуется, `drawBehind` или `Canvas` обычно проще.
</MouseTrap>

## Anatomy of `Painter`

Custom painter обычно переопределяет:

- `onDraw()`: основная draw-логика внутри `DrawScope`;
- `intrinsicSize`: желаемый размер, если painter может его сообщить.

Painter также может переопределять:

- `applyAlpha(alpha)`: принять alpha без внешнего layer;
- `applyColorFilter(colorFilter)`: принять color filter;
- `applyLayoutDirection(layoutDirection)`: отреагировать на RTL/LTR;
- `equals` и `hashCode`: если painter содержит параметры и вы хотите предсказуемое сравнение.

```kotlin
import androidx.compose.ui.geometry.Size
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.drawscope.DrawScope
import androidx.compose.ui.graphics.painter.Painter

class StatusDotPainter(
    private val color: Color,
    private val diameterPx: Float
) : Painter() {

    override val intrinsicSize: Size
        get() = Size(
            width = diameterPx,
            height = diameterPx
        )

    override fun DrawScope.onDraw() {
        drawCircle(
            color = color,
            radius = size.minDimension / 2f
        )
    }
}
```

`onDraw` получает фактический `size`, в который painter рисуется сейчас. Он может отличаться от `intrinsicSize`, если caller задал другой размер или `ContentScale`.

## `intrinsicSize`

`intrinsicSize` — это размер, который painter предпочел бы иметь, если parent не задал жесткие constraints.

Возможные варианты:

- конкретный `Size(width, height)`;
- `Size.Unspecified`, если painter не имеет естественного размера;
- размер bitmap;
- размер vector viewport после conversion;
- размер, вычисленный из параметров painter-а.

```kotlin
import androidx.compose.foundation.Image
import androidx.compose.runtime.Composable
import androidx.compose.runtime.remember
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.painter.Painter
import androidx.compose.ui.platform.LocalDensity
import androidx.compose.ui.unit.dp

@Composable
fun IntrinsicDotImage() {
    val density = LocalDensity.current

    val painter: Painter = remember(density) {
        with(density) {
            StatusDotPainter(
                color = Color(0xFF2E7D32),
                diameterPx = 18.dp.toPx()
            )
        }
    }

    Image(
        painter = painter,
        contentDescription = "Состояние: активно"
    )
}
```

Если `Image` не получает explicit size, он может использовать intrinsic size painter-а. Для UI, где размер должен быть стабильным, чаще задают `Modifier.size`.

## Custom painter для initials avatar

Painter может рисовать shape, фон и текстоподобные элементы. Но полноценный текст в `DrawScope` лучше делать через text APIs, а для обычного текста использовать composables.

Ниже painter рисует только графический фон для initials avatar, а сами initials остаются `Text`, чтобы сохранить typographic behavior и accessibility.

```kotlin
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.foundation.layout.wrapContentSize
import androidx.compose.material3.MaterialTheme
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.runtime.remember
import androidx.compose.ui.Alignment
import androidx.compose.ui.Modifier
import androidx.compose.ui.draw.paint
import androidx.compose.ui.geometry.Size
import androidx.compose.ui.graphics.Brush
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.drawscope.DrawScope
import androidx.compose.ui.graphics.painter.Painter
import androidx.compose.ui.unit.dp

class AvatarBackgroundPainter(
    private val startColor: Color,
    private val endColor: Color
) : Painter() {

    override val intrinsicSize: Size
        get() = Size.Unspecified

    override fun DrawScope.onDraw() {
        val brush = Brush.radialGradient(
            colors = listOf(
                startColor,
                endColor
            ),
            radius = size.minDimension
        )

        drawCircle(
            brush = brush,
            radius = size.minDimension / 2f
        )
    }
}

@Composable
fun InitialsAvatar(
    initials: String
) {
    val painter = remember {
        AvatarBackgroundPainter(
            startColor = Color(0xFF42A5F5),
            endColor = Color(0xFF1E88E5)
        )
    }

    Box(
        contentAlignment = Alignment.Center,
        modifier = Modifier
            .size(56.dp)
            .paint(painter)
    ) {
        Text(
            text = initials,
            color = Color.White,
            style = MaterialTheme.typography.titleMedium,
            modifier = Modifier.wrapContentSize()
        )
    }
}
```

Такой разделенный подход лучше, чем рисовать буквы вручную: Compose text сохраняет font scale, baseline, accessibility и text rendering quality.

## `Modifier.paint`

`Modifier.paint` рисует painter в bounds любого composable.

Важные параметры:

- `painter`: объект рисования;
- `sizeToIntrinsics`: использовать intrinsic size painter-а для размера modifier-а, если это возможно;
- `alignment`: где разместить painter внутри bounds;
- `contentScale`: как масштабировать painter;
- `alpha`: прозрачность painter-а;
- `colorFilter`: преобразование цвета painter-а.

```kotlin
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.remember
import androidx.compose.ui.Modifier
import androidx.compose.ui.draw.paint
import androidx.compose.ui.geometry.Size
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.drawscope.DrawScope
import androidx.compose.ui.graphics.painter.Painter
import androidx.compose.ui.layout.ContentScale
import androidx.compose.ui.unit.dp

class DiagonalStripesPainter(
    private val backgroundColor: Color,
    private val stripeColor: Color
) : Painter() {

    override val intrinsicSize: Size
        get() = Size.Unspecified

    override fun DrawScope.onDraw() {
        drawRect(color = backgroundColor)

        val stripeWidth = 10.dp.toPx()
        val gap = 18.dp.toPx()
        var x = -size.height

        while (x < size.width) {
            drawLine(
                color = stripeColor,
                start = androidx.compose.ui.geometry.Offset(
                    x = x,
                    y = size.height
                ),
                end = androidx.compose.ui.geometry.Offset(
                    x = x + size.height,
                    y = 0f
                ),
                strokeWidth = stripeWidth
            )
            x += gap
        }
    }
}

@Composable
fun StripedWarningBackground() {
    val painter = remember {
        DiagonalStripesPainter(
            backgroundColor = Color(0xFFFFF8E1),
            stripeColor = Color(0xFFFFD54F)
        )
    }

    Box(
        modifier = Modifier
            .size(
                width = 220.dp,
                height = 88.dp
            )
            .paint(
                painter = painter,
                contentScale = ContentScale.Crop
            )
    )
}
```

Здесь painter рисует decorative background. Content, если он появится внутри `Box`, будет нарисован поверх painter-а.

## `BitmapPainter`

`BitmapPainter` рисует `ImageBitmap`. Он полезен, когда bitmap уже загружен или сгенерирован.

```kotlin
import androidx.compose.foundation.Image
import androidx.compose.runtime.Composable
import androidx.compose.runtime.remember
import androidx.compose.ui.graphics.ImageBitmap
import androidx.compose.ui.graphics.painter.BitmapPainter
import androidx.compose.ui.res.imageResource

@Composable
fun BitmapPainterExample() {
    val image = ImageBitmap.imageResource(id = R.drawable.cover)
    val painter = remember(image) {
        BitmapPainter(image = image)
    }

    Image(
        painter = painter,
        contentDescription = "Обложка"
    )
}
```

Для ресурсов в обычном UI чаще достаточно `painterResource`. `BitmapPainter` нужен, когда вы управляете `ImageBitmap` сами.

## `ColorPainter` и `BrushPainter`

`ColorPainter` и `BrushPainter` превращают цвет или brush в painter.

```kotlin
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.remember
import androidx.compose.ui.Modifier
import androidx.compose.ui.draw.paint
import androidx.compose.ui.graphics.Brush
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.painter.BrushPainter
import androidx.compose.ui.unit.dp

@Composable
fun BrushPainterBackground() {
    val painter = remember {
        BrushPainter(
            Brush.linearGradient(
                colors = listOf(
                    Color(0xFF00838F),
                    Color(0xFF7CB342)
                )
            )
        )
    }

    Box(
        modifier = Modifier
            .size(
                width = 200.dp,
                height = 100.dp
            )
            .paint(painter)
    )
}
```

Если вам нужен только фон, `Modifier.background(brush)` обычно проще. `BrushPainter` полезен, когда API ожидает painter.

## Alpha и `ColorFilter`

Painter может быть нарисован с alpha и colorFilter через `Image` или `Modifier.paint`.

```kotlin
import androidx.compose.foundation.Image
import androidx.compose.runtime.Composable
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.ColorFilter
import androidx.compose.ui.res.painterResource

@Composable
fun DisabledPainterIcon(
    enabled: Boolean
) {
    Image(
        painter = painterResource(id = R.drawable.ic_download),
        contentDescription = "Скачать",
        alpha = if (enabled) {
            1f
        } else {
            0.38f
        },
        colorFilter = if (enabled) {
            null
        } else {
            ColorFilter.tint(Color.Gray)
        }
    )
}
```

Для disabled state одного alpha мало. Интерактивный компонент должен также отключить действие через `enabled = false` или не принимать клик.

## Layout direction в painter

Если graphic зависит от направления интерфейса, painter может учитывать `LayoutDirection`.

```kotlin
import androidx.compose.ui.geometry.Size
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.drawscope.DrawScope
import androidx.compose.ui.graphics.painter.Painter
import androidx.compose.ui.unit.LayoutDirection

class DirectionArrowPainter(
    private val color: Color
) : Painter() {

    private var currentLayoutDirection: LayoutDirection = LayoutDirection.Ltr

    override val intrinsicSize: Size
        get() = Size.Unspecified

    override fun applyLayoutDirection(
        layoutDirection: LayoutDirection
    ): Boolean {
        currentLayoutDirection = layoutDirection
        return true
    }

    override fun DrawScope.onDraw() {
        val points = if (currentLayoutDirection == LayoutDirection.Ltr) {
            listOf(
                androidx.compose.ui.geometry.Offset(0f, 0f),
                androidx.compose.ui.geometry.Offset(size.width, size.height / 2f),
                androidx.compose.ui.geometry.Offset(0f, size.height)
            )
        } else {
            listOf(
                androidx.compose.ui.geometry.Offset(size.width, 0f),
                androidx.compose.ui.geometry.Offset(0f, size.height / 2f),
                androidx.compose.ui.geometry.Offset(size.width, size.height)
            )
        }

        val path = androidx.compose.ui.graphics.Path().apply {
            moveTo(points[0].x, points[0].y)
            lineTo(points[1].x, points[1].y)
            lineTo(points[2].x, points[2].y)
            close()
        }

        drawPath(
            path = path,
            color = color
        )
    }
}
```

Если иконка имеет directional meaning, лучше не забывать RTL. Для недирекционных icons зеркалирование не требуется.

## Переиспользование painter-а

Painter может хранить параметры. Если параметры неизменны, создавайте painter через `remember`.

```kotlin
import androidx.compose.runtime.Composable
import androidx.compose.runtime.remember
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.painter.Painter

@Composable
fun rememberStatusDotPainter(
    color: Color,
    diameterPx: Float
): Painter {
    return remember(
        color,
        diameterPx
    ) {
        StatusDotPainter(
            color = color,
            diameterPx = diameterPx
        )
    }
}
```

Не создавайте новый painter на каждую recomposition, если он не зависит от изменившихся параметров.

## Painter и stability

Painter как параметр composable требует осторожности. Многие painter-ы содержат bitmap, resource handle или mutable drawing internals. Из-за этого compiler не всегда может считать такой параметр стабильным.

Практическое правило:

- в public UI API чаще передавайте `@DrawableRes Int`, URL или domain image model;
- создавайте painter ближе к месту отображения;
- memoize painter через `remember`;
- не передавайте `Painter` через несколько слоев архитектуры без причины;
- для design system components painter-параметр допустим, если это осознанная часть API.

```kotlin
import androidx.annotation.DrawableRes
import androidx.compose.foundation.Image
import androidx.compose.runtime.Composable
import androidx.compose.ui.res.painterResource

@Composable
fun CourseCover(
    @DrawableRes coverResId: Int,
    title: String
) {
    Image(
        painter = painterResource(id = coverResId),
        contentDescription = "Обложка курса: $title"
    )
}
```

Такой API проще для callers и обычно стабильнее, чем передача готового painter-а.

## Типичные ошибки

### Делать painter composable

`Painter` — обычный объект. Он не должен напрямую вызывать composable functions внутри `onDraw`.

### Декодировать bitmap внутри `onDraw`

`onDraw` может вызываться часто. Загрузка и декодирование должны происходить заранее.

### Игнорировать `intrinsicSize`

Если painter имеет естественный размер, сообщите его. Если размера нет, используйте `Size.Unspecified` и задавайте size снаружи.

### Рисовать текст вручную без причины

Text composables уже решают font scale, baseline, accessibility и style. Custom painter для букв нужен редко.

### Забывать content description

Painter рисует картинку, но accessibility задает caller через `Image`, `Icon`, semantics или surrounding component.

## Итоги

`Painter` — переносимый объект рисования:

- `onDraw` выполняется в `DrawScope`;
- `intrinsicSize` может влиять на measurement;
- `Image` и `Modifier.paint` являются основными точками применения;
- `BitmapPainter`, `ColorPainter` и `BrushPainter` покрывают частые случаи;
- custom painter хорош для reusable graphics;
- `Canvas` и draw modifiers часто проще для локального drawing;
- painter нужно помнить через `remember`, если создание не бесплатное;
- в public API часто лучше передавать resource id или URL, а не painter;
- accessibility не возникает из painter-а автоматически.
