## Что такое `Canvas`

`Canvas` — composable для собственное рисование. Он дает `DrawScope`, внутри которого можно рисовать линии, прямоугольники, круги, paths, arcs, изображения и текст. Внутри `Canvas` вы работаете не с layout tree, а с drawing commands.

`Canvas` полезен, когда стандартных composables и modifiers недостаточно:

- диаграммы;
- пользовательский прогресс indicators;
- charts;
- gauges;
- waveforms;
- декоративные backgrounds;
- selection frames;
- пользовательские элементы управления;
- drawing-heavy UI.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun SimpleCanvasCircle() {
    Canvas(
        modifier = Modifier.size(96.dp)
    ) {
        drawCircle(
            color = Color.Blue,
            radius = size.minDimension / 2f
        )
    }
}
```

`Canvas` сам по себе не задает размер. Размер приходит из `modifier`. Если не задать size constraints, canvas может получить нулевой или неожиданный размер в зависимости от parent layout.

<CatTip>
Думайте о `Canvas` как о собственное рисование surface внутри Compose layout. Layout решает, сколько места дать, а `DrawScope` решает, что нарисовать внутри этого места.
</CatTip>

## `Canvas` и `drawBehind`

`Canvas` во многом является удобной формой drawing modifier. Если нужно нарисовать пользовательский контент как отдельный элемент layout, используйте `Canvas`. Если нужно дорисовать фон или наложение к существующему composable, часто лучше `drawBehind` или `drawWithContent`.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.foundation.layout.height
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun DividerCanvas() {
    Canvas(
        modifier = Modifier
            .fillMaxWidth()
            .height(1.dp)
    ) {
        drawRect(color = Color.LightGray)
    }
}
```

Для простой линии обычно достаточно `HorizontalDivider`. Canvas нужен, когда standard component не покрывает задачу.

## Координаты

Координатная система `DrawScope`:

- начало координат `(0, 0)` находится в левом верхнем углу drawing area;
- X растет вправо;
- Y растет вниз;
- все значения в пикселях;
- `size.width` и `size.height` дают доступную область;
- `center` возвращает центр области.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun CoordinateDemo() {
    Canvas(
        modifier = Modifier.size(160.dp)
    ) {
        drawCircle(
            color = Color.Red,
            radius = 6.dp.toPx(),
            center = Offset(
                x = 0f,
                y = 0f
            )
        )

        drawCircle(
            color = Color.Blue,
            radius = 6.dp.toPx(),
            center = center
        )

        drawCircle(
            color = Color.Green,
            radius = 6.dp.toPx(),
            center = Offset(
                x = size.width,
                y = size.height
            )
        )
    }
}
```

Красная точка находится в левом верхнем углу, синяя — в центре, зеленая — в правом нижнем углу.

## Density и `dp.toPx()`

В `DrawScope` можно использовать `dp.toPx()`, потому что scope знает density.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun DensityAwareLine() {
    Canvas(
        modifier = Modifier.size(120.dp)
    ) {
        val strokeWidth = 2.dp.toPx()

        drawLine(
            color = Color.Black,
            start = Offset.Zero,
            end = Offset(
                x = size.width,
                y = size.height
            ),
            strokeWidth = strokeWidth
        )
    }
}
```

Не храните pixel values как constants на уровне файла, если они зависят от density. Переводите `dp` в px внутри density-aware context.

## `drawLine`

`drawLine` рисует линию между двумя точками.

Параметры:

- `color` или `brush`;
- `start`;
- `end`;
- `strokeWidth`;
- `cap`;
- `pathEffect`;
- `alpha`;
- `blendMode`.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.foundation.layout.height
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.StrokeCap
import androidx.compose.ui.unit.dp

@Composable
fun RoundedProgressLine(
    progress: Float
) {
    val clampedProgress = progress.coerceIn(0f, 1f)

    Canvas(
        modifier = Modifier
            .fillMaxWidth()
            .height(24.dp)
    ) {
        val stroke = 8.dp.toPx()
        val y = size.height / 2f

        drawLine(
            color = Color.LightGray,
            start = Offset(
                x = 0f,
                y = y
            ),
            end = Offset(
                x = size.width,
                y = y
            ),
            strokeWidth = stroke,
            cap = StrokeCap.Round
        )

        drawLine(
            color = Color.Blue,
            start = Offset(
                x = 0f,
                y = y
            ),
            end = Offset(
                x = size.width * clampedProgress,
                y = y
            ),
            strokeWidth = stroke,
            cap = StrokeCap.Round
        )
    }
}
```

`StrokeCap.Round` добавляет округление концов. Учтите, что round cap визуально выходит за start/end на половину stroke width.

## `drawRect`

`drawRect` рисует прямоугольник.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.geometry.Size
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun RectQuadrants() {
    Canvas(
        modifier = Modifier.size(160.dp)
    ) {
        val quadrant = Size(
            width = size.width / 2f,
            height = size.height / 2f
        )

        drawRect(
            color = Color.Red,
            topLeft = Offset.Zero,
            size = quadrant
        )
        drawRect(
            color = Color.Blue,
            topLeft = Offset(
                x = size.width / 2f,
                y = 0f
            ),
            size = quadrant
        )
        drawRect(
            color = Color.Green,
            topLeft = Offset(
                x = 0f,
                y = size.height / 2f
            ),
            size = quadrant
        )
        drawRect(
            color = Color.Yellow,
            topLeft = Offset(
                x = size.width / 2f,
                y = size.height / 2f
            ),
            size = quadrant
        )
    }
}
```

Если не передать `topLeft`, default будет `Offset.Zero`. Если не передать `size`, default обычно занимает всю drawing area.

## `drawCircle`

`drawCircle` рисует круг.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.drawscope.Stroke
import androidx.compose.ui.unit.dp

@Composable
fun Ring() {
    Canvas(
        modifier = Modifier.size(96.dp)
    ) {
        drawCircle(
            color = Color.Blue,
            radius = size.minDimension / 2f,
            style = Stroke(
                width = 6.dp.toPx()
            )
        )
    }
}
```

Если `style` не задан, круг будет залитым. Для outline используйте `Stroke`.

## `drawRoundRect`

`drawRoundRect` рисует прямоугольник со скругленными углами.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.CornerRadius
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun RoundedCardBackground() {
    Canvas(
        modifier = Modifier.size(
            width = 200.dp,
            height = 96.dp
        )
    ) {
        drawRoundRect(
            color = Color(0xFFE3F2FD),
            cornerRadius = CornerRadius(
                x = 16.dp.toPx(),
                y = 16.dp.toPx()
            )
        )
    }
}
```

Для обычных карточек лучше использовать `Card` или `Surface`. `Canvas` нужен, если форма является частью собственное рисование.

## `drawArc`

`drawArc` рисует дугу или сектор.

Параметры:

- `startAngle`: стартовый угол в градусах;
- `sweepAngle`: длина дуги в градусах;
- `useCenter`: рисовать сектор к центру или только дугу;
- `topLeft` и `size`: bounding rectangle;
- `style`: fill или stroke.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.StrokeCap
import androidx.compose.ui.graphics.drawscope.Stroke
import androidx.compose.ui.unit.dp

@Composable
fun CircularProgressCanvas(
    progress: Float
) {
    val clampedProgress = progress.coerceIn(0f, 1f)

    Canvas(
        modifier = Modifier.size(96.dp)
    ) {
        val strokeWidth = 8.dp.toPx()

        drawArc(
            color = Color.LightGray,
            startAngle = -90f,
            sweepAngle = 360f,
            useCenter = false,
            style = Stroke(
                width = strokeWidth,
                cap = StrokeCap.Round
            )
        )

        drawArc(
            color = Color.Blue,
            startAngle = -90f,
            sweepAngle = 360f * clampedProgress,
            useCenter = false,
            style = Stroke(
                width = strokeWidth,
                cap = StrokeCap.Round
            )
        )
    }
}
```

В Compose angles идут по часовой стрелке. `-90f` обычно используется, чтобы прогресс начинался сверху.

## `drawPath`

`Path` описывает сложную фигуру через команды `moveTo`, `lineTo`, `quadraticBezierTo`, `cubicTo`, `close`.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.Path
import androidx.compose.ui.unit.dp

@Composable
fun TriangleCanvas() {
    Canvas(
        modifier = Modifier.size(140.dp)
    ) {
        val triangle = Path().apply {
            moveTo(
                x = size.width / 2f,
                y = 0f
            )
            lineTo(
                x = size.width,
                y = size.height
            )
            lineTo(
                x = 0f,
                y = size.height
            )
            close()
        }

        drawPath(
            path = triangle,
            color = Color(0xFFFF7043)
        )
    }
}
```

Если path зависит только от size, в боевой среде лучше создавать его в `drawWithCache`, чтобы не выделять новый object на каждый draw.

## `drawImage`

`drawImage` рисует `ImageBitmap` внутри `DrawScope`. Это полезно для пользовательский image processing, sprites, charts с bitmap markers.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.ImageBitmap
import androidx.compose.ui.unit.dp

@Composable
fun BitmapMarkerCanvas(
    marker: ImageBitmap
) {
    Canvas(
        modifier = Modifier.size(160.dp)
    ) {
        drawImage(
            image = marker,
            topLeft = androidx.compose.ui.geometry.Offset(
                x = center.x - marker.width / 2f,
                y = center.y - marker.height / 2f
            )
        )
    }
}
```

Загрузка и decode bitmap не должны происходить внутри `Canvas` draw block. Передавайте уже готовый `ImageBitmap` или используйте image загрузка слой выше.

## Transformations

`DrawScope` поддерживает transformations:

- `translate`;
- `rotate`;
- `scale`;
- `inset`;
- `withTransform`.

Transformations влияют только на drawing commands внутри лямбда. Они не меняют layout size и position.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun RotatedRectCanvas() {
    Canvas(
        modifier = Modifier.size(160.dp)
    ) {
        rotate(
            degrees = 45f,
            pivot = center
        ) {
            drawRect(
                color = Color.Gray,
                topLeft = androidx.compose.ui.geometry.Offset(
                    x = size.width / 4f,
                    y = size.height / 4f
                ),
                size = androidx.compose.ui.geometry.Size(
                    width = size.width / 2f,
                    height = size.height / 2f
                )
            )
        }
    }
}
```

Повернутый прямоугольник может выйти за drawing bounds визуально, но layout останется прежним.

## `withTransform`

Если нужно применить несколько transformations, `withTransform` эффективнее и читаемее, чем вложенные `translate`, `rotate`, `scale`.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.drawscope.withTransform
import androidx.compose.ui.unit.dp

@Composable
fun TransformedCircle() {
    Canvas(
        modifier = Modifier.size(160.dp)
    ) {
        withTransform(
            transformBlock = {
                translate(
                    left = size.width / 2f,
                    top = size.height / 2f
                )
                rotate(degrees = 30f)
                scale(
                    scaleX = 1.4f,
                    scaleY = 0.8f
                )
            }
        ) {
            drawCircle(
                color = Color.Magenta,
                radius = 24.dp.toPx()
            )
        }
    }
}
```

Все transformations применяются как единый transform block.

## `inset`

`inset` уменьшает drawing area и сдвигает origin.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun InsetRect() {
    Canvas(
        modifier = Modifier.size(120.dp)
    ) {
        drawRect(color = Color.LightGray)

        inset(
            horizontal = 16.dp.toPx(),
            vertical = 16.dp.toPx()
        ) {
            drawRect(color = Color.Blue)
        }
    }
}
```

`inset` удобен для drawing padding без изменения layout.

## `drawIntoCanvas`

`drawIntoCanvas` дает доступ к низкоуровневому `Canvas`. Он нужен редко, например для взаимодействия с Android `Drawable` или API, которому нужен native canvas.

```kotlin
import android.graphics.drawable.Drawable
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.drawscope.drawIntoCanvas
import androidx.compose.ui.graphics.nativeCanvas
import androidx.compose.ui.unit.dp

@Composable
fun DrawableCanvas(
    drawable: Drawable
) {
    Canvas(
        modifier = Modifier.size(120.dp)
    ) {
        drawIntoCanvas { canvas ->
            drawable.setBounds(
                0,
                0,
                size.width.toInt(),
                size.height.toInt()
            )
            drawable.draw(canvas.nativeCanvas)
        }
    }
}
```

Используйте `drawIntoCanvas` только когда high-level `DrawScope` API недостаточно.

## Боевой-пример: sparkline chart

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.foundation.layout.height
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.Path
import androidx.compose.ui.graphics.drawscope.Stroke
import androidx.compose.ui.unit.dp

@Composable
fun SparklineChart(
    values: List<Float>,
    modifier: Modifier = Modifier
) {
    Canvas(
        modifier = modifier
            .fillMaxWidth()
            .height(80.dp)
    ) {
        if (values.size < 2) {
            return@Canvas
        }

        val minValue = values.minOrNull() ?: 0f
        val maxValue = values.maxOrNull() ?: 1f
        val range = (maxValue - minValue).takeIf { value ->
            value != 0f
        } ?: 1f

        val path = Path()

        values.forEachIndexed { index, value ->
            val x = size.width * index / (values.lastIndex)
            val normalized = (value - minValue) / range
            val y = size.height - normalized * size.height

            if (index == 0) {
                path.moveTo(x, y)
            } else {
                path.lineTo(x, y)
            }
        }

        drawPath(
            path = path,
            color = Color(0xFF1E88E5),
            style = Stroke(
                width = 2.dp.toPx()
            )
        )
    }
}
```

Для часто обновляемого chart path лучше кешировать или выносить подготовку данных, если список большой.

## Боевой-пример: circular gauge

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.StrokeCap
import androidx.compose.ui.graphics.drawscope.Stroke
import androidx.compose.ui.unit.dp

@Composable
fun Gauge(
    value: Float,
    modifier: Modifier = Modifier
) {
    val clampedValue = value.coerceIn(0f, 1f)

    Canvas(
        modifier = modifier.size(160.dp)
    ) {
        val strokeWidth = 14.dp.toPx()
        val startAngle = 135f
        val totalSweep = 270f

        drawArc(
            color = Color(0xFFE0E0E0),
            startAngle = startAngle,
            sweepAngle = totalSweep,
            useCenter = false,
            style = Stroke(
                width = strokeWidth,
                cap = StrokeCap.Round
            )
        )

        drawArc(
            color = Color(0xFF43A047),
            startAngle = startAngle,
            sweepAngle = totalSweep * clampedValue,
            useCenter = false,
            style = Stroke(
                width = strokeWidth,
                cap = StrokeCap.Round
            )
        )
    }
}
```

Gauge не имеет semantics сам по себе. Если он передает важное значение, добавьте content description или state description на wrapper.

## Accessibility

Canvas drawing не создает semantics автоматически. Если рисунок несет смысл, добавьте semantics:

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.progressBarRangeInfo
import androidx.compose.ui.semantics.ProgressBarRangeInfo
import androidx.compose.ui.semantics.semantics
import androidx.compose.ui.unit.dp

@Composable
fun AccessibleGauge(
    value: Float
) {
    Gauge(
        value = value,
        modifier = Modifier.semantics {
            progressBarRangeInfo = ProgressBarRangeInfo(
                current = value.coerceIn(0f, 1f),
                range = 0f..1f,
                steps = 0
            )
        }
    )
}
```

Программа чтения с экрана не видит ваши `drawArc` и `drawPath`. Semantics нужно задавать отдельно.

## Performance

Canvas performance зависит от drawing work:

- не создавайте сложные `Path` и `Brush` на каждый frame без необходимости;
- используйте `drawWithCache` для size-dependent objects;
- не декодируйте изображения в draw block;
- не сортируйте большие списки внутри `Canvas`;
- анимируйте простые numeric values;
- проверяйте overdraw;
- помните, что собственное рисование может быть дешевле большого layout tree, но дорогой drawing тоже может jank-ать.

## Типичные ошибки

### Не задать размер `Canvas`

Без constraints canvas может получить неожиданный размер. Используйте `size`, `fillMaxWidth`, `height`, `aspectRatio`.

### Путать dp и px

DrawScope работает в пикселях. Используйте `dp.toPx()`.

### Создавать тяжелые объекты в draw block

Path, shader и complex brush лучше кешировать.

### Ожидать automatic accessibility

Canvas не создает semantics для нарисованных объектов.

### Использовать Canvas вместо обычных composables

Если UI можно собрать из `Text`, `Icon`, `Card`, `ProgressIndicator`, лучше начать с них. Canvas нужен для собственное рисование.

### Игнорировать layout bounds

Transformations могут нарисовать объект за пределами области, но layout size не изменится.

## Итоги

`Canvas` дает прямой доступ к собственное рисование:

- размер задается modifier-ами;
- `DrawScope` работает в пикселях;
- `size`, `center`, `density` помогают рассчитывать geometry;
- `drawLine`, `drawRect`, `drawCircle`, `drawRoundRect`, `drawArc`, `drawPath`, `drawImage` покрывают базовое drawing;
- `translate`, `rotate`, `scale`, `inset`, `withTransform` меняют drawing coordinate system;
- `drawIntoCanvas` нужен для низкоуровневого interop;
- Canvas не создает accessibility semantics автоматически;
- тяжелую geometry лучше кешировать;
- собственное рисование должно быть оправдано задачей, а не заменять обычные composables без причины.
