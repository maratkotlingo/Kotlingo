## Почему текстовый ввод требует отдельной темы

`TextField` кажется обычным компонентом: пользователь видит поле, вводит текст, получает ошибку. Но для accessibility, IME, password manager, autofill service и тестов текстовое поле является не просто прямоугольником с буквами. У него есть имя, тип данных, состояние ошибки, cursor/selection, keyboard configuration, hint, supporting text, правила автозаполнения и privacy expectations.

Хорошее поле ввода отвечает на несколько вопросов:

- что нужно ввести;
- обязательно ли это поле;
- какой тип данных ожидается;
- есть ли ошибка;
- что конкретно нужно исправить;
- можно ли заполнить значение через autofill;
- безопасно ли показывать введенный текст;
- какая клавиатура должна открыться;
- какое действие IME должно выполнить;
- как поле звучит для программа чтения с экрана;
- что проверять в UI tests.

В Compose часть этой работы уже встроена в `TextField` и `OutlinedTextField`, но боевой-форма все равно требует явного дизайна.

```kotlin
import androidx.compose.material3.OutlinedTextField
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.text.input.KeyboardType
import androidx.compose.foundation.text.KeyboardOptions

@Composable
fun EmailField(
    value: String,
    onValueChange: (String) -> Unit,
    isError: Boolean,
    errorMessage: String?,
    modifier: Modifier = Modifier
) {
    OutlinedTextField(
        value = value,
        onValueChange = onValueChange,
        label = {
            Text(text = "Email")
        },
        supportingText = {
            if (errorMessage != null) {
                Text(text = errorMessage)
            }
        },
        isError = isError,
        keyboardOptions = KeyboardOptions(
            keyboardType = KeyboardType.Email
        ),
        singleLine = true,
        modifier = modifier
    )
}
```

`label` здесь важнее, чем заглушка. Label остается частью смыслового описания поля, а заглушка исчезает, когда пользователь начинает вводить текст.

<CatTip>
Для текстового поля label — это имя элемента, а заглушка — временная подсказка. Если оставить только заглушка, пользователь может потерять контекст ровно в тот момент, когда начал вводить данные.
</CatTip>

## Label, заглушка и supporting text

У поля ввода обычно есть три разных текстовых слоя:

- `label`: постоянное имя поля;
- `placeholder`: пример или мягкая подсказка внутри пустого поля;
- `supportingText`: дополнительное объяснение, ограничение или ошибка.

Не смешивайте их в один текст. Каждый слой нужен для своего сценария.

```kotlin
import androidx.compose.material3.OutlinedTextField
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier

@Composable
fun DisplayNameField(
    value: String,
    onValueChange: (String) -> Unit,
    modifier: Modifier = Modifier
) {
    OutlinedTextField(
        value = value,
        onValueChange = onValueChange,
        label = {
            Text(text = "Имя в профиле")
        },
        placeholder = {
            Text(text = "Например, Мария")
        },
        supportingText = {
            Text(text = "Будет видно другим участникам")
        },
        singleLine = true,
        modifier = modifier
    )
}
```

Плохой вариант:

```kotlin
OutlinedTextField(
    value = value,
    onValueChange = onValueChange,
    placeholder = {
        Text(text = "Введите имя в профиле")
    }
)
```

В этом варианте поле теряет имя после начала ввода. Визуально экран еще может казаться понятным, но программа чтения с экрана и пользователь с когнитивной нагрузкой получают менее устойчивый интерфейс.

## Ошибки ввода

Ошибка у поля должна быть видна, слышна и связана с конкретным действием исправления. Обычно нужны три вещи:

- `isError = true`;
- текст ошибки в `supportingText`;
- semantics `error`, если вы строите пользовательский field или хотите уточнить announcement.

```kotlin
import androidx.compose.material3.OutlinedTextField
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.error
import androidx.compose.ui.semantics.semantics

@Composable
fun PasswordConfirmationField(
    value: String,
    onValueChange: (String) -> Unit,
    mismatch: Boolean,
    modifier: Modifier = Modifier
) {
    val message = if (mismatch) {
        "Пароли не совпадают"
    } else {
        null
    }

    OutlinedTextField(
        value = value,
        onValueChange = onValueChange,
        label = {
            Text(text = "Повторите пароль")
        },
        supportingText = {
            if (message != null) {
                Text(text = message)
            }
        },
        isError = mismatch,
        singleLine = true,
        modifier = modifier.semantics {
            if (message != null) {
                error(message)
            }
        }
    )
}
```

`isError` отвечает за визуальное состояние Material field. `supportingText` объясняет, что не так. `semantics { error(...) }` делает ошибку частью accessibility contract.

<MouseTrap>
Красная рамка без текста ошибки — это не validation UX. Пользователь должен понять не только "что-то не так", но и "что сделать дальше".
</MouseTrap>

## Error announcements

Если ошибка появилась после submit, полезно объявить ее отдельным сообщением. Для этого можно использовать `liveRegion`, но только для действительно важных изменений.

```kotlin
import androidx.compose.foundation.layout.Column
import androidx.compose.material3.Button
import androidx.compose.material3.OutlinedTextField
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.runtime.mutableStateOf
import androidx.compose.runtime.remember
import androidx.compose.runtime.setValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.LiveRegionMode
import androidx.compose.ui.semantics.liveRegion
import androidx.compose.ui.semantics.semantics

@Composable
fun LoginFormWithAnnouncement(
    onSubmit: (String) -> Unit
) {
    var email by remember {
        mutableStateOf("")
    }
    var errorMessage by remember {
        mutableStateOf<String?>(null)
    }

    Column {
        OutlinedTextField(
            value = email,
            onValueChange = {
                email = it
                errorMessage = null
            },
            label = {
                Text(text = "Email")
            },
            isError = errorMessage != null,
            supportingText = {
                errorMessage?.let { message ->
                    Text(text = message)
                }
            },
            singleLine = true
        )

        errorMessage?.let { message ->
            Text(
                text = message,
                modifier = Modifier.semantics {
                    liveRegion = LiveRegionMode.Assertive
                }
            )
        }

        Button(
            onClick = {
                if ("@" !in email) {
                    errorMessage = "Введите email с символом @"
                } else {
                    onSubmit(email)
                }
            }
        ) {
            Text(text = "Войти")
        }
    }
}
```

`Assertive` перебивает текущую речь программа чтения с экрана. Используйте его для ошибок submit, но не для каждого символа во время набора. Для мягких обновлений подходит `Polite`.

## Autofill через `ContentType`

Autofill service не должен угадывать, что поле является email, username или password, только по visual label. В Compose поле включается в autofill через `ContentType` в semantics.

```kotlin
import androidx.compose.material3.OutlinedTextField
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.autofill.ContentType
import androidx.compose.ui.semantics.contentType
import androidx.compose.ui.semantics.semantics

@Composable
fun UsernameAutofillField(
    value: String,
    onValueChange: (String) -> Unit,
    modifier: Modifier = Modifier
) {
    OutlinedTextField(
        value = value,
        onValueChange = onValueChange,
        label = {
            Text(text = "Логин")
        },
        singleLine = true,
        modifier = modifier.semantics {
            contentType = ContentType.Username
        }
    )
}
```

Для поля, которое принимает несколько типов данных, `ContentType` можно объединять:

```kotlin
import androidx.compose.ui.autofill.ContentType
import androidx.compose.ui.semantics.contentType
import androidx.compose.ui.semantics.semantics

val loginModifier = Modifier.semantics {
    contentType = ContentType.Username + ContentType.EmailAddress
}
```

Это нормальный сценарий для login field, где пользователь может ввести username или email.

## Сохранение данных autofill

Когда пользователь покидает экран, Compose может автоматически завершить autofill context. Но если у формы есть явная кнопка сохранения новых credentials, можно вызвать `LocalAutofillManager.current?.commit()`.

```kotlin
import androidx.compose.foundation.layout.Column
import androidx.compose.foundation.layout.Spacer
import androidx.compose.foundation.layout.height
import androidx.compose.material3.Button
import androidx.compose.material3.OutlinedTextField
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.runtime.mutableStateOf
import androidx.compose.runtime.remember
import androidx.compose.runtime.setValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.autofill.ContentType
import androidx.compose.ui.platform.LocalAutofillManager
import androidx.compose.ui.semantics.contentType
import androidx.compose.ui.semantics.semantics
import androidx.compose.ui.text.input.PasswordVisualTransformation
import androidx.compose.ui.unit.dp

@Composable
fun CreateCredentialsForm(
    onCreate: (String, String) -> Unit
) {
    val autofillManager = LocalAutofillManager.current
    var username by remember {
        mutableStateOf("")
    }
    var password by remember {
        mutableStateOf("")
    }

    Column {
        OutlinedTextField(
            value = username,
            onValueChange = {
                username = it
            },
            label = {
                Text(text = "Новый логин")
            },
            singleLine = true,
            modifier = Modifier.semantics {
                contentType = ContentType.NewUsername
            }
        )

        Spacer(modifier = Modifier.height(12.dp))

        OutlinedTextField(
            value = password,
            onValueChange = {
                password = it
            },
            label = {
                Text(text = "Новый пароль")
            },
            visualTransformation = PasswordVisualTransformation(),
            singleLine = true,
            modifier = Modifier.semantics {
                contentType = ContentType.NewPassword
            }
        )

        Spacer(modifier = Modifier.height(16.dp))

        Button(
            onClick = {
                onCreate(username, password)
                autofillManager?.commit()
            }
        ) {
            Text(text = "Создать аккаунт")
        }
    }
}
```

Если пользователь отменил ввод, можно вызвать `autofillManager?.cancel()`, чтобы не предлагать сохранить черновые или ошибочные данные.

## Password fields

Пароль — это не просто `TextField` с точками вместо символов. Для password field обычно нужны:

- `visualTransformation = PasswordVisualTransformation()`;
- `keyboardOptions = KeyboardOptions(keyboardType = KeyboardType.Password)`;
- `ContentType.Password` или `ContentType.NewPassword`;
- отсутствие логирования значения;
- осторожное сохранение в state;
- кнопка показа пароля с понятной semantics;
- запрет autofill для полей, где это действительно необходимо по security policy.

```kotlin
import androidx.compose.foundation.layout.Row
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.material3.IconButton
import androidx.compose.material3.OutlinedTextField
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.runtime.mutableStateOf
import androidx.compose.runtime.remember
import androidx.compose.runtime.setValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.autofill.ContentType
import androidx.compose.ui.semantics.contentDescription
import androidx.compose.ui.semantics.contentType
import androidx.compose.ui.semantics.semantics
import androidx.compose.ui.text.input.KeyboardType
import androidx.compose.ui.text.input.PasswordVisualTransformation
import androidx.compose.ui.text.input.VisualTransformation
import androidx.compose.foundation.text.KeyboardOptions

@Composable
fun AccessiblePasswordField(
    value: String,
    onValueChange: (String) -> Unit,
    modifier: Modifier = Modifier
) {
    var passwordVisible by remember {
        mutableStateOf(false)
    }

    OutlinedTextField(
        value = value,
        onValueChange = onValueChange,
        label = {
            Text(text = "Пароль")
        },
        visualTransformation = if (passwordVisible) {
            VisualTransformation.None
        } else {
            PasswordVisualTransformation()
        },
        keyboardOptions = KeyboardOptions(
            keyboardType = KeyboardType.Password
        ),
        singleLine = true,
        trailingIcon = {
            IconButton(
                onClick = {
                    passwordVisible = !passwordVisible
                },
                modifier = Modifier.semantics {
                    contentDescription = if (passwordVisible) {
                        "Скрыть пароль"
                    } else {
                        "Показать пароль"
                    }
                }
            ) {
                Text(
                    text = if (passwordVisible) {
                        "Скрыть"
                    } else {
                        "Показать"
                    }
                )
            }
        },
        modifier = modifier
            .fillMaxWidth()
            .semantics {
                contentType = ContentType.Password
            }
    )
}
```

В настоящем приложении вместо текста в `IconButton` обычно будет иконка. Важно не забыть понятный `contentDescription`, потому что "иконка глаза" не является универсальным текстовым контрактом.

## `VisualTransformation` не меняет данные

`VisualTransformation` влияет на отображение, но не меняет значение. Это удобно для паролей, телефонов, карт и масок, но опасно, если перепутать UI-format и domain-format.

```kotlin
import androidx.compose.ui.text.AnnotatedString
import androidx.compose.ui.text.input.OffsetMapping
import androidx.compose.ui.text.input.TransformedText
import androidx.compose.ui.text.input.VisualTransformation

class FourDigitGroupingTransformation : VisualTransformation {

    override fun filter(text: AnnotatedString): TransformedText {
        val raw = text.text.filter {
            it.isDigit()
        }
        val formatted = raw.chunked(4).joinToString(" ")

        return TransformedText(
            text = AnnotatedString(formatted),
            offsetMapping = object : OffsetMapping {
                override fun originalToTransformed(offset: Int): Int {
                    val spacesBefore = offset / 4
                    return (offset + spacesBefore).coerceAtMost(formatted.length)
                }

                override fun transformedToOriginal(offset: Int): Int {
                    val spacesBefore = offset / 5
                    return (offset - spacesBefore).coerceAtLeast(0)
                }
            }
        )
    }
}
```

Offset mapping должен быть корректным. Если он ломается, cursor, selection, TalkBack editing и тесты начинают вести себя странно.

<MouseTrap>
Маска ввода не должна быть единственным валидатором. Domain layer должен получать нормализованное значение и проверять его независимо от того, как оно красиво показано в UI.
</MouseTrap>

## Keyboard options и IME actions

Accessibility текстового ввода тесно связана с keyboard UX. Если поле ожидает email, телефон или число, скажите об этом клавиатуре.

```kotlin
import androidx.compose.foundation.text.KeyboardActions
import androidx.compose.foundation.text.KeyboardOptions
import androidx.compose.material3.OutlinedTextField
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.text.input.ImeAction
import androidx.compose.ui.text.input.KeyboardType

@Composable
fun SearchQueryField(
    query: String,
    onQueryChange: (String) -> Unit,
    onSearch: () -> Unit
) {
    OutlinedTextField(
        value = query,
        onValueChange = onQueryChange,
        label = {
            Text(text = "Поиск")
        },
        singleLine = true,
        keyboardOptions = KeyboardOptions(
            keyboardType = KeyboardType.Text,
            imeAction = ImeAction.Search
        ),
        keyboardActions = KeyboardActions(
            onSearch = {
                onSearch()
            }
        )
    )
}
```

`ImeAction.Search` сообщает пользователю, что enter/action key выполнит поиск. Если UI показывает кнопку "Найти", а IME action отправляет форму регистрации, интерфейс становится недоверяемым.

## State-based text fields

Современные Material text fields поддерживают state-based model через `TextFieldState`. Она лучше хранит text, selection и composition, особенно для сложного ввода. Но архитектурное правило остается тем же: UI-состояние должен быть отделен от domain state.

```kotlin
import androidx.compose.foundation.text.input.TextFieldState
import androidx.compose.foundation.text.input.rememberTextFieldState
import androidx.compose.material3.OutlinedTextField
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.autofill.ContentType
import androidx.compose.ui.semantics.contentType
import androidx.compose.ui.semantics.semantics

@Composable
fun StateBasedEmailField(
    state: TextFieldState = rememberTextFieldState(),
    modifier: Modifier = Modifier
) {
    OutlinedTextField(
        state = state,
        label = {
            Text(text = "Email")
        },
        modifier = modifier.semantics {
            contentType = ContentType.EmailAddress
        }
    )
}
```

State-based field не отменяет validation. Он просто дает более надежную модель редактирования текста.

## Форма как state machine

Для боевой-формы удобно хранить не набор разрозненных boolean flags, а явную модель.

```kotlin
data class SignInFormState(
    val email: String = "",
    val password: String = "",
    val emailError: String? = null,
    val passwordError: String? = null,
    val submitting: Boolean = false
) {
    val canSubmit: Boolean
        get() = email.isNotBlank() && password.isNotBlank() && !submitting
}
```

UI получает state и события:

```kotlin
import androidx.compose.foundation.layout.Column
import androidx.compose.foundation.layout.Spacer
import androidx.compose.foundation.layout.height
import androidx.compose.material3.Button
import androidx.compose.material3.OutlinedTextField
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.autofill.ContentType
import androidx.compose.ui.semantics.contentType
import androidx.compose.ui.semantics.error
import androidx.compose.ui.semantics.semantics
import androidx.compose.ui.text.input.PasswordVisualTransformation
import androidx.compose.ui.unit.dp

@Composable
fun SignInForm(
    state: SignInFormState,
    onEmailChange: (String) -> Unit,
    onPasswordChange: (String) -> Unit,
    onSubmit: () -> Unit
) {
    Column {
        OutlinedTextField(
            value = state.email,
            onValueChange = onEmailChange,
            label = {
                Text(text = "Email")
            },
            isError = state.emailError != null,
            supportingText = {
                state.emailError?.let {
                    Text(text = it)
                }
            },
            singleLine = true,
            modifier = Modifier.semantics {
                contentType = ContentType.EmailAddress
                state.emailError?.let {
                    error(it)
                }
            }
        )

        Spacer(modifier = Modifier.height(12.dp))

        OutlinedTextField(
            value = state.password,
            onValueChange = onPasswordChange,
            label = {
                Text(text = "Пароль")
            },
            isError = state.passwordError != null,
            supportingText = {
                state.passwordError?.let {
                    Text(text = it)
                }
            },
            visualTransformation = PasswordVisualTransformation(),
            singleLine = true,
            modifier = Modifier.semantics {
                contentType = ContentType.Password
                state.passwordError?.let {
                    error(it)
                }
            }
        )

        Spacer(modifier = Modifier.height(16.dp))

        Button(
            enabled = state.canSubmit,
            onClick = onSubmit
        ) {
            Text(
                text = if (state.submitting) {
                    "Входим..."
                } else {
                    "Войти"
                }
            )
        }
    }
}
```

Такой UI легко тестировать: ошибки, включенное состояние и autofill semantics являются производными от одной модели.

## Проверка программа чтения с экрана сценариев

Для текстового ввода проверяйте не только "можно ли напечатать". Минимальный accessibility проверочный список:

- label читается до значения;
- заглушка не является единственным именем поля;
- ошибка объявляется после submit;
- ошибка остается рядом с полем;
- поле можно найти по label в tests;
- password visibility toggle имеет понятное описание;
- focus переходит в ожидаемом порядке;
- IME action соответствует визуальному действию;
- autofill chip появляется для credentials;
- новые credentials предлагают сохранение только после успешного submit;
- sensitive поля не пишутся в logs.

## Тестирование

Compose UI tests работают через semantics tree, поэтому label, error и text важны не только для accessibility.

```kotlin
import androidx.compose.ui.test.assertIsDisplayed
import androidx.compose.ui.test.junit4.createComposeRule
import androidx.compose.ui.test.onNodeWithText
import androidx.compose.ui.test.performClick
import androidx.compose.ui.test.performTextInput
import com.google.common.truth.Truth.assertThat
import org.junit.Rule
import org.junit.Test

class SignInFormTest {

    @get:Rule
    val composeRule = createComposeRule()

    @Test
    fun invalidEmail_showsErrorMessage() {
        var currentState = SignInFormState()

        composeRule.setContent {
            SignInForm(
                state = currentState,
                onEmailChange = {
                    currentState = currentState.copy(email = it)
                },
                onPasswordChange = {
                    currentState = currentState.copy(password = it)
                },
                onSubmit = {
                    currentState = currentState.copy(
                        emailError = "Введите email с символом @"
                    )
                }
            )
        }

        composeRule
            .onNodeWithText("Email")
            .performTextInput("broken")

        composeRule
            .onNodeWithText("Войти")
            .performClick()

        composeRule
            .onNodeWithText("Введите email с символом @")
            .assertIsDisplayed()

        assertThat(currentState.email).isEqualTo("broken")
    }
}
```

В реальном тесте state обычно приходит из test ViewModel или держатель состояния. Здесь пример оставлен компактным, чтобы показать связь ввода, submit и ошибки.

## Типичные ошибки

### Использовать заглушка вместо label

Пользователь начинает вводить текст, заглушка исчезает, и поле теряет имя.

### Показывать ошибку только цветом

Цветовая рамка не объясняет проблему и может быть незаметна при color blindness.

### Объявлять каждую мелочь через live region

Программа чтения с экрана превращается в шумную ленту событий. Объявляйте только важные изменения.

### Забывать `ContentType`

Autofill service получает меньше контекста, а password manager не понимает форму.

### Писать пароль в logs

Ни validation, ни аналитика, ни crash breadcrumbs не должны сохранять raw password.

### Смешивать visual mask и domain value

UI может показывать `1234 5678`, но domain обычно должен работать с нормализованным `12345678`.

### Делать submit button единственным источником ошибки

Ошибка должна быть связана с конкретным полем, иначе пользователь не понимает, где исправлять.

## Итоги

Доступный текстовый ввод в Compose строится из нескольких слоев:

- `label` дает полю устойчивое имя;
- `placeholder` показывает пример, но не заменяет label;
- `supportingText` объясняет ограничения и ошибки;
- `isError` включает Material error state;
- `semantics { error(...) }` делает ошибку частью accessibility tree;
- `liveRegion` подходит для важных announcements после submit;
- `ContentType` включает современный autofill path;
- `LocalAutofillManager` позволяет явно `commit` или `cancel`;
- password fields требуют visual transformation, keyboard type и privacy discipline;
- `KeyboardOptions` и `KeyboardActions` согласуют UI с IME;
- форма должна иметь явную state model, а не набор случайных флагов.
