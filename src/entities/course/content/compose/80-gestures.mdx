## Что такое gesture в Compose

Gesture — это не одно pointer event, а распознанная последовательность событий: tap, long press, drag, fling, pinch zoom, rotation, swipe, dismiss. Compose дает готовые API жестов, чтобы вам не приходилось вручную собирать state machine из raw pointer events.

Иерархия выбора API:

- `Button`, `IconButton`, `Switch`, `Slider`, `SwipeToDismissBox`: готовый компонент;
- `clickable`, `combinedClickable`, `toggleable`, `selectable`: interaction modifier;
- `draggable`, `scrollable`, `transformable`: high-level gesture modifier;
- `detectTapGestures`, `detectDragGestures`, `detectTransformGestures`: detector внутри `pointerInput`;
- `pointerInput` с `awaitPointerEventScope`: низкоуровневый protocol.

```kotlin
import androidx.compose.foundation.gestures.detectTapGestures
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.input.pointer.pointerInput
import androidx.compose.ui.unit.dp

@Composable
fun TapGestureSurface(
    onTap: (Offset) -> Unit,
    onLongPress: (Offset) -> Unit
) {
    Box(
        modifier = Modifier
            .size(160.dp)
            .pointerInput(Unit) {
                detectTapGestures(
                    onTap = onTap,
                    onLongPress = onLongPress
                )
            }
    )
}
```

Этот код хорошо показывает gesture detector, но для обычной кнопки он слишком низкоуровневый. У него нет роли, label и keyboard activation.

<CatTip>
Если пользователь воспринимает элемент как кнопку, checkbox, switch или list item, начните с компонента или interaction modifier. Gesture detector выбирайте, когда поведение действительно жестовое.
</CatTip>

## Tap, press, double tap и long press

`detectTapGestures` поддерживает:

- `onPress`: вызывается на down и может ждать release/cancel;
- `onTap`: обычный tap;
- `onDoubleTap`: двойное нажатие;
- `onLongPress`: долгое нажатие.

```kotlin
import androidx.compose.foundation.background
import androidx.compose.foundation.gestures.detectTapGestures
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.runtime.mutableStateOf
import androidx.compose.runtime.remember
import androidx.compose.runtime.setValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.input.pointer.pointerInput
import androidx.compose.ui.unit.dp

@Composable
fun PressAwareSurface(
    onOpen: () -> Unit,
    onShowMenu: () -> Unit
) {
    var pressed by remember {
        mutableStateOf(false)
    }

    Box(
        modifier = Modifier
            .size(120.dp)
            .background(
                if (pressed) {
                    Color(0xFFBBDEFB)
                } else {
                    Color(0xFFE3F2FD)
                }
            )
            .pointerInput(Unit) {
                detectTapGestures(
                    onPress = {
                        pressed = true
                        val released = tryAwaitRelease()
                        pressed = false

                        if (!released) {
                            pressed = false
                        }
                    },
                    onTap = {
                        onOpen()
                    },
                    onLongPress = {
                        onShowMenu()
                    }
                )
            }
    )
}
```

`tryAwaitRelease()` возвращает `true`, если pointer был отпущен нормальным release. Если gesture отменен, нужно сбросить pressed state.

<MouseTrap>
Не прячьте важное действие только за long press или double tap. Эти жесты хуже обнаруживаются, сложнее для motor accessibility и не всегда удобны на desktop.
</MouseTrap>

## `draggable`

`Modifier.draggable` распознает drag вдоль одной оси и отдает delta. Он не меняет offset сам. Вы сами решаете, как state влияет на UI.

```kotlin
import androidx.compose.foundation.background
import androidx.compose.foundation.gestures.Orientation
import androidx.compose.foundation.gestures.draggable
import androidx.compose.foundation.gestures.rememberDraggableState
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.offset
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.runtime.mutableFloatStateOf
import androidx.compose.runtime.remember
import androidx.compose.runtime.setValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.IntOffset
import androidx.compose.ui.unit.dp
import kotlin.math.roundToInt

@Composable
fun HorizontalDragHandle() {
    var offsetX by remember {
        mutableFloatStateOf(0f)
    }

    val draggableState = rememberDraggableState { delta ->
        offsetX = (offsetX + delta).coerceIn(0f, 240f)
    }

    Box(
        modifier = Modifier
            .offset {
                IntOffset(
                    x = offsetX.roundToInt(),
                    y = 0
                )
            }
            .size(56.dp)
            .background(Color(0xFF1976D2))
            .draggable(
                state = draggableState,
                orientation = Orientation.Horizontal
            )
    )
}
```

`draggable` хорош для handles, sliders, panels и пользовательские элементы управления. Для scrollable content используйте `verticalScroll`, `LazyColumn` или `scrollable`.

## Drag lifecycle

У `draggable` есть обратные вызовы:

- `onDragStarted`: gesture активировался;
- `onDragStopped`: drag закончился, можно обработать velocity;
- `enabled`: включен ли drag;
- `reverseDirection`: нужно ли инвертировать delta.

```kotlin
import androidx.compose.foundation.gestures.Orientation
import androidx.compose.foundation.gestures.draggable
import androidx.compose.foundation.gestures.rememberDraggableState
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.runtime.mutableFloatStateOf
import androidx.compose.runtime.mutableStateOf
import androidx.compose.runtime.remember
import androidx.compose.runtime.setValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.unit.dp

@Composable
fun DragLifecycleBox(
    onDragFinished: (Float) -> Unit
) {
    var offsetX by remember {
        mutableFloatStateOf(0f)
    }
    var dragging by remember {
        mutableStateOf(false)
    }

    Box(
        modifier = Modifier
            .size(96.dp)
            .draggable(
                state = rememberDraggableState { delta ->
                    offsetX += delta
                },
                orientation = Orientation.Horizontal,
                onDragStarted = {
                    dragging = true
                },
                onDragStopped = { velocity ->
                    dragging = false
                    onDragFinished(velocity)
                }
            )
    )
}
```

Velocity приходит в пикселях в секунду. Для snap, fling и settle-animation используйте animation APIs, а не мгновенное округление.

## `detectDragGestures`

`detectDragGestures` дает больше контроля, чем `draggable`, и работает внутри `pointerInput`.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.gestures.detectDragGestures
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.mutableStateListOf
import androidx.compose.runtime.remember
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.input.pointer.pointerInput
import androidx.compose.ui.unit.dp

@Composable
fun FreehandGestureCanvas() {
    val points = remember {
        mutableStateListOf<Offset>()
    }

    Canvas(
        modifier = Modifier
            .size(260.dp)
            .pointerInput(Unit) {
                detectDragGestures(
                    onDragStart = { start ->
                        points.add(start)
                    },
                    onDrag = { change, dragAmount ->
                        change.consume()
                        points.add(change.position)
                    }
                )
            }
    ) {
        points.windowed(
            size = 2,
            step = 1
        ).forEach { segment ->
            drawLine(
                color = Color(0xFF1565C0),
                start = segment[0],
                end = segment[1],
                strokeWidth = 3.dp.toPx()
            )
        }
    }
}
```

Для drawing surface pointer input оправдан. Но если это просто перемещение panel по X, `draggable` будет проще.

## Swipe

Swipe — это drag с намерением перейти в другое состояние. Важны:

- направление;
- порог distance;
- velocity;
- состояние после отпускания;
- отмена;
- доступная альтернатива для accessibility.

Для списков используйте готовые компоненты, если они подходят.

```kotlin
import androidx.compose.material3.ListItem
import androidx.compose.material3.SwipeToDismissBox
import androidx.compose.material3.SwipeToDismissBoxValue
import androidx.compose.material3.Text
import androidx.compose.material3.rememberSwipeToDismissBoxState
import androidx.compose.runtime.Composable

@Composable
fun DismissibleMessageRow(
    title: String,
    onDismiss: () -> Unit
) {
    val dismissState = rememberSwipeToDismissBoxState(
        confirmValueChange = { value ->
            if (value == SwipeToDismissBoxValue.EndToStart) {
                onDismiss()
                true
            } else {
                false
            }
        }
    )

    SwipeToDismissBox(
        state = dismissState,
        backgroundContent = {
            Text(text = "Удалить")
        }
    ) {
        ListItem(
            headlineContent = {
                Text(text = title)
            }
        )
    }
}
```

Swipe-to-dismiss должен иметь альтернативу: кнопку, menu action или `CustomAccessibilityAction`.

## Fling

Fling — движение после отпускания с velocity. В Compose fling обычно обрабатывают:

- scroll containers;
- `draggable` через `onDragStopped`;
- `Animatable` с decay animation;
- `ScrollableState`;
- готовые компоненты вроде pager или sheet.

```kotlin
import androidx.compose.animation.core.Animatable
import androidx.compose.animation.core.exponentialDecay
import androidx.compose.foundation.background
import androidx.compose.foundation.gestures.Orientation
import androidx.compose.foundation.gestures.draggable
import androidx.compose.foundation.gestures.rememberDraggableState
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.offset
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.remember
import androidx.compose.runtime.rememberCoroutineScope
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.IntOffset
import androidx.compose.ui.unit.dp
import kotlinx.coroutines.launch
import kotlin.math.roundToInt

@Composable
fun FlingingHandle() {
    val scope = rememberCoroutineScope()
    val offsetX = remember {
        Animatable(0f)
    }
    val decay = remember {
        exponentialDecay<Float>()
    }

    Box(
        modifier = Modifier
            .offset {
                IntOffset(
                    x = offsetX.value.roundToInt(),
                    y = 0
                )
            }
            .size(56.dp)
            .background(Color(0xFF00897B))
            .draggable(
                orientation = Orientation.Horizontal,
                state = rememberDraggableState { delta ->
                    scope.launch {
                        offsetX.snapTo(
                            targetValue = (offsetX.value + delta).coerceIn(0f, 320f)
                        )
                    }
                },
                onDragStopped = { velocity ->
                    scope.launch {
                        offsetX.animateDecay(
                            initialVelocity = velocity,
                            animationSpec = decay
                        )
                    }
                }
            )
    )
}
```

В боевой нужно ограничивать decay bounds и корректно останавливать animation при новом drag. Для сложных anchored states лучше использовать anchored APIs.

## Transform gestures

Transform gesture объединяет pan, zoom и rotation. Он нужен для карт, изображений, canvas editors и media viewers.

```kotlin
import androidx.compose.foundation.Image
import androidx.compose.foundation.gestures.detectTransformGestures
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.runtime.mutableFloatStateOf
import androidx.compose.runtime.mutableStateOf
import androidx.compose.runtime.remember
import androidx.compose.runtime.setValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.input.pointer.pointerInput
import androidx.compose.ui.layout.ContentScale
import androidx.compose.ui.res.painterResource
import androidx.compose.ui.unit.dp

@Composable
fun TransformableImage() {
    var scale by remember {
        mutableFloatStateOf(1f)
    }
    var rotation by remember {
        mutableFloatStateOf(0f)
    }
    var offset by remember {
        mutableStateOf(Offset.Zero)
    }

    Image(
        painter = painterResource(id = R.drawable.cover),
        contentDescription = "Изображение для просмотра",
        contentScale = ContentScale.Crop,
        modifier = Modifier
            .size(280.dp)
            .graphicsLayer {
                scaleX = scale
                scaleY = scale
                rotationZ = rotation
                translationX = offset.x
                translationY = offset.y
            }
            .pointerInput(Unit) {
                detectTransformGestures { centroid, pan, zoom, gestureRotation ->
                    scale = (scale * zoom).coerceIn(0.75f, 4f)
                    rotation += gestureRotation
                    offset += pan
                }
            }
    )
}
```

Для accessibility добавьте явные controls: увеличить, уменьшить, сбросить, повернуть. Multi-touch не должен быть единственным способом управления.

## `transformable`

`Modifier.transformable` отделяет gesture state от UI.

```kotlin
import androidx.compose.foundation.gestures.rememberTransformableState
import androidx.compose.foundation.gestures.transformable
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.runtime.mutableFloatStateOf
import androidx.compose.runtime.mutableStateOf
import androidx.compose.runtime.remember
import androidx.compose.runtime.setValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

@Composable
fun TransformableSurface() {
    var scale by remember {
        mutableFloatStateOf(1f)
    }
    var rotation by remember {
        mutableFloatStateOf(0f)
    }
    var offset by remember {
        mutableStateOf(Offset.Zero)
    }

    val transformState = rememberTransformableState { zoomChange, panChange, rotationChange ->
        scale = (scale * zoomChange).coerceIn(0.5f, 3f)
        offset += panChange
        rotation += rotationChange
    }

    Box(
        modifier = Modifier
            .size(220.dp)
            .graphicsLayer {
                scaleX = scale
                scaleY = scale
                rotationZ = rotation
                translationX = offset.x
                translationY = offset.y
            }
            .transformable(transformState)
    )
}
```

`transformable` удобен, когда вы хотите использовать стандартную multi-touch механику без ручного `pointerInput`.

## `anchoredDraggable`

`anchoredDraggable` связывает drag с дискретными anchors. Это подход для sheets, drawers, swipe controls и panels, которые должны оседать в определенных состояниях.

```kotlin
import androidx.compose.animation.core.exponentialDecay
import androidx.compose.animation.core.tween
import androidx.compose.foundation.ExperimentalFoundationApi
import androidx.compose.foundation.background
import androidx.compose.foundation.gestures.AnchoredDraggableState
import androidx.compose.foundation.gestures.DraggableAnchors
import androidx.compose.foundation.gestures.Orientation
import androidx.compose.foundation.gestures.anchoredDraggable
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.foundation.layout.height
import androidx.compose.foundation.layout.offset
import androidx.compose.runtime.Composable
import androidx.compose.runtime.LaunchedEffect
import androidx.compose.runtime.remember
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.platform.LocalDensity
import androidx.compose.ui.unit.IntOffset
import androidx.compose.ui.unit.dp
import kotlin.math.roundToInt

enum class SheetAnchor {
    Collapsed,
    Expanded
}

@OptIn(ExperimentalFoundationApi::class)
@Composable
fun AnchoredBottomPanel() {
    val density = LocalDensity.current

    val state = remember(density) {
        AnchoredDraggableState(
            initialValue = SheetAnchor.Collapsed,
            positionalThreshold = { distance ->
                distance * 0.5f
            },
            velocityThreshold = {
                with(density) {
                    120.dp.toPx()
                }
            },
            snapAnimationSpec = tween(durationMillis = 220),
            decayAnimationSpec = exponentialDecay()
        )
    }

    val anchors = with(density) {
        DraggableAnchors {
            SheetAnchor.Collapsed at 0f
            SheetAnchor.Expanded at -320.dp.toPx()
        }
    }

    LaunchedEffect(
        state,
        anchors
    ) {
        state.updateAnchors(anchors)
    }

    Box(
        modifier = Modifier
            .offset {
                IntOffset(
                    x = 0,
                    y = state.requireOffset().roundToInt()
                )
            }
            .fillMaxWidth()
            .height(360.dp)
            .background(Color.White)
            .anchoredDraggable(
                state = state,
                orientation = Orientation.Vertical
            )
    )
}
```

Anchors зависят от density и layout size. Если anchors зависят от измеренного размера, обновляйте их после получения size, например через `onSizeChanged`.

## Конфликт жестов

Gesture conflict возникает, когда несколько элементов хотят один pointer stream:

- child horizontal drag внутри parent vertical scroll;
- карта внутри scroll screen;
- image zoom внутри pager;
- swipe-to-dismiss внутри lazy list;
- drawer gesture поверх content gestures.

Стратегии:

- использовать готовые components, которые уже решают conflict;
- ограничить направление gesture;
- consume-ить change только после уверенного распознавания;
- использовать nested scroll;
- держать gestures на одном уровне hierarchy;
- не смешивать raw pointer input и high-level gestures без причины.

## Accessibility gestures

Любой complex gesture требует альтернативы:

- кнопка для swipe action;
- пользовательский accessibility action;
- menu item;
- keyboard shortcut;
- focusable control;
- visible affordance;
- state description.

```kotlin
import androidx.compose.material3.ListItem
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.CustomAccessibilityAction
import androidx.compose.ui.semantics.customActions
import androidx.compose.ui.semantics.semantics

@Composable
fun AccessibleSwipeAlternative(
    title: String,
    onRemove: () -> Unit
) {
    ListItem(
        headlineContent = {
            Text(text = title)
        },
        modifier = Modifier.semantics {
            customActions = listOf(
                CustomAccessibilityAction(
                    label = "Удалить",
                    action = {
                        onRemove()
                        true
                    }
                )
            )
        }
    )
}
```

Сложный жест без альтернативы исключает часть пользователей.

## Типичные ошибки

### Использовать detector вместо компонента

Если есть `SwipeToDismissBox`, `Slider`, `Button`, `LazyColumn` или `Pager`, начинайте с них.

### Не различать drag и swipe

Drag может быть непрерывным перемещением. Swipe обычно приводит к состоянию после отпускания.

### Не учитывать velocity

Пользователь может сделать короткий, но быстрый swipe. Порог только по distance часто ощущается плохо.

### Терять pressed state при cancel

`onPress` всегда должен сбрасывать визуальное состояние после release или cancellation.

### Давать только multi-touch управление

Zoom и rotate должны иметь доступные альтернативы.

## Итоги

Gestures в Compose строятся слоями:

- готовые components дают лучший baseline;
- `clickable` и `combinedClickable` подходят для interaction, а не raw gestures;
- `detectTapGestures` покрывает tap, double tap, long press и press lifecycle;
- `draggable` дает осевой drag и velocity;
- `detectDragGestures` подходит для собственное рисование и freeform drag;
- `SwipeToDismissBox` закрывает частый dismiss pattern;
- fling требует velocity-aware animation;
- transform gestures дают zoom, pan и rotation;
- `anchoredDraggable` связывает drag с дискретными состояниями;
- каждый сложный gesture требует accessibility alternative.
