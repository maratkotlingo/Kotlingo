## Что такое бесконечная анимация

Бесконечная анимация — это animation, которая повторяется, пока ее composable находится в composition. В Compose основной API для таких случаев — `rememberInfiniteTransition`. Он создает `InfiniteTransition`, внутри которого можно объявить дочерние анимации через `animateFloat`, `animateColor` или `animateValue`.

Бесконечные анимации подходят для UI, где состояние действительно продолжается во времени:

- загрузка shimmer;
- pulse для активного индикатора;
- breathing highlight;
- looping прогресс;
- анимированный skeleton;
- декоративное движение в onboarding;
- мягкий attention cue для нового элемента.

```kotlin
import androidx.compose.animation.core.RepeatMode
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.infiniteRepeatable
import androidx.compose.animation.core.rememberInfiniteTransition
import androidx.compose.animation.core.tween
import androidx.compose.foundation.background
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.height
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.draw.alpha
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun PulsingLoadingBar() {
    val infiniteTransition = rememberInfiniteTransition(
        label = "loading pulse"
    )

    val alpha by infiniteTransition.animateFloat(
        initialValue = 0.35f,
        targetValue = 1f,
        animationSpec = infiniteRepeatable(
            animation = tween(durationMillis = 700),
            repeatMode = RepeatMode.Reverse
        ),
        label = "loading alpha"
    )

    Box(
        modifier = Modifier
            .fillMaxWidth()
            .height(12.dp)
            .alpha(alpha)
            .background(Color.LightGray)
    )
}
```

Этот компонент будет пульсировать, пока находится в composition. Когда composable исчезнет, transition остановится вместе с ним.

<CatTip>
Бесконечная анимация должна быть условной. Если данные загрузились, уберите shimmer или pulse из composition, а не оставляйте его под настоящим контентом.
</CatTip>

## `rememberInfiniteTransition`

`rememberInfiniteTransition` создает объект `InfiniteTransition`, который управляет дочерними бесконечными анимациями.

Ключевые свойства:

- создается в composition;
- запоминается в место вызова;
- дочерние animations стартуют при входе в composition;
- продолжаются бесконечно, пока composable не удален;
- может управлять несколькими child animations;
- не имеет target state как `updateTransition`;
- не является suspend API как `Animatable`;
- подходит только для loops.

```kotlin
val infiniteTransition = rememberInfiniteTransition(
    label = "component infinite transition"
)
```

`label` помогает отладке в tooling. Для повторяющихся компонентов лучше давать label, который описывает роль анимации, а не просто "animation".

## `animateFloat`

`InfiniteTransition.animateFloat` — самый частый вариант. Он подходит для alpha, scale, rotation, offset, shimmer прогресс и собственное рисование прогресс.

```kotlin
import androidx.compose.animation.core.LinearEasing
import androidx.compose.animation.core.RepeatMode
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.infiniteRepeatable
import androidx.compose.animation.core.rememberInfiniteTransition
import androidx.compose.animation.core.tween
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

@Composable
fun RotatingLoadingIcon() {
    val transition = rememberInfiniteTransition(
        label = "rotating loading icon"
    )

    val rotation by transition.animateFloat(
        initialValue = 0f,
        targetValue = 360f,
        animationSpec = infiniteRepeatable(
            animation = tween(
                durationMillis = 1_200,
                easing = LinearEasing
            ),
            repeatMode = RepeatMode.Restart
        ),
        label = "rotation"
    )

    Box(
        modifier = Modifier
            .size(40.dp)
            .graphicsLayer {
                // Вращение применяем на graphics layer, не меняя layout.
                rotationZ = rotation
            }
    )
}
```

Для rotation loops обычно используйте `LinearEasing`, иначе скорость будет ускоряться и замедляться каждый цикл.

## `RepeatMode.Restart` и `RepeatMode.Reverse`

`RepeatMode.Restart` каждый цикл начинает с initial value и идет к target value. Подходит для rotation, прогресс sweep, moving shimmer.

`RepeatMode.Reverse` идет от initial к target, затем обратно от target к initial. Подходит для pulse, breathing alpha, scale bounce без резкого скачка.

```kotlin
import androidx.compose.animation.core.RepeatMode
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.infiniteRepeatable
import androidx.compose.animation.core.rememberInfiniteTransition
import androidx.compose.animation.core.tween
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

@Composable
fun BreathingDot() {
    val transition = rememberInfiniteTransition(
        label = "breathing dot"
    )

    val scale by transition.animateFloat(
        initialValue = 0.82f,
        targetValue = 1.12f,
        animationSpec = infiniteRepeatable(
            animation = tween(durationMillis = 900),
            repeatMode = RepeatMode.Reverse
        ),
        label = "dot scale"
    )

    Box(
        modifier = Modifier
            .size(18.dp)
            .graphicsLayer {
                scaleX = scale
                scaleY = scale
            }
    )
}
```

Если для pulse использовать `Restart`, пользователь увидит скачок с target обратно к initial в начале каждого цикла.

## `animateColor`

`InfiniteTransition.animateColor` полезен для мягкой смены акцентного цвета, skeleton highlight или декоративного looping status.

```kotlin
import androidx.compose.animation.animateColor
import androidx.compose.animation.core.RepeatMode
import androidx.compose.animation.core.infiniteRepeatable
import androidx.compose.animation.core.rememberInfiniteTransition
import androidx.compose.animation.core.tween
import androidx.compose.foundation.background
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun SyncingColorIndicator() {
    val transition = rememberInfiniteTransition(
        label = "syncing color indicator"
    )

    val color by transition.animateColor(
        initialValue = Color(0xFF90CAF9),
        targetValue = Color(0xFF42A5F5),
        animationSpec = infiniteRepeatable(
            animation = tween(durationMillis = 800),
            repeatMode = RepeatMode.Reverse
        ),
        label = "sync color"
    )

    Box(
        modifier = Modifier
            .size(16.dp)
            .background(color)
    )
}
```

Следите за контрастом, если цвет влияет на текст или important status. Бесконечная смена цвета не должна мешать чтению.

## `animateValue`

`animateValue` позволяет анимировать собственный тип через `TwoWayConverter`.

```kotlin
import androidx.compose.animation.core.AnimationVector2D
import androidx.compose.animation.core.RepeatMode
import androidx.compose.animation.core.TwoWayConverter
import androidx.compose.animation.core.animateValue
import androidx.compose.animation.core.infiniteRepeatable
import androidx.compose.animation.core.rememberInfiniteTransition
import androidx.compose.animation.core.tween
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

data class OrbitPoint(
    val x: Float,
    val y: Float
)

val OrbitPointConverter = TwoWayConverter<OrbitPoint, AnimationVector2D>(
    convertToVector = { point ->
        AnimationVector2D(
            v1 = point.x,
            v2 = point.y
        )
    },
    convertFromVector = { vector ->
        OrbitPoint(
            x = vector.v1,
            y = vector.v2
        )
    }
)

@Composable
fun MovingPointIndicator() {
    val transition = rememberInfiniteTransition(
        label = "moving point"
    )

    val point by transition.animateValue(
        initialValue = OrbitPoint(
            x = 30f,
            y = 30f
        ),
        targetValue = OrbitPoint(
            x = 150f,
            y = 150f
        ),
        typeConverter = OrbitPointConverter,
        animationSpec = infiniteRepeatable(
            animation = tween(durationMillis = 1_000),
            repeatMode = RepeatMode.Reverse
        ),
        label = "point"
    )

    Canvas(
        modifier = Modifier.size(180.dp)
    ) {
        drawCircle(
            color = Color.Magenta,
            radius = 8.dp.toPx(),
            center = Offset(
                x = point.x,
                y = point.y
            )
        )
    }
}
```

Для большинства UI достаточно `animateFloat` и `animateColor`. `animateValue` используйте, когда пользовательский тип действительно делает код яснее.

## `infiniteRepeatable`

`infiniteRepeatable` задает повторяющийся spec. Он принимает:

- `animation`: базовый finite spec, например `tween` или `keyframes`;
- `repeatMode`: `Restart` или `Reverse`;
- `initialStartOffset`: необязательное смещение старта.

```kotlin
import androidx.compose.animation.core.StartOffset
import androidx.compose.animation.core.StartOffsetType
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.infiniteRepeatable
import androidx.compose.animation.core.rememberInfiniteTransition
import androidx.compose.animation.core.tween
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue

@Composable
fun rememberDelayedPulseAlpha(): Float {
    val transition = rememberInfiniteTransition(
        label = "delayed pulse"
    )

    val alpha by transition.animateFloat(
        initialValue = 0.25f,
        targetValue = 1f,
        animationSpec = infiniteRepeatable(
            animation = tween(durationMillis = 600),
            initialStartOffset = StartOffset(
                offsetMillis = 200,
                offsetType = StartOffsetType.Delay
            )
        ),
        label = "delayed alpha"
    )

    return alpha
}
```

Start offset полезен для staggered индикаторы загрузки, где несколько элементов должны двигаться в одной фазовой системе, но с разным сдвигом.

## Shimmer

Shimmer — типичная бесконечная анимация. Он создает ощущение загрузки skeleton-контента. В Compose shimmer обычно строят через анимированный прогресс и `Brush.linearGradient`.

```kotlin
import androidx.compose.animation.core.LinearEasing
import androidx.compose.animation.core.RepeatMode
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.infiniteRepeatable
import androidx.compose.animation.core.rememberInfiniteTransition
import androidx.compose.animation.core.tween
import androidx.compose.foundation.background
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.fillMaxWidth
import androidx.compose.foundation.layout.height
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.graphics.Brush
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp

@Composable
fun ShimmerLine(
    modifier: Modifier = Modifier
) {
    val transition = rememberInfiniteTransition(
        label = "shimmer line"
    )

    val progress by transition.animateFloat(
        initialValue = 0f,
        targetValue = 1f,
        animationSpec = infiniteRepeatable(
            animation = tween(
                durationMillis = 1_100,
                easing = LinearEasing
            ),
            repeatMode = RepeatMode.Restart
        ),
        label = "shimmer progress"
    )

    val width = 1_000f
    val startX = -width + width * 2f * progress
    val brush = Brush.linearGradient(
        colors = listOf(
            Color(0xFFE0E0E0),
            Color(0xFFF5F5F5),
            Color(0xFFE0E0E0)
        ),
        start = Offset(
            x = startX,
            y = 0f
        ),
        end = Offset(
            x = startX + width,
            y = 0f
        )
    )

    Box(
        modifier = modifier
            .fillMaxWidth()
            .height(18.dp)
            .background(brush)
    )
}
```

Этот пример прост для понимания, но в боевой среде можно оптимизировать brush через `drawWithCache`, если shimmer применяется в большом списке.

<MouseTrap>
Не оставляйте shimmer активным под загруженным контентом. Он продолжит обновлять кадры и тратить ресурсы, даже если пользователь его не видит.
</MouseTrap>

## Staggered dots

Три точки загрузки можно сделать через один `InfiniteTransition` и разные start offsets.

```kotlin
import androidx.compose.animation.core.RepeatMode
import androidx.compose.animation.core.StartOffset
import androidx.compose.animation.core.animateFloat
import androidx.compose.animation.core.infiniteRepeatable
import androidx.compose.animation.core.rememberInfiniteTransition
import androidx.compose.animation.core.tween
import androidx.compose.foundation.layout.Arrangement
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.Row
import androidx.compose.foundation.layout.size
import androidx.compose.foundation.shape.CircleShape
import androidx.compose.foundation.background
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

@Composable
fun LoadingDots() {
    val transition = rememberInfiniteTransition(
        label = "loading dots"
    )

    @Composable
    fun AnimatedDot(
        index: Int
    ) {
        val scale by transition.animateFloat(
            initialValue = 0.7f,
            targetValue = 1.2f,
            animationSpec = infiniteRepeatable(
                animation = tween(durationMillis = 450),
                repeatMode = RepeatMode.Reverse,
                initialStartOffset = StartOffset(
                    offsetMillis = index * 120
                )
            ),
            label = "dot $index scale"
        )

        Box(
            modifier = Modifier
                .size(8.dp)
                .graphicsLayer {
                    scaleX = scale
                    scaleY = scale
                }
                .background(
                    color = Color.Gray,
                    shape = CircleShape
                )
        )
    }

    Row(
        horizontalArrangement = Arrangement.spacedBy(6.dp)
    ) {
        repeat(3) { index ->
            AnimatedDot(index = index)
        }
    }
}
```

Для маленького индикатора такой код нормален. Для большого количества элементов лучше избегать создания множества бесконечных child animations.

## Lifecycle и остановка

`rememberInfiniteTransition` не имеет кнопки pause. Он активен, пока находится в composition. Поэтому остановка делается архитектурно:

- не показывать composable, если animation не нужна;
- переключать на статичный variant;
- использовать `Animatable`, если нужна pause/resume логика;
- учитывать lifecycle экрана;
- не держать бесконечную анимацию в item, который часто остается offscreen.

```kotlin
import androidx.compose.runtime.Composable

@Composable
fun LoadingContent(
    loading: Boolean,
    content: @Composable () -> Unit
) {
    if (loading) {
        ShimmerLine()
    } else {
        content()
    }
}
```

Это лучший способ остановки shimmer: убрать его из composition.

## Battery и performance

Бесконечная анимация создает постоянные frame updates. Это нормально для короткого состояние загрузки, но плохо для постоянного декоративного шума.

Практические правила:

- не используйте бесконечную анимацию как фоновое украшение всего приложения;
- не запускайте shimmer для десятков offscreen элементов;
- используйте lazy list keys и показывайте skeleton только для видимых элементов;
- избегайте тяжелых object allocations на каждый кадр;
- для gradients и paths используйте `drawWithCache`;
- предпочитайте `graphicsLayer` для scale, alpha, rotation;
- учитывайте accessibility и motion sensitivity;
- прекращайте animation, когда состояние завершилось.

## Accessibility

Бесконечная анимация может раздражать или отвлекать. Особенно это касается shimmer, мигания и постоянного движения.

Рекомендации:

- не делайте быстрые мерцающие loops;
- не анимируйте важный текст бесконечно;
- не используйте motion как единственный способ объяснить состояние;
- для загрузка добавляйте semantics, если индикатор важен;
- не перекрывайте TalkBack focus декоративной animation.

```kotlin
import androidx.compose.foundation.layout.Box
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.semantics.contentDescription
import androidx.compose.ui.semantics.semantics

@Composable
fun AccessibleLoadingShimmer() {
    Box(
        modifier = Modifier.semantics {
            contentDescription = "Загрузка данных"
        }
    ) {
        ShimmerLine()
    }
}
```

Если рядом уже есть `CircularProgressIndicator` с понятной semantics, декоративному shimmer может не понадобиться отдельное описание.

```kotlin
import androidx.compose.foundation.lazy.LazyColumn
import androidx.compose.foundation.lazy.items
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable

@Composable
fun ArticleListWithSkeletons(
    loading: Boolean,
    articles: List<String>
) {
    if (loading) {
        LazyColumn {
            items(
                items = List(5) { index ->
                    "skeleton-$index"
                },
                key = { key ->
                    key
                }
            ) {
                ArticleSkeletonCard()
            }
        }
    } else {
        LazyColumn {
            items(
                items = articles,
                key = { article ->
                    article
                }
            ) { article ->
                Text(text = article)
            }
        }
    }
}
```

Когда `loading == false`, skeleton composables исчезают, и их infinite transitions прекращаются.

## Типичные ошибки

### Использовать бесконечную анимацию как постоянный декор

Постоянное движение быстро утомляет и расходует ресурсы. Motion должен поддерживать состояние, а не заполнять пустоту.

### Не удалять animation из composition

Если индикатор больше не нужен, не скрывайте его alpha в ноль. Удалите composable.

### Использовать `RepeatMode.Restart` для pulse

Pulse должен быть плавным туда-обратно. Для него обычно нужен `RepeatMode.Reverse`.

### Создавать тяжелый brush на каждый кадр без нужды

Для сложных gradients используйте `drawWithCache`, особенно в списках.

### Делать слишком много независимых infinite transitions

Один экран с десятками loops может стать дорогим. Группируйте, ограничивайте область применения и проверяйте performance.

## Итоги

Бесконечные анимации в Compose строятся вокруг `rememberInfiniteTransition`:

- `InfiniteTransition` живет, пока composable в composition;
- `animateFloat`, `animateColor` и `animateValue` создают child animations;
- `infiniteRepeatable` задает повторение;
- `RepeatMode.Restart` подходит для циклов с возвратом к началу;
- `RepeatMode.Reverse` подходит для pulse и breathing effects;
- shimmer, dots и циклы загрузки нужно удалять из composition после загрузки;
- бесконечные анимации требуют внимания к battery, accessibility и performance;
- если нужна pause/resume логика, часто лучше использовать `Animatable`.
