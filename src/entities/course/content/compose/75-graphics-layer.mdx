## Что такое `graphicsLayer`

`Modifier.graphicsLayer` создает graphics layer для composable и позволяет применять к нему transform и compositing-настройки. В отличие от обычного layout modifier, `graphicsLayer` не меняет измерение child. Он влияет на то, как уже измеренный и нарисованный content будет преобразован на этапе drawing.

Через `graphicsLayer` можно управлять:

- `scaleX`, `scaleY`;
- `alpha`;
- `translationX`, `translationY`;
- `rotationX`, `rotationY`, `rotationZ`;
- `cameraDistance`;
- `transformOrigin`;
- `shadowElevation`;
- `ambientShadowColor`;
- `spotShadowColor`;
- `shape`;
- `clip`;
- `renderEffect`;
- `compositingStrategy`.

```kotlin
import androidx.compose.foundation.background
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

@Composable
fun ScaledSquare() {
    Box(
        modifier = Modifier
            .size(96.dp)
            .graphicsLayer {
                scaleX = 1.10f
                scaleY = 1.10f
            }
            .background(Color(0xFF90CAF9))
    )
}
```

Здесь layout size остается `96.dp`, но drawing увеличивается. Соседние элементы в layout не узнают, что визуально квадрат стал больше.

<CatTip>
`graphicsLayer` — это инструмент drawing phase. Он отлично подходит для анимаций transform, но не должен использоваться вместо layout, если элемент действительно должен занимать больше места.
</CatTip>

## Lambda overload и property overload

Чаще всего используется lambda overload:

```kotlin
Modifier.graphicsLayer {
    alpha = 0.8f
    rotationZ = 12f
}
```

Он удобен, когда нужно задать несколько свойств и читать animated values. Есть и overload с параметрами, но lambda обычно лучше читается в сложных случаях.

`graphicsLayer` важен для performance-анимаций: изменение layer properties может не требовать полного повторного drawing content. Но если вы меняете state, от которого зависит composition или layout, recomposition все равно произойдет.

## `translationX` и `translationY`

`translationX` и `translationY` сдвигают drawing в пикселях. Это не то же самое, что `Modifier.offset`, потому что offset участвует в layout positioning, а translation применяется на graphics layer.

```kotlin
import androidx.compose.animation.core.animateFloatAsState
import androidx.compose.foundation.background
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

@Composable
fun SwipeHintLayer(
    revealed: Boolean
) {
    val translation by animateFloatAsState(
        targetValue = if (revealed) {
            48f
        } else {
            0f
        },
        label = "translation hint"
    )

    Box(
        modifier = Modifier
            .size(
                width = 180.dp,
                height = 64.dp
            )
            .graphicsLayer {
                translationX = translation
            }
            .background(Color(0xFFE3F2FD))
    )
}
```

Если translation должен влиять на hit testing и соседний layout, используйте layout modifiers. Если это чисто визуальная анимация, `graphicsLayer` часто подходит лучше.

## `scaleX` и `scaleY`

Scale меняет drawing относительно `transformOrigin`.

```kotlin
import androidx.compose.animation.core.animateFloatAsState
import androidx.compose.foundation.background
import androidx.compose.foundation.interaction.MutableInteractionSource
import androidx.compose.foundation.interaction.collectIsPressedAsState
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.foundation.clickable
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.runtime.remember
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

@Composable
fun PressScaleTile(
    onClick: () -> Unit
) {
    val interactionSource = remember {
        MutableInteractionSource()
    }
    val pressed by interactionSource.collectIsPressedAsState()
    val scale by animateFloatAsState(
        targetValue = if (pressed) {
            0.96f
        } else {
            1f
        },
        label = "press scale"
    )

    Box(
        modifier = Modifier
            .size(96.dp)
            .graphicsLayer {
                scaleX = scale
                scaleY = scale
            }
            .background(Color(0xFFFFF3E0))
            .clickable(
                interactionSource = interactionSource,
                indication = null,
                onClick = onClick
            )
    )
}
```

Нажатие через scale хорошо работает для tiles и custom surfaces. Для стандартных кнопок сначала используйте Material components.

## `rotationZ`

`rotationZ` вращает элемент в плоскости экрана.

```kotlin
import androidx.compose.animation.core.animateFloatAsState
import androidx.compose.foundation.Image
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.res.painterResource
import androidx.compose.ui.unit.dp

@Composable
fun ExpandArrow(
    expanded: Boolean
) {
    val rotation by animateFloatAsState(
        targetValue = if (expanded) {
            180f
        } else {
            0f
        },
        label = "arrow rotation"
    )

    Image(
        painter = painterResource(id = R.drawable.ic_expand_more),
        contentDescription = null,
        modifier = Modifier
            .size(24.dp)
            .graphicsLayer {
                rotationZ = rotation
            }
    )
}
```

Для icons `graphicsLayer` часто удобнее, чем пересоздавать vector asset под каждый state.

## `rotationX`, `rotationY` и `cameraDistance`

`rotationX` и `rotationY` создают 3D-поворот относительно горизонтальной или вертикальной оси. `cameraDistance` влияет на перспективу: слишком маленькое значение дает сильное искажение, слишком большое делает эффект почти плоским.

```kotlin
import androidx.compose.animation.core.animateFloatAsState
import androidx.compose.foundation.background
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

@Composable
fun FlipCardFace(
    flipped: Boolean
) {
    val rotation by animateFloatAsState(
        targetValue = if (flipped) {
            180f
        } else {
            0f
        },
        label = "flip rotation"
    )

    Box(
        modifier = Modifier
            .size(
                width = 180.dp,
                height = 120.dp
            )
            .graphicsLayer {
                rotationY = rotation
                cameraDistance = 12f * density
            }
            .background(Color(0xFFE8EAF6))
    )
}
```

Для полноценной flip card обычно нужно переключать front/back content около 90 градусов и следить, чтобы обратная сторона не была зеркальной.

## `transformOrigin`

`transformOrigin` задает точку, вокруг которой применяются scale и rotation. Значения задаются как доли ширины и высоты.

```kotlin
import androidx.compose.animation.core.animateFloatAsState
import androidx.compose.foundation.background
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.TransformOrigin
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

@Composable
fun HingePanel(
    open: Boolean
) {
    val rotation by animateFloatAsState(
        targetValue = if (open) {
            -12f
        } else {
            0f
        },
        label = "hinge rotation"
    )

    Box(
        modifier = Modifier
            .size(
                width = 220.dp,
                height = 80.dp
            )
            .graphicsLayer {
                transformOrigin = TransformOrigin(
                    pivotFractionX = 0f,
                    pivotFractionY = 0.5f
                )
                rotationZ = rotation
            }
            .background(Color(0xFFC8E6C9))
    )
}
```

`TransformOrigin.Center` — default. Для дверцы, hinge, anchored menu и раскрывающихся panels pivot часто должен быть на краю.

## `alpha`

`alpha` управляет прозрачностью всего layer. `1f` означает полностью видимо, `0f` означает невидимо.

```kotlin
import androidx.compose.animation.core.animateFloatAsState
import androidx.compose.foundation.background
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.runtime.getValue
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

@Composable
fun FadingStatusDot(
    visible: Boolean
) {
    val alphaValue by animateFloatAsState(
        targetValue = if (visible) {
            1f
        } else {
            0f
        },
        label = "dot alpha"
    )

    Box(
        modifier = Modifier
            .size(16.dp)
            .graphicsLayer {
                alpha = alphaValue
            }
            .background(Color(0xFF2E7D32))
    )
}
```

Когда `alpha` меньше `1f`, Compose может создать offscreen buffer, если `compositingStrategy` не говорит иначе. Это важно для performance и blend behavior.

<MouseTrap>
`alpha = 0f` скрывает drawing, но не удаляет composable из composition, layout и semantics. Если элемент должен исчезнуть из accessibility tree или освободить место, используйте состояние composition/layout, а не только alpha.
</MouseTrap>

## `CompositingStrategy`

`CompositingStrategy` говорит, как layer объединяется с уже нарисованным содержимым экрана.

Основные варианты:

- `CompositingStrategy.Auto`: Compose сам выбирает стратегию;
- `CompositingStrategy.Offscreen`: сначала рисовать layer в offscreen buffer, затем compositing;
- `CompositingStrategy.ModulateAlpha`: применять alpha к draw calls без offscreen buffer, если это возможно.

```kotlin
import androidx.compose.foundation.Canvas
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.geometry.Offset
import androidx.compose.ui.graphics.BlendMode
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.CompositingStrategy
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

@Composable
fun CutoutWithOffscreenLayer() {
    Canvas(
        modifier = Modifier
            .size(140.dp)
            .graphicsLayer {
                compositingStrategy = CompositingStrategy.Offscreen
            }
    ) {
        drawCircle(
            color = Color(0xFF1976D2),
            radius = size.minDimension / 2f
        )
        drawCircle(
            color = Color.Transparent,
            radius = 28.dp.toPx(),
            center = Offset(
                x = size.width / 2f,
                y = size.height / 2f
            ),
            blendMode = BlendMode.Clear
        )
    }
}
```

`Offscreen` важен для эффектов вроде `BlendMode.Clear`, когда нужно вырезать часть именно из текущего layer, а не из всего, что уже нарисовано под ним.

## `shape` и `clip`

Через `graphicsLayer` можно задать clipping shape.

```kotlin
import androidx.compose.foundation.Image
import androidx.compose.foundation.layout.size
import androidx.compose.foundation.shape.RoundedCornerShape
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.layout.ContentScale
import androidx.compose.ui.res.painterResource
import androidx.compose.ui.unit.dp

@Composable
fun RoundedCoverLayer() {
    Image(
        painter = painterResource(id = R.drawable.cover),
        contentDescription = null,
        contentScale = ContentScale.Crop,
        modifier = Modifier
            .size(
                width = 220.dp,
                height = 140.dp
            )
            .graphicsLayer {
                shape = RoundedCornerShape(24.dp)
                clip = true
            }
    )
}
```

Если нужно только clipping, `Modifier.clip(shape)` обычно читается проще. `graphicsLayer` выбирайте, когда clipping комбинируется с transform, alpha, shadow или render effect.

## `shadowElevation`

`shadowElevation` добавляет shadow на graphics layer. Для Material surfaces чаще используйте `Surface` или `Modifier.shadow`, но layer shadow полезна при transform-анимациях.

```kotlin
import androidx.compose.foundation.background
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.foundation.shape.RoundedCornerShape
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

@Composable
fun LayerShadowCard() {
    val shape = RoundedCornerShape(18.dp)

    Box(
        modifier = Modifier
            .size(
                width = 180.dp,
                height = 110.dp
            )
            .graphicsLayer {
                shadowElevation = 12.dp.toPx()
                this.shape = shape
                clip = false
            }
            .background(
                color = Color.White,
                shape = shape
            )
    )
}
```

`shadowElevation` задается в пикселях внутри lambda, поэтому `12.dp.toPx()` уместен.

## `renderEffect`

`renderEffect` позволяет применить platform render effect к layer. Например, blur доступен на Android 12 и выше.

```kotlin
import android.graphics.RenderEffect
import android.graphics.Shader
import android.os.Build
import androidx.compose.foundation.Image
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.asComposeRenderEffect
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.layout.ContentScale
import androidx.compose.ui.res.painterResource
import androidx.compose.ui.unit.dp

@Composable
fun BlurredCover(
    blurEnabled: Boolean
) {
    Image(
        painter = painterResource(id = R.drawable.cover),
        contentDescription = null,
        contentScale = ContentScale.Crop,
        modifier = Modifier
            .size(
                width = 220.dp,
                height = 140.dp
            )
            .graphicsLayer {
                if (blurEnabled && Build.VERSION.SDK_INT >= Build.VERSION_CODES.S) {
                    renderEffect = RenderEffect
                        .createBlurEffect(
                            18f,
                            18f,
                            Shader.TileMode.CLAMP
                        )
                        .asComposeRenderEffect()
                }
            }
    )
}
```

Render effects могут быть дорогими. Не применяйте blur к большим scrolling surfaces без профилирования.

## Layer и invalidation

`graphicsLayer` может быть эффективен для анимаций, потому что changing layer properties может не требовать перерисовки внутреннего content. Но это не магия:

- если меняется composable state, который влияет на content, будет recomposition;
- если меняются constraints, будет layout;
- если меняется draw code, будет drawing invalidation;
- если меняются только layer properties, часто достаточно обновить layer;
- offscreen buffer потребляет память;
- render effect и alpha могут добавить compositing cost.

## Hit testing и layout bounds

Transforms не меняют layout bounds. Это влияет на touch:

```kotlin
import androidx.compose.foundation.background
import androidx.compose.foundation.clickable
import androidx.compose.foundation.layout.Box
import androidx.compose.foundation.layout.size
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.graphicsLayer
import androidx.compose.ui.unit.dp

@Composable
fun VisuallyLargeClickableBox(
    onClick: () -> Unit
) {
    Box(
        modifier = Modifier
            .size(48.dp)
            .graphicsLayer {
                scaleX = 1.6f
                scaleY = 1.6f
            }
            .background(Color(0xFFFFCDD2))
            .clickable(onClick = onClick)
    )
}
```

Визуально элемент больше `48.dp`, но его layout и touch area остаются прежними. Если нужна большая touch target, увеличивайте layout size или padding.

## Когда использовать `graphicsLayer`

Используйте `graphicsLayer`, когда нужно:

- анимировать alpha, scale, rotation или translation без изменения layout;
- сделать 3D rotation;
- настроить transform origin;
- применить render effect;
- управлять compositing strategy;
- сделать layer clipping вместе с transform;
- использовать `BlendMode.Clear` безопасно через offscreen layer;
- визуально поднять layer при transform-анимации.

Не используйте `graphicsLayer`, когда:

- нужно изменить фактический размер layout;
- нужно изменить порядок элементов в parent layout;
- достаточно `Modifier.alpha`, `Modifier.rotate` или `Modifier.scale` для простого случая;
- эффект можно выразить через Material component;
- слой создается для каждого item списка без необходимости;
- вы скрываете важный semantics-only content через alpha.

## Типичные ошибки

### Путать translation и offset

Translation меняет drawing, offset влияет на positioning. Для жестов это различие особенно важно.

### Делать `alpha = 0f` вместо удаления элемента

Элемент может остаться фокусируемым, кликабельным или доступным для screen reader.

### Включать offscreen без причины

Offscreen buffer помогает для blend effects, но стоит памяти и compositing work.

### Слишком маленький `cameraDistance`

3D rotation начинает выглядеть агрессивно и ломает восприятие размеров.

### Забывать про `transformOrigin`

Если panel должна открываться от края, default center pivot даст странный motion.

## Итоги

`graphicsLayer` — мощный graphics modifier:

- он работает после layout;
- transform не меняет размер и позицию в parent layout;
- scale, rotation и translation подходят для motion;
- alpha может создать offscreen buffer;
- `CompositingStrategy` управляет способом compositing;
- `shape` и `clip` позволяют обрезать layer;
- `shadowElevation` задается в пикселях;
- `renderEffect` открывает platform effects;
- hit area не обязана совпадать с визуально transformed bounds;
- любые layer-эффекты нужно проверять на performance и accessibility.
