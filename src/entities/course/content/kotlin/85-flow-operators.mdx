## Operators собирают pipeline

Flow operators позволяют описать stream processing без ручного управления coroutine lifecycle.

Есть три группы:

- creation operators: `flow`, `flowOf`, `asFlow`;
- intermediate operators: `map`, `filter`, `transform`, `take`, `drop`, `debounce`, `combine`, `catch`;
- terminal operators: `collect`, `toList`, `first`, `single`, `fold`, `reduce`, `launchIn`.

```kotlin
import kotlinx.coroutines.flow.asFlow
import kotlinx.coroutines.flow.filter
import kotlinx.coroutines.flow.map
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

fun operatorPipelineExample() = runBlocking {
    val titles = listOf("flow", "operators", "stateflow")
        .asFlow()
        .filter { word -> word.length > 4 }
        .map { word -> word.replaceFirstChar { char -> char.uppercase() } }
        .toList()

    println(titles)
}
```

Pipeline is lazy until terminal operator.

<CatTip>
Операторы Flow — это кошачья тропинка: каждый шаг понятен отдельно, а вместе они ведут значение от источника к collector-у.
</CatTip>

## `map`

`map` превращает each value into another value.

```kotlin
import kotlinx.coroutines.flow.asFlow
import kotlinx.coroutines.flow.map
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

fun mapExample() = runBlocking {
    val cards = listOf("84-flow", "85-flow-operators")
        .asFlow()
        .map { slug ->
            LessonCard(
                slug = slug,
                title = slug.substringAfter("-").replace("-", " ")
            )
        }
        .toList()

    println(cards)
}

data class LessonCard(
    val slug: String,
    val title: String
)
```

Use cases:

- DTO to domain;
- domain to UI state;
- text normalization;
- extracting fields;
- formatting values.

## `filter`

`filter` пропускает только values, которые satisfy predicate.

```kotlin
import kotlinx.coroutines.flow.asFlow
import kotlinx.coroutines.flow.filter
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

fun filterExample() = runBlocking {
    val advanced = listOf(
        LessonMeta("84-flow", difficulty = "Продвинутый"),
        LessonMeta("01-introduction", difficulty = "Начальный")
    )
        .asFlow()
        .filter { lesson -> lesson.difficulty == "Продвинутый" }
        .toList()

    println(advanced)
}

data class LessonMeta(
    val slug: String,
    val difficulty: String
)
```

Related filtering operators:

- `filterNot` — inverse predicate;
- `filterIsInstance` — оставить values конкретного runtime type;
- `filterNotNull` — убрать null values;
- `dropWhile` — пропускать values while predicate true;
- `takeWhile` — брать values while predicate true.

```kotlin
import kotlinx.coroutines.flow.asFlow
import kotlinx.coroutines.flow.filterNotNull
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

fun filterNotNullExample() = runBlocking {
    val slugs = listOf("84-flow", null, "85-flow-operators")
        .asFlow()
        .filterNotNull()
        .toList()

    println(slugs)
}
```

## `transform`

`transform` is more general than `map`: для одного input value можно emit zero, one or many output values.

```kotlin
import kotlinx.coroutines.flow.asFlow
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.flow.transform
import kotlinx.coroutines.runBlocking

fun transformExample() = runBlocking {
    val lines = listOf("84-flow", "85-flow-operators")
        .asFlow()
        .transform { slug ->
            emit("Начинаем $slug")
            emit("Готово $slug")
        }
        .toList()

    println(lines)
}
```

Use `transform`, когда:

- надо emit loading state before data;
- надо skip some values without separate `filter`;
- надо emit multiple domain events per input;
- надо call suspending function and conditionally emit.

```kotlin
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.asFlow
import kotlinx.coroutines.flow.transform

fun lessonCards(slugs: List<String>): Flow<LessonUiEvent> =
    slugs.asFlow().transform { slug ->
        emit(LessonUiEvent.Loading(slug))

        val card = loadCard(slug)

        emit(LessonUiEvent.Ready(card))
    }

sealed interface LessonUiEvent {
    data class Loading(
        val slug: String
    ) : LessonUiEvent

    data class Ready(
        val card: LessonCard
    ) : LessonUiEvent
}

suspend fun loadCard(slug: String): LessonCard {
    return LessonCard(slug, "Урок $slug")
}
```

## `take` и `drop`

`take(n)` берет первые `n` values and then cancels upstream collection.

```kotlin
import kotlinx.coroutines.flow.asFlow
import kotlinx.coroutines.flow.take
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

fun takeExample() = runBlocking {
    val firstTwo = (1..10)
        .asFlow()
        .take(2)
        .toList()

    println(firstTwo)
}
```

`drop(n)` skips first `n` values.

```kotlin
import kotlinx.coroutines.flow.asFlow
import kotlinx.coroutines.flow.drop
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

fun dropExample() = runBlocking {
    val withoutIntro = listOf("loading", "84-flow", "85-flow-operators")
        .asFlow()
        .drop(1)
        .toList()

    println(withoutIntro)
}
```

`take` часто применяют с hot streams, чтобы terminal operator мог завершиться.

```kotlin
import kotlinx.coroutines.flow.MutableSharedFlow
import kotlinx.coroutines.flow.take
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.launch
import kotlinx.coroutines.runBlocking

fun takeFromHotFlowExample() = runBlocking {
    val events = MutableSharedFlow<String>()

    val collector = launch {
        val firstTwo = events.take(2).toList()
        println(firstTwo)
    }

    events.emit("opened")
    events.emit("completed")

    collector.join()
}
```

## `debounce`

`debounce` waits for a quiet period and emits latest value.

Это идеально для search input.

```kotlin
import kotlinx.coroutines.FlowPreview
import kotlinx.coroutines.delay
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.debounce
import kotlinx.coroutines.flow.distinctUntilChanged
import kotlinx.coroutines.flow.flow
import kotlinx.coroutines.flow.mapLatest
import kotlin.time.Duration.Companion.milliseconds

@OptIn(FlowPreview::class)
fun searchResults(queryChanges: Flow<String>): Flow<List<String>> =
    queryChanges
        .debounce(300.milliseconds)
        .distinctUntilChanged()
        .mapLatest { query ->
            searchLessons(query)
        }

suspend fun searchLessons(query: String): List<String> {
    delay(100)
    return listOf("Результат для $query")
}

fun fakeQueryChanges(): Flow<String> = flow {
    emit("f")
    emit("fl")
    emit("flo")
    emit("flow")
}
```

Если пользователь печатает быстро, intermediate values suppressed.

## `sample`

`sample` emits latest value periodically.

Подходит для telemetry, progress bars and high-frequency updates, где не нужно каждое значение.

```kotlin
import kotlinx.coroutines.FlowPreview
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.sample
import kotlin.time.Duration.Companion.seconds

@OptIn(FlowPreview::class)
fun sampledProgress(progress: Flow<Int>): Flow<Int> {
    return progress.sample(1.seconds)
}
```

Разница:

- `debounce` waits for silence;
- `sample` emits latest value on schedule;
- `conflate` skips intermediate values when collector is slow;
- `collectLatest` cancels previous processing when new value arrives.

## `combine`

`combine` объединяет latest values from multiple flows. Когда любой source emits, transform получает latest values всех sources.

```kotlin
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.MutableStateFlow
import kotlinx.coroutines.flow.combine
import kotlinx.coroutines.runBlocking

fun combineExample() = runBlocking {
    val title = MutableStateFlow("Flow")
    val completed = MutableStateFlow(false)

    val uiState: Flow<LessonHeaderState> =
        combine(title, completed) { currentTitle, isCompleted ->
            LessonHeaderState(
                title = currentTitle,
                badge = if (isCompleted) "Завершено" else "В процессе"
            )
        }

    title.value = "Flow operators"
    completed.value = true
}

data class LessonHeaderState(
    val title: String,
    val badge: String
)
```

Use cases:

- UI state from multiple state holders;
- filters + query + sorting;
- authenticated user + permissions + data;
- settings + remote config.

## `zip`

`zip` pairs values by position: first with first, second with second.

```kotlin
import kotlinx.coroutines.flow.asFlow
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.flow.zip
import kotlinx.coroutines.runBlocking

fun zipExample() = runBlocking {
    val slugs = listOf("84-flow", "85-flow-operators").asFlow()
    val titles = listOf("Flow", "Flow operators").asFlow()

    val cards = slugs.zip(titles) { slug, title ->
        LessonCard(slug, title)
    }.toList()

    println(cards)
}
```

Use `zip`, когда streams logically synchronized by position. Для UI state чаще нужен `combine`, потому что state changes are latest-based.

<MouseTrap>
Не заменяй `combine` на `zip` в UI state. `zip` ждет парные значения и может «застрять», если один поток обновляется чаще другого.
</MouseTrap>

## `flatMapConcat`

`flatMapConcat` превращает each value into inner flow and collects inner flows sequentially.

```kotlin
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.asFlow
import kotlinx.coroutines.flow.flatMapConcat
import kotlinx.coroutines.flow.flow
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

fun flatMapConcatExample() = runBlocking {
    val events = listOf("84-flow", "85-flow-operators")
        .asFlow()
        .flatMapConcat { slug -> lessonEvents(slug) }
        .toList()

    println(events)
}

fun lessonEvents(slug: String): Flow<String> = flow {
    emit("start:$slug")
    emit("finish:$slug")
}
```

Sequential property matters when order must be preserved.

## `flatMapMerge`

`flatMapMerge` collects multiple inner flows concurrently.

```kotlin
import kotlinx.coroutines.ExperimentalCoroutinesApi
import kotlinx.coroutines.delay
import kotlinx.coroutines.flow.asFlow
import kotlinx.coroutines.flow.flatMapMerge
import kotlinx.coroutines.flow.flow
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

@OptIn(ExperimentalCoroutinesApi::class)
fun flatMapMergeExample() = runBlocking {
    val loaded = listOf("84-flow", "85-flow-operators", "86-stateflow-and-sharedflow")
        .asFlow()
        .flatMapMerge(concurrency = 2) { slug ->
            flow {
                delay(100)
                emit("loaded:$slug")
            }
        }
        .toList()

    println(loaded)
}
```

Use when independent operations can run concurrently and order of completion is acceptable.

## `flatMapLatest`

`flatMapLatest` cancels previous inner flow when new upstream value arrives.

```kotlin
import kotlinx.coroutines.ExperimentalCoroutinesApi
import kotlinx.coroutines.delay
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.flatMapLatest
import kotlinx.coroutines.flow.flow

@OptIn(ExperimentalCoroutinesApi::class)
fun liveSearch(queryChanges: Flow<String>): Flow<List<String>> =
    queryChanges.flatMapLatest { query ->
        flow {
            emit(searchLessons(query))
        }
    }
```

Use cases:

- search suggestions;
- selected tab changes;
- user id changes;
- route parameter changes;
- refresh trigger where only latest request matters.

## `catch`

`catch` handles upstream exceptions and can emit fallback.

```kotlin
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.catch
import kotlinx.coroutines.flow.flow

fun lessonStream(): Flow<String> =
    flow {
        emit("84-flow")
        error("Источник уроков недоступен")
    }.catch { exception ->
        emit("fallback:${exception.message}")
    }
```

`catch` does not catch downstream exceptions.

```kotlin
import kotlinx.coroutines.flow.catch
import kotlinx.coroutines.flow.flowOf
import kotlinx.coroutines.flow.map

fun catchBoundaryExample() =
    flowOf("84-flow")
        .catch {
            emit("fallback")
        }
        .map {
            error("Этот exception downstream от catch")
        }
```

## `retry` и `retryWhen`

`retry` restarts upstream flow on exception.

```kotlin
import kotlinx.coroutines.delay
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.flow
import kotlinx.coroutines.flow.retry

fun lessonFeedWithRetry(): Flow<String> =
    flow {
        emit(loadFeedPage())
    }.retry(retries = 3) { exception ->
        delay(100)
        exception is RuntimeException
    }

suspend fun loadFeedPage(): String {
    return "Страница ленты"
}
```

`retryWhen` gives attempt number.

```kotlin
import kotlinx.coroutines.delay
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.flow
import kotlinx.coroutines.flow.retryWhen

fun retryWhenExample(): Flow<String> =
    flow {
        emit(loadFeedPage())
    }.retryWhen { cause, attempt ->
        val canRetry = cause is RuntimeException && attempt < 3

        if (canRetry) {
            delay(100L * (attempt + 1))
        }

        canRetry
    }
```

Do not retry cancellation.

```kotlin
import kotlinx.coroutines.CancellationException
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.retry

fun safeRetry(source: Flow<String>): Flow<String> =
    source.retry(retries = 3) { exception ->
        exception !is CancellationException && exception is RuntimeException
    }
```

## `onEach`, `onStart`, `onCompletion`

`onEach` performs side effect for each value and passes value downstream.

```kotlin
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.onEach

fun logEachLesson(source: Flow<String>): Flow<String> =
    source.onEach { slug ->
        println("Получен урок $slug")
    }
```

`onStart` emits or performs side effect before upstream starts.

```kotlin
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.onStart

fun withLoading(source: Flow<LessonLoadState>): Flow<LessonLoadState> =
    source.onStart {
        emit(LessonLoadState.Loading)
    }

sealed interface LessonLoadState {
    data object Loading : LessonLoadState

    data class Ready(
        val title: String
    ) : LessonLoadState
}
```

`onCompletion` observes normal completion, exception or cancellation.

```kotlin
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.onCompletion

fun observeCompletion(source: Flow<String>): Flow<String> =
    source.onCompletion { cause ->
        if (cause == null) {
            println("Flow завершился успешно")
        } else {
            println("Flow завершился с cause: ${cause.message}")
        }
    }
```

## `collectLatest`

`collectLatest` cancels previous collector block when new value arrives.

```kotlin
import kotlinx.coroutines.delay
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.collectLatest

suspend fun renderSearchResults(results: Flow<List<String>>) {
    results.collectLatest { currentResults ->
        // Если придут новые results, текущий rendering будет отменен.
        delay(200)
        println("Рендерим ${currentResults.size} результатов")
    }
}
```

Use cases:

- rendering latest UI state;
- image preview generation;
- search result rendering;
- progress display where old rendering is irrelevant.

## Production pipeline: search UI

```kotlin
import kotlinx.coroutines.FlowPreview
import kotlinx.coroutines.ExperimentalCoroutinesApi
import kotlinx.coroutines.delay
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.catch
import kotlinx.coroutines.flow.debounce
import kotlinx.coroutines.flow.distinctUntilChanged
import kotlinx.coroutines.flow.filter
import kotlinx.coroutines.flow.flatMapLatest
import kotlinx.coroutines.flow.flow
import kotlinx.coroutines.flow.map
import kotlinx.coroutines.flow.onStart
import kotlin.time.Duration.Companion.milliseconds

class LessonSearchInteractor(
    private val search: LessonSearchService
) {
    @OptIn(FlowPreview::class, ExperimentalCoroutinesApi::class)
    fun states(queryChanges: Flow<String>): Flow<SearchState> =
        queryChanges
            .map { query -> query.trim() }
            .debounce(300.milliseconds)
            .distinctUntilChanged()
            .filter { query -> query.length >= 2 }
            .flatMapLatest { query ->
                flow {
                    emit(SearchState.Loading(query))

                    val results = search.search(query)

                    emit(SearchState.Ready(query, results))
                }.catch { exception ->
                    emit(SearchState.Failed(query, "Поиск временно недоступен: ${exception.message}"))
                }
            }
            .onStart {
                emit(SearchState.Idle)
            }
}

sealed interface SearchState {
    data object Idle : SearchState

    data class Loading(
        val query: String
    ) : SearchState

    data class Ready(
        val query: String,
        val results: List<String>
    ) : SearchState

    data class Failed(
        val query: String,
        val message: String
    ) : SearchState
}

class LessonSearchService {
    suspend fun search(query: String): List<String> {
        delay(100)
        return listOf("Урок по запросу $query")
    }
}
```

Этот pipeline:

- trims input;
- waits until typing settles;
- ignores duplicate query;
- rejects too-short query;
- cancels previous search when new query arrives;
- emits loading, ready and failed states explicitly;
- keeps cancellation behavior controlled by caller scope.

<CatTip>
Когда pipeline читается сверху вниз как история user intent, Flow работает особенно хорошо: событие вошло, лишний шум отфильтровался, актуальная работа запустилась, UI получил состояние.
</CatTip>
