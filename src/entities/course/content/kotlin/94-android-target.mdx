## Android target в KMP

Android target компилирует общий модуль так, чтобы его можно было использовать в Android application или Android library.

Типичная роль:

- общий business logic in `commonMain`;
- Адаптеры хранилища/сети/платформы для Android в `androidMain`;
- Android UI in app module or общий Compose module;
- интеграция с жизненным циклом, ресурсами и разрешениями Android;
- `Dispatchers.Main` through Android coroutine module.

```kotlin
plugins {
    kotlin("multiplatform") version "2.4.0"
    id("com.android.library") version "9.0.0"
}

kotlin {
    androidTarget()
}

android {
    namespace = "dev.kotlingo.shared"
    compileSdk = 36

    defaultConfig {
        minSdk = 24
    }
}
```

<CatTip>
Android target — это кошачий мостик между общий логикой и Android-миром: common code идет по центру, а `androidMain` держит Context, Parcelable, lifecycle и Jetpack.
</CatTip>

## `androidMain`

`androidMain` может использовать Android framework and AndroidX.

```text
src/
  commonMain/
    kotlin/
      progress/
        ProgressStore.kt
  androidMain/
    kotlin/
      progress/
        AndroidPreferencesProgressStore.kt
```

Common contract:

```kotlin
// commonMain/kotlin/progress/ProgressStore.kt
package progress

interface ProgressStore {
    suspend fun completedSlugs(): Set<String>

    suspend fun markCompleted(slug: String)
}
```

Android implementation:

```kotlin
// androidMain/kotlin/progress/AndroidPreferencesProgressStore.kt
package progress

import android.content.SharedPreferences
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.withContext

class AndroidPreferencesProgressStore(
    private val preferences: SharedPreferences
) : ProgressStore {
    override suspend fun completedSlugs(): Set<String> = withContext(Dispatchers.IO) {
        // SharedPreferences API синхронный, поэтому читаем вне Main thread.
        preferences.getStringSet("completed_slugs", emptySet()).orEmpty()
    }

    override suspend fun markCompleted(slug: String): Unit = withContext(Dispatchers.IO) {
        val current = preferences.getStringSet("completed_slugs", emptySet()).orEmpty()

        preferences.edit()
            .putStringSet("completed_slugs", current + slug)
            .apply()
    }
}
```

`commonMain` знает только `ProgressStore`, а Android details stay in `androidMain`.

<MouseTrap>
Не протаскивай `Context` в common code. Если common code просит Android Context, общий boundary уже течет. Передай interface, value object или конкретный adapter из Android layer.
</MouseTrap>

## Android Gradle Plugin

KMP Android target requires Android Gradle Plugin configuration.

```kotlin
plugins {
    kotlin("multiplatform") version "2.4.0"
    id("com.android.library") version "9.0.0"
}
```

Android block:

```kotlin
android {
    namespace = "dev.kotlingo.shared"
    compileSdk = 36

    defaultConfig {
        minSdk = 24
        consumerProguardFiles("consumer-rules.pro")
    }

    testOptions {
        unitTests.isIncludeAndroidResources = true
    }
}
```

Remember:

- one Android target per Gradle subproject;
- Android app module usually applies `com.android.application`;
- общий Android library module usually applies `com.android.library`;
- Зависимости, специфичные для Android, относятся к `androidMain`;
- Модульные тесты Android относятся к `androidUnitTest`;
- instrumented tests belong in Android test source sets depending on project настройка.

## Android dependencies

```kotlin
kotlin {
    sourceSets {
        commonMain.dependencies {
            implementation("org.jetbrains.kotlinx:kotlinx-coroutines-core:1.11.0")
        }

        androidMain.dependencies {
            implementation("androidx.core:core-ktx:1.19.0")
            implementation("androidx.lifecycle:lifecycle-runtime-ktx:2.11.0")
            implementation("org.jetbrains.kotlinx:kotlinx-coroutines-android:1.11.0")
        }
    }
}
```

Put AndroidX dependencies in `androidMain`, unless the library explicitly provides a common multiplatform артефакт.

Пример с регистратором Android:

```kotlin
// commonMain/kotlin/logging/Logger.kt
package logging

interface Logger {
    fun info(message: String)

    fun error(message: String, throwable: Throwable?)
}
```

```kotlin
// androidMain/kotlin/logging/AndroidLogger.kt
package logging

import android.util.Log

class AndroidLogger(
    private val tag: String
) : Logger {
    override fun info(message: String) {
        Log.i(tag, message)
    }

    override fun error(message: String, throwable: Throwable?) {
        Log.e(tag, message, throwable)
    }
}
```

## Main dispatcher

В Android `Dispatchers.Main` поддерживается основным потоком Android, когда доступен `kotlinx-coroutines-android`.

```kotlin
import kotlinx.coroutines.CoroutineScope
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.launch
import kotlinx.coroutines.withContext

class LessonScreenPresenter(
    private val scope: CoroutineScope,
    private val repository: AndroidLessonRepository
) {
    fun open(slug: String) {
        scope.launch(Dispatchers.Main) {
            renderLoading()

            val lesson = withContext(Dispatchers.IO) {
                repository.loadBlocking(slug)
            }

            renderLesson(lesson)
        }
    }

    private fun renderLoading() {
        println("Показываем загрузку")
    }

    private fun renderLesson(lesson: Lesson) {
        println("Показываем ${lesson.title}")
    }
}

data class Lesson(
    val slug: String,
    val title: String
)

class AndroidLessonRepository {
    fun loadBlocking(slug: String): Lesson {
        return Lesson(slug, "Урок $slug")
    }
}
```

Используйте области с учетом жизненного цикла в коде приложения:

```kotlin
// Android app module example
import androidx.lifecycle.ViewModel
import androidx.lifecycle.viewModelScope
import kotlinx.coroutines.launch

class LessonViewModel(
    private val useCase: OpenLessonUseCase
) : ViewModel() {
    fun open(slug: String) {
        viewModelScope.launch {
            val lesson = useCase.open(slug)
            println(lesson.title)
        }
    }
}
```

Общий KMP module can expose suspending сценарии использования; Android app decides lifecycle scope.

## Parcelize

`Parcelable` зависит от Android. Используйте `kotlin-parcelize` только в наборах исходного кода Android или модуле приложения Android.

```kotlin
plugins {
    id("kotlin-parcelize")
}
```

Android DTO:

```kotlin
// androidMain/kotlin/navigation/LessonArgs.kt
package navigation

import android.os.Parcelable
import kotlinx.parcelize.Parcelize

@Parcelize
data class LessonArgs(
    val slug: String,
    val source: String
) : Parcelable
```

Общая модель не должна реализовывать `Parcelable`.

```kotlin
// commonMain/kotlin/lesson/LessonId.kt
package lesson

@JvmInline
value class LessonId(
    val value: String
) {
    init {
        require(value.isNotBlank()) { "LessonId не должен быть пустым" }
    }
}
```

Сопоставление на границе Android:

```kotlin
// androidMain/kotlin/navigation/LessonArgsMapper.kt
package navigation

import lesson.LessonId

fun LessonArgs.toLessonId(): LessonId {
    return LessonId(slug)
}
```

<MouseTrap>
Не делай common domain model `Parcelable` только ради navigation. Android navigation arguments are adapter-layer concern, not domain contract.
</MouseTrap>

## Compose overview

Существует две распространенные архитектуры:

- native Android UI consumes общий Kotlin сценарии использования;
- Compose Multiplatform использует общий пользовательский интерфейс на Android, iOS, настольных компьютерах и в Интернете.

Native Android UI with общий domain:

```kotlin
// commonMain/kotlin/lesson/OpenLessonUseCase.kt
package lesson

class OpenLessonUseCase(
    private val source: LessonSource
) {
    suspend fun open(slug: String): LessonScreenState {
        val lesson = source.load(slug)

        return LessonScreenState.Ready(
            title = lesson.title,
            body = lesson.body
        )
    }
}

interface LessonSource {
    suspend fun load(slug: String): Lesson
}

data class Lesson(
    val title: String,
    val body: String
)

sealed interface LessonScreenState {
    data object Loading : LessonScreenState

    data class Ready(
        val title: String,
        val body: String
    ) : LessonScreenState
}
```

Android Compose использует его:

```kotlin
// Android app module
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import lesson.LessonScreenState

@Composable
fun LessonScreen(state: LessonScreenState) {
    when (state) {
        LessonScreenState.Loading -> Text("Загрузка")
        is LessonScreenState.Ready -> {
            Text(state.title)
            Text(state.body)
        }
    }
}
```

Compose Multiplatform общий UI can live in `commonMain`, but platform APIs still require platform source sets or expect/actual boundaries.

## Ресурсы и API платформы

Ресурсы Android специфичны для Android. Общий код не должен напрямую зависеть от `R.string`.

Bad boundary:

```kotlin
// commonMain
class BadErrorFormatter {
    fun message(): Int {
        // Нельзя возвращать Android resource id из common code.
        // return R.string.error_network
        return 0
    }
}
```

Good boundary:

```kotlin
// commonMain/kotlin/ui/TextMessage.kt
package ui

sealed interface TextMessage {
    data class Plain(
        val value: String
    ) : TextMessage

    data class ResourceKey(
        val key: String
    ) : TextMessage
}
```

Android maps resource keys:

```kotlin
// androidMain/kotlin/ui/AndroidTextResolver.kt
package ui

import android.content.Context

class AndroidTextResolver(
    private val context: Context
) {
    fun resolve(message: TextMessage): String {
        return when (message) {
            is TextMessage.Plain -> message.value
            is TextMessage.ResourceKey -> when (message.key) {
                "network_error" -> context.getString(android.R.string.dialog_alert_title)
                else -> message.key
            }
        }
    }
}
```

## Проверочный список для боевого кода

Для цели Android:

- Версии плагинов Android и Kotlin совместимы;
- `namespace`, `compileSdk`, `minSdk` are explicit;
- Зависимости только для Android находятся в `androidMain`;
- `Dispatchers.Main` поддерживается `kotlinx-coroutines-android`;
- общий код не зависит от `Context`, `Parcelable`, `R`, `Activity` или `ViewModel`;
- владение жизненным циклом остается на уровне Android;
- общий код exposes suspending APIs or observable state;
- Parcelize остается на уровне Android;
- Граница разделения композиций является преднамеренной.

<CatTip>
Хороший Android target делает Android-слой тонким, но не искусственно пустым: lifecycle, resources, permissions and UI остаются там, где platform действительно важна.
</CatTip>
