## Зачем Android-разработчику Kotlin

Kotlin — основной язык современной Android-разработки. На нем пишут экраны,
модели данных, ViewModel, работу с сетью, базой данных, корутинами и Jetpack
Compose.

На старте тебе не нужно знать все направления Kotlin. Достаточно понимать:
ты пишешь код в Android Studio или IntelliJ IDEA, проект собирается Gradle, а
Kotlin помогает делать код короче, безопаснее и понятнее, чем старый Java-код
с большим количеством шаблонных методов.

Главная польза Kotlin для новичка в Android:

- `null` видно прямо в типе, поэтому меньше падений из-за `NullPointerException`;
- `data class` быстро описывает данные экрана или ответа сервера;
- коллекции удобно фильтровать, сортировать и преобразовывать;
- корутины помогают писать асинхронный код без цепочек callback;
- Flow и StateFlow хорошо ложатся на состояние экрана;
- Compose выглядит естественно, потому что UI описывается обычными Kotlin-функциями.

## Что важно выучить сначала

У Kotlin много возможностей, но новичку нужен не справочник, а крепкая база.
Вот порядок, который пригодится в большинстве Android-задач.

1. Переменные и типы: `val`, `var`, `String`, `Int`, `Boolean`.
2. Условия и циклы: `if`, `when`, `for`, `while`.
3. Функции: параметры, возвращаемые значения, значения по умолчанию.
4. Null-safety: `String?`, `?.`, `?:`, `let`.
5. Классы и объекты: обычные классы, `data class`, `object`.
6. Коллекции: `List`, `Set`, `Map`, `filter`, `map`, `firstOrNull`.
7. Лямбды и scope-функции: `let`, `apply`, `run`, `also`.
8. Базовые корутины: `launch`, `async`, `suspend`, отмена.
9. Flow, StateFlow и SharedFlow для данных и событий.
10. Аннотации из готовых библиотек: Room, Hilt, Retrofit, kotlinx.serialization.

Это не все, что есть в Kotlin. Это минимум, который позволяет читать и писать
обычный Android-код без постоянного чувства, что язык говорит на чужом языке.

## Что пока можно не учить

Kotlin умеет работать не только на Android: есть Kotlin/JVM, Kotlin/JS,
Kotlin/Wasm, Kotlin/Native и Kotlin Multiplatform. Это полезные направления, но
новичку Android они не нужны в первый заход. Достаточно знать, что они существуют.

Также можно спокойно отложить:

- создание собственных DSL и аннотаций;
- KSP, kapt и внутренности compiler plugins;
- reflection и бинарную совместимость библиотек;
- перегрузку операторов, `tailrec`, value classes;
- сложные generics вроде variance, star projections и builder inference;
- низкоуровневые типы, битовые операции и тонкости Unicode.

<MouseTrap title="Ловушка: знать больше не всегда значит учиться быстрее">
  Если ты изучаешь Kotlin для Android, не пытайся пройти весь язык как академический
  справочник. Лучше уверенно написать пять простых экранов, чем идеально помнить
  редкую конструкцию, которую ты увидишь раз в год.
</MouseTrap>

## Маленький пример из Android-жизни

Представь экран курса: у него есть название, количество уроков и признак,
прошел ли пользователь курс. В Kotlin это удобно описать как `data class`.

```kotlin
data class CourseCard(
    val title: String,
    val lessonsCount: Int,
    val completed: Boolean
)

fun CourseCard.statusText(): String {
    return if (completed) {
        "Курс пройден"
    } else {
        "Осталось уроков: $lessonsCount"
    }
}

fun main() {
    val kotlinCourse = CourseCard(
        title = "Kotlin для Android",
        lessonsCount = 24,
        completed = false
    )

    println(kotlinCourse.title)
    println(kotlinCourse.statusText())
}
```

В этом примере уже есть несколько важных идей:

- `data class` хранит данные для UI;
- `val` защищает переменные от случайного переназначения;
- функция `statusText` превращает состояние в текст для экрана;
- extension-функция вызывается как метод: `kotlinCourse.statusText()`;
- строковый шаблон `$lessonsCount` подставляет значение в строку.

Такой код ближе к реальной Android-разработке, чем абстрактные `Foo` и `Bar`.
Ты почти каждый день будешь описывать карточки, строки списков, состояние кнопок,
результаты загрузки и сообщения об ошибках.

## Как выглядит Kotlin в Android-проекте

Обычный проект в Android Studio содержит несколько важных мест.

```text
app
├── build.gradle.kts
└── src
    └── main
        ├── AndroidManifest.xml
        └── java или kotlin
            └── com.example.app
                ├── MainActivity.kt
                ├── ui
                ├── data
                └── domain
```

`MainActivity.kt` запускает приложение и обычно вызывает `setContent`, если UI
написан на Compose. Папка `ui` содержит экраны и компоненты. В `data` часто
лежат репозитории, API-клиенты и работа с базой. В `domain` могут быть модели и
правила приложения, если проект уже вырос.

На старте не нужно строить идеальную архитектуру. Достаточно понимать, где
находится точка входа, где лежит экран и где хранится код, который приносит
данные для экрана.

## Kotlin помогает компилятору помогать тебе

Новички часто воспринимают компилятор как строгого проверяющего. Лучше думать о
нем как о напарнике, который ловит ошибки до запуска приложения.

```kotlin
fun greeting(name: String?): String {
    val visibleName = name ?: "гость"
    return "Привет, $visibleName"
}
```

Здесь `name` может быть `null`, потому что тип записан как `String?`. Kotlin не
позволит просто написать `name.length`, пока ты не проверишь значение. Это
сначала раздражает, а потом экономит часы поиска падений.

## Правила стиля с первого дня

Пиши так, чтобы код было легко читать через месяц.

- Используй `val` по умолчанию, а `var` только когда значение реально меняется.
- Давай переменным имена из предметной области: `userName`, `courseTitle`,
  `isLoading`, `items`.
- Не прячь важное состояние в случайных глобальных объектах.
- Делай функции короткими: одна функция должна отвечать за одну понятную вещь.
- Не используй сложную возможность языка только потому, что она существует.
- Если код стал хитрым, сначала попробуй упростить модель данных.
