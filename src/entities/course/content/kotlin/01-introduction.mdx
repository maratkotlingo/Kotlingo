## Роль Kotlin в Android

Kotlin — основной язык современной Android-разработки. На нем пишут UI, модели
данных, ViewModel, работу с сетью и базой данных, корутины, Flow и Jetpack
Compose.

Ключевые преимущества:

- null-safety: `String` и `String?` являются разными типами;
- меньше шаблонного кода: `data class`, top-level функции, значения по умолчанию;
- выразительные коллекции: `map`, `filter`, `groupBy`, `firstOrNull`;
- удобная асинхронность: `suspend`, coroutines, `Flow`, `StateFlow`;
- естественная модель UI в Compose: интерфейс описывается Kotlin-функциями.

## Минимальный набор тем

Для обычного Android-кода сначала нужны:

1. Переменные и типы: `val`, `var`, `String`, `Int`, `Boolean`.
2. Выражения и управление потоком: `if`, `when`, `for`, `while`.
3. Функции: параметры, возвращаемые значения, значения по умолчанию.
4. Null-safety: `?`, `?.`, `?:`, `!!`, `let`.
5. Классы: свойства, конструкторы, `data class`, `object`, `companion object`.
6. Коллекции: `List`, `Set`, `Map`, read-only и mutable варианты.
7. Лямбды и function types.
8. Scope-функции: `let`, `run`, `with`, `apply`, `also`.
9. Coroutines: `suspend`, `launch`, `async`, отмена, structured concurrency.
10. Flow API: `Flow`, `StateFlow`, `SharedFlow`, основные операторы.

## Что можно отложить

На старте Android-разработчику не обязательно глубоко изучать:

- Kotlin/JS, Kotlin/Wasm, Kotlin/Native и Multiplatform;
- собственные DSL, compiler plugins, KSP и kapt;
- reflection и бинарную совместимость библиотек;
- перегрузку операторов, `tailrec`, value classes;
- сложные generics: variance, star projections, builder inference;
- низкоуровневые числовые типы, битовые операции и детали Unicode.

## Базовый пример

```kotlin
data class CourseCard(
    val title: String,
    val lessonsCount: Int,
    val completed: Boolean
)

fun CourseCard.statusText(): String {
    return if (completed) {
        "Курс пройден"
    } else {
        "Осталось уроков: $lessonsCount"
    }
}

fun main() {
    val course = CourseCard(
        title = "Kotlin для Android",
        lessonsCount = 24,
        completed = false
    )

    println(course.title)
    println(course.statusText())
}
```

В примере используются:

- `data class` для состояния экрана;
- `val` для неизменяемых ссылок;
- `Boolean` для логического состояния;
- extension-функция `CourseCard.statusText`;
- строковый шаблон `$lessonsCount`.

## Где Kotlin лежит в Android-проекте

Типичная структура модуля:

```text
app
├── build.gradle.kts
└── src
    └── main
        ├── AndroidManifest.xml
        └── java или kotlin
            └── com.example.app
                ├── MainActivity.kt
                ├── ui
                ├── data
                └── domain
```

- `MainActivity.kt` — точка входа Android-экрана.
- `ui` — экраны, composable-функции, UI-состояние.
- `data` — API-клиенты, репозитории, база данных, DTO.
- `domain` — бизнес-модели и правила, если проект требует отдельного слоя.

## Компилятор как часть дизайна

Kotlin переносит часть ошибок из runtime в compile time.

```kotlin
fun greeting(name: String?): String {
    val visibleName = name ?: "гость"
    return "Привет, $visibleName"
}
```

`name.length` здесь запрещен без проверки: тип `String?` допускает `null`.
Нужно использовать проверку, safe call `?.`, Elvis `?:` или smart cast.

## Базовые правила стиля

- Начинай с `val`; используй `var`, только если значение действительно меняется.
- Называй значения по предметной области: `userName`, `courseTitle`, `isLoading`.
- Выражай отсутствие данных типом: `String?`, `Result`, sealed-состояние.
- Держи функции короткими и с одной ответственностью.
- Не добавляй сложную конструкцию языка без практической причины.
- Для внешнего API предпочитай явные типы и понятные имена.
