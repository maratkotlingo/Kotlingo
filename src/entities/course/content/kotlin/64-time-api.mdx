## Time API решает две разные задачи

В Kotlin стандартная Time API в пакете `kotlin.time` в первую очередь помогает:

- описывать длительности через `Duration`;
- измерять elapsed time через monotonic time source.

```kotlin
import kotlin.time.Duration.Companion.minutes

val lessonDuration = 45.minutes

println(lessonDuration) // 45m
```

Это не полноценная calendar/date-time библиотека. Для дат, часовых поясов и календарной арифметики обычно используют `java.time` на JVM или `kotlinx-datetime` в multiplatform-проектах.

<CatTip>
`Duration` отвечает «сколько времени длится», а не «какая сейчас дата». Это как разница между «кот спал 20 минут» и «кот проснулся 9 июля».
</CatTip>

## `Duration`

`Duration` представляет промежуток времени.

```kotlin
import kotlin.time.Duration
import kotlin.time.Duration.Companion.seconds

val timeout: Duration = 30.seconds

println(timeout.inWholeSeconds) // 30
```

`Duration` поддерживает arithmetic.

```kotlin
import kotlin.time.Duration.Companion.minutes
import kotlin.time.Duration.Companion.seconds

val reading = 20.minutes
val quiz = 90.seconds
val total = reading + quiz

println(total) // 21m 30s
```

Можно вычитать, умножать и делить.

```kotlin
val half = total / 2
val doubled = quiz * 2

println(half)
println(doubled)
```

## Duration units

Duration companion extensions создают durations из чисел.

```kotlin
import kotlin.time.Duration.Companion.days
import kotlin.time.Duration.Companion.hours
import kotlin.time.Duration.Companion.microseconds
import kotlin.time.Duration.Companion.milliseconds
import kotlin.time.Duration.Companion.minutes
import kotlin.time.Duration.Companion.nanoseconds
import kotlin.time.Duration.Companion.seconds

val warmup = 500.milliseconds
val lesson = 45.minutes
val day = 1.days
val tiny = 250.nanoseconds
```

Доступны extensions для `Int`, `Long`, `Double`.

```kotlin
val precise = 1.5.seconds

println(precise.inWholeMilliseconds) // 1500
```

`inWhole...` возвращает целое число с отсечением дробной части.

```kotlin
val duration = 90.seconds

println(duration.inWholeMinutes) // 1
```

Если нужна дробная конверсия, используй `toDouble(DurationUnit.X)`.

```kotlin
import kotlin.time.DurationUnit

println(duration.toDouble(DurationUnit.MINUTES)) // 1.5
```

## Comparison

Durations можно сравнивать.

```kotlin
import kotlin.time.Duration.Companion.minutes

val shortLesson = 15.minutes
val longLesson = 60.minutes

println(shortLesson < longLesson) // true
```

Equality учитывает фактическую длительность, а не исходную unit.

```kotlin
import kotlin.time.Duration.Companion.hours

println(30.minutes == 0.5.hours) // true
```

## Special values

`Duration` поддерживает zero, infinite и negative durations.

```kotlin
import kotlin.time.Duration

val none = Duration.ZERO
val never = Duration.INFINITE

println(none.isPositive()) // false
println(never.isInfinite()) // true
```

Negative duration может появиться при subtraction.

```kotlin
val remaining = 5.seconds - 10.seconds

println(remaining.isNegative()) // true
```

В бизнес-логике часто нужно normalize.

```kotlin
fun remainingOrZero(total: Duration, elapsed: Duration): Duration {
    return (total - elapsed).coerceAtLeast(Duration.ZERO)
}
```

## `toComponents`

`toComponents` разбирает duration на parts.

```kotlin
import kotlin.time.Duration.Companion.minutes

val formatted = 125.minutes.toComponents { hours, minutes, _, _ ->
    // Неиспользуемые seconds и nanoseconds пропускаем через underscore.
    "${hours}ч ${minutes}м"
}

println(formatted) // 2ч 5м
```

Это лучше ручного деления, когда нужно форматирование для UI.

## `measureTime`

`measureTime` измеряет время выполнения блока и возвращает `Duration`.

```kotlin
import kotlin.time.measureTime

val elapsed = measureTime {
    // Здесь может быть вычисление, запрос или синхронная операция.
    Thread.sleep(100)
}

println(elapsed)
```

По умолчанию используется monotonic time source. Такой источник подходит для измерений elapsed time, потому что не зависит от часов, timezone и перевода системного времени.

## `measureTimedValue`

`measureTimedValue` возвращает и value, и elapsed duration.

```kotlin
import kotlin.time.measureTimedValue

val result = measureTimedValue {
    Thread.sleep(50)
    listOf("intro", "types", "collections")
}

println(result.value)
println(result.duration)
```

Можно destructuring.

```kotlin
val (lessons, loadingTime) = measureTimedValue {
    // Возвращаем значение, которое одновременно измеряем.
    listOf("01-introduction", "02-types")
}

println("Загружено ${lessons.size} за $loadingTime")
```

## `TimeSource` и `TimeMark`

`TimeSource` создает marks. `TimeMark` представляет момент в рамках конкретного source.

```kotlin
import kotlin.time.TimeSource

val source = TimeSource.Monotonic
val started = source.markNow()

Thread.sleep(10)

val elapsed = started.elapsedNow()

println(elapsed)
```

Можно создать deadline.

```kotlin
import kotlin.time.Duration.Companion.seconds

val deadline = TimeSource.Monotonic.markNow() + 5.seconds

if (deadline.hasNotPassedNow()) {
    println("Еще можно продолжать")
}
```

Сравнивать и вычитать marks безопасно только в рамках одного time source.

## Timeout helper

```kotlin
import kotlin.time.Duration
import kotlin.time.TimeSource

class TimeoutBudget(
    total: Duration,
    private val source: TimeSource = TimeSource.Monotonic
) {
    private val deadline = source.markNow() + total

    fun hasTimeLeft(): Boolean {
        return deadline.hasNotPassedNow()
    }

    fun remaining(): Duration {
        // Отрицательные значения срезаем до нуля для удобства caller-а.
        return deadline
            .minus(source.markNow())
            .coerceAtLeast(Duration.ZERO)
    }
}
```

Этот класс хранит timeout как deadline, а не пересчитывает elapsed вручную.

<MouseTrap>
Не измеряй elapsed time через wall-clock timestamps вроде `System.currentTimeMillis()`. Системные часы могут измениться, и измерение станет неверным. Для длительности используй monotonic time source.
</MouseTrap>

## Interop with Java time

На JVM часто нужно перейти между `kotlin.time.Duration` и `java.time.Duration`.

```kotlin
import kotlin.time.Duration.Companion.seconds
import kotlin.time.toJavaDuration
import kotlin.time.toKotlinDuration

val kotlinTimeout = 30.seconds
val javaTimeout: java.time.Duration = kotlinTimeout.toJavaDuration()

val backToKotlin = javaTimeout.toKotlinDuration()
```

Для calendar dates используй `java.time.Instant`, `LocalDate`, `ZonedDateTime`, `Clock`.

```kotlin
import java.time.Clock
import java.time.Instant

class AuditClock(
    private val clock: Clock
) {
    fun now(): Instant {
        // Instant — момент на глобальной временной шкале, а не Duration.
        return Instant.now(clock)
    }
}
```

`Duration` и `Instant` отвечают на разные вопросы.

## Production-пример: измерение загрузки уроков

```kotlin
import kotlin.time.Duration
import kotlin.time.Duration.Companion.seconds
import kotlin.time.measureTimedValue

data class Lesson(
    val slug: String,
    val title: String
)

data class LoadReport(
    val lessons: List<Lesson>,
    val elapsed: Duration,
    val slow: Boolean
)

interface LessonSource {
    fun loadAll(): List<Lesson>
}

fun loadWithReport(source: LessonSource): LoadReport {
    val measured = measureTimedValue {
        source.loadAll()
    }

    return LoadReport(
        lessons = measured.value,
        elapsed = measured.duration,
        slow = measured.duration > 2.seconds
    )
}
```

Здесь `Duration` остается typed value до самого UI, а не превращается преждевременно в `Long`.

## Практические правила

Используй `Duration`, если говоришь о длительности, timeout, delay, TTL, latency.

```kotlin
val cacheTtl = 15.minutes
```

Используй `measureTime` и `measureTimedValue`, если нужно измерить блок.

```kotlin
val time = measureTime { rebuildIndex() }
```

Используй `TimeSource.Monotonic`, если строишь deadline или сравниваешь elapsed time.

```kotlin
val mark = TimeSource.Monotonic.markNow()
```

Используй `java.time` или `kotlinx-datetime`, если задача про календарные даты, timezone или timestamps.

<CatTip>
Самая частая ошибка со временем — смешать длительность и дату. Держи `Duration` для «сколько», а date-time API для «когда».
</CatTip>
