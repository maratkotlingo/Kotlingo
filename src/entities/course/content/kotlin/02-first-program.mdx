## Минимальная программа

```kotlin
fun main() {
    println("Привет, kotlingo!")
}
```

- `fun` объявляет функцию.
- `main` — стандартная точка входа.
- `println` печатает значение и перевод строки.
- Точка с запятой в конце строки обычно не нужна.
- Выполняемые инструкции должны находиться внутри функции, `init`-блока,
  getter/setter или другого исполняемого контекста.

Некорректно:

```kotlin
println("Привет")
```

Корректно:

```kotlin
fun main() {
    println("Привет")
}
```

## Порядок элементов в `.kt`-файле

```kotlin
@file:JvmName("KotlingoApp")

package academy.kotlingo.basics

import kotlin.math.max

const val COURSE_NAME = "kotlingo"

data class LessonSummary(
    val title: String,
    val minutes: Int
)

fun formatSummary(summary: LessonSummary): String {
    val safeMinutes = max(summary.minutes, 0)
    return "${summary.title}: $safeMinutes мин."
}

fun main() {
    val summary = LessonSummary("Первая программа", 45)
    println(formatSummary(summary))
    println("Курс: $COURSE_NAME")
}
```

Порядок:

1. File annotations: `@file:JvmName`, `@file:Suppress`, `@file:OptIn`.
2. `package`.
3. `import`.
4. Top-level объявления: свойства, функции, классы, интерфейсы, объекты.

## File annotations

File annotation применяется ко всему файлу и пишется до `package`.

```kotlin
@file:JvmName("CourseEntryPoint")
@file:Suppress("MagicNumber")

package academy.kotlingo.entry
```

Частые варианты:

- `@file:JvmName("Name")` меняет JVM-имя класса для top-level объявлений;
- `@file:JvmMultifileClass` объединяет несколько файлов под JVM facade;
- `@file:Suppress("DiagnosticName")` подавляет диагностику для всего файла;
- `@file:OptIn(Api::class)` включает experimental API для всего файла.

## `package`

`package` задает пространство имен.

```kotlin
package academy.kotlingo.lessons

data class LessonId(val value: String)
```

Полное имя класса: `academy.kotlingo.lessons.LessonId`.

Файл может физически лежать в любом каталоге, но в проектах обычно синхронизируют
путь и пакет:

```text
src/main/kotlin/academy/kotlingo/lessons/LessonId.kt
```

Пакет по умолчанию допустим для маленьких примеров, но нежелателен в приложении:
его труднее импортировать, группировать и сопровождать.

## `import`

`import` позволяет обращаться к имени без полного пути.

```kotlin
package academy.kotlingo.imports

import kotlin.math.PI
import kotlin.math.roundToInt

fun main() {
    val radius = 4.0
    val area = PI * radius * radius
    println(area.roundToInt())
}
```

Варианты:

```kotlin
import kotlin.text.Regex as KotlinRegex
import kotlin.math.*
```

- Alias полезен при конфликте имен.
- Star import подключает все имена из пакета, но в production-коде явные импорты
  обычно читаются лучше.
- Многие имена доступны без импорта через default imports: `kotlin.*`,
  `kotlin.collections.*`, `kotlin.io.*`, `kotlin.ranges.*`, `kotlin.text.*`.

## Top-level объявления

Kotlin разрешает объявлять функции, свойства и типы на уровне файла.

```kotlin
package academy.kotlingo.top.level

const val MAX_VISIBLE_LESSONS = 10

data class Lesson(
    val title: String,
    val visible: Boolean
)

fun visibleTitle(lesson: Lesson): String {
    return if (lesson.visible) lesson.title else "скрытый урок"
}
```

Top-level подходит, когда:

- функция не хранит состояние;
- константа относится к модулю;
- нужна фабричная или extension-функция;
- объявление должно импортироваться отдельно.

Класс или объект лучше выбрать, когда нужно состояние, lifecycle, интерфейс,
заменяемая реализация или явная зависимость.

## Варианты `main`

### Без аргументов

```kotlin
fun main() {
    println("Запуск")
}
```

Основной вариант для учебных программ и простых CLI.

### С аргументами командной строки

```kotlin
fun main(args: Array<String>) {
    val command = args.getOrNull(0) ?: "help"
    println("Команда: $command")
}
```

`args[0]` бросит исключение, если аргументов нет. Для optional-аргументов
используй `getOrNull`.

### Expression body

```kotlin
fun main() = println("Запуск")
```

Уместно для коротких примеров. Для валидации и нескольких шагов лучше block body.

### `suspend main`

```kotlin
suspend fun main() {
    val message = loadGreeting()
    println(message)
}

suspend fun loadGreeting(): String = "Привет"
```

Нужен, когда точка входа должна вызывать suspend API. `suspend` не создает поток;
он разрешает приостановку и продолжение выполнения.

## Консольный ввод и вывод

| Функция | Назначение |
| --- | --- |
| `println(value)` | печатает значение и перевод строки |
| `print(value)` | печатает без перевода строки |
| `readln()` | читает строку или бросает исключение при отсутствии ввода |
| `readlnOrNull()` | читает строку или возвращает `null` |
| `toInt()` | парсит `Int` или бросает `NumberFormatException` |
| `toIntOrNull()` | парсит `Int` или возвращает `null` |

Безопасный ввод числа:

```kotlin
fun main() {
    print("Номер урока: ")

    val lessonNumber = readlnOrNull()
        ?.trim()
        ?.toIntOrNull()

    if (lessonNumber == null) {
        println("Нужен целый номер урока")
        return
    }

    println("Открываем урок $lessonNumber")
}
```

## CLI-программа с аргументами

```kotlin
fun printHelp() {
    println("open <название>      открыть урок")
    println("complete <название>  отметить урок пройденным")
    println("help                 показать справку")
}

fun normalizeTitle(rawTitle: String): String {
    return rawTitle
        .trim()
        .replace(Regex("\\s+"), " ")
}

fun main(args: Array<String>) {
    val command = args.getOrNull(0) ?: "help"
    val title = normalizeTitle(args.drop(1).joinToString(" "))

    when (command) {
        "open" -> println("Открываем урок: $title")
        "complete" -> println("Урок пройден: $title")
        "help" -> printHelp()
        else -> {
            println("Неизвестная команда: $command")
            printHelp()
        }
    }
}
```

Важные приемы:

- `args.getOrNull(0)` безопасно читает первый аргумент;
- `args.drop(1)` берет аргументы после команды;
- `joinToString(" ")` собирает название из нескольких слов;
- `when` явно описывает все команды и fallback.

## Запуск через Gradle

Минимальная структура:

```text
settings.gradle.kts
build.gradle.kts
src/main/kotlin/Main.kt
```

`build.gradle.kts`:

```kotlin
plugins {
    kotlin("jvm") version "<версия Kotlin в проекте>"
    application
}

application {
    mainClass.set("MainKt")
}
```

Команды:

```text
./gradlew run
./gradlew build
./gradlew test
```

На Windows:

```text
gradlew.bat run
gradlew.bat build
gradlew.bat test
```

Если `Main.kt` находится в пакете `academy.kotlingo`, main class будет
`academy.kotlingo.MainKt`.

## Запуск через Kotlin CLI

`Main.kt`:

```kotlin
fun main() {
    println("Запуск через Kotlin CLI")
}
```

Компиляция в jar:

```text
kotlinc Main.kt -include-runtime -d app.jar
```

Запуск:

```text
java -jar app.jar
```

Без `-include-runtime` jar не содержит Kotlin runtime и требует корректного
classpath.

## Несколько файлов в одном package

`Lesson.kt`:

```kotlin
package academy.kotlingo.multi

data class Lesson(val title: String, val minutes: Int)
```

`Formatting.kt`:

```kotlin
package academy.kotlingo.multi

fun formatLesson(lesson: Lesson): String {
    return "${lesson.title}: ${lesson.minutes} мин."
}
```

`Main.kt`:

```kotlin
package academy.kotlingo.multi

fun main() {
    println(formatLesson(Lesson("Несколько файлов", 30)))
}
```

Объявления из одного package доступны друг другу без import.

## Частые ошибки

Неверная сигнатура точки входа:

```kotlin
fun main(args: List<String>) {
    println(args)
}
```

Правильно:

```kotlin
fun main(args: Array<String>) {
    println(args.joinToString())
}
```

Небезопасный доступ к аргументу:

```kotlin
val command = args[0]
```

Безопаснее:

```kotlin
val command = args.getOrNull(0) ?: "help"
```

Класс только ради `main`:

```kotlin
class Application {
    fun run() = println("Запуск")
}

fun main() {
    Application().run()
}
```

Для простой программы достаточно:

```kotlin
fun main() {
    println("Запуск")
}
```
