## Токены

Токен — минимальная значимая единица исходного кода: ключевое слово, имя,
литерал, оператор, разделитель, комментарий или пробел.

```kotlin
val lessonTitle = "Лексика языка"
```

Токены:

- `val` — ключевое слово;
- `lessonTitle` — идентификатор;
- `=` — оператор присваивания;
- `"Лексика языка"` — строковый литерал.

Пробелы обычно только разделяют токены. Комментарии сохраняются в исходном
файле, но не становятся исполняемым кодом.

## Базовая структура файла

```kotlin
@file:Suppress("MagicNumber")

package academy.kotlingo.lexis

import kotlin.math.PI

const val DEFAULT_RADIUS = 4.0

fun circleArea(radius: Double): Double {
    return PI * radius * radius
}

fun main() {
    val area = circleArea(DEFAULT_RADIUS)
    println("Площадь: $area")
}
```

Порядок элементов:

1. File annotations.
2. `package`.
3. `import`.
4. Top-level объявления.

## Идентификаторы

Идентификатор — имя переменной, функции, класса, интерфейса, объекта,
параметра, пакета, type alias, generic-параметра или label.

Обычный идентификатор:

- может содержать буквы, цифры и `_`;
- не может начинаться с цифры;
- не может совпадать с hard keyword;
- должен быть понятным для читателя, а не только допустимым для компилятора.

```kotlin
val lesson1 = "Введение"
val maxRetryCount = 3

fun normalizeTitle(title: String): String = title.trim()

class LessonRepository
interface ProgressStore
```

Некорректно:

```kotlin
val 1lesson = "Ошибка"      // начинается с цифры
val class = "Ошибка"        // hard keyword
val lesson-title = "Ошибка" // дефис читается как минус
```

## Соглашения об именах

| Сущность | Стиль |
| --- | --- |
| локальные переменные, функции, свойства | `camelCase` |
| классы, интерфейсы, enum, object, type alias | `PascalCase` |
| compile-time константы | `UPPER_SNAKE_CASE` |
| короткие generic-параметры | `T`, `R`, `E` |
| доменные generic-параметры | `Entity`, `Command`, `Result` |

```kotlin
const val MAX_VISIBLE_LESSONS = 10

data class LessonCard(
    val title: String,
    val estimatedMinutes: Int
)

fun formatLessonTitle(rawTitle: String): String {
    return rawTitle.trim()
}
```

Кириллица допустима в идентификаторах, но в production API обычно мешает
поиску, Java interop, shell-командам, автогенерации и командным соглашениям.

## Идентификаторы в обратных кавычках

Обратные кавычки позволяют использовать имя с пробелами, keyword или символами,
которые обычный идентификатор не принимает.

```kotlin
fun `открыть урок`() {
    println("Урок открыт")
}

fun main() {
    `открыть урок`()
}
```

Основные случаи:

- читаемые имена тестов;
- вызов Java API, где метод называется как Kotlin keyword;
- временное сохранение внешнего имени, которое нельзя изменить.

```kotlin
class LessonTitleTest {
    fun `trim удаляет пробелы по краям названия`() {
        val rawTitle = "  Лексика языка  "
        kotlin.test.assertEquals("Лексика языка", rawTitle.trim())
    }
}
```

Для обычного бизнес-кода лучше использовать нормальные API-имена:

```kotlin
fun createUserWhenEmailIsValidAndPlanIsActive() {
    println("Понятное, хотя и длинное имя")
}
```

## Hard keywords

Hard keyword нельзя использовать как обычный идентификатор без обратных кавычек.

```text
as, break, class, continue, do, else, false, for, fun, if, in,
interface, is, null, object, package, return, super, this, throw,
true, try, typealias, typeof, val, var, when, while
```

```kotlin
fun main() {
    val score = 90

    if (score in 80..100) {
        println(true)
    } else {
        println(false)
    }
}
```

`fun`, `val`, `if`, `in`, `else`, `true`, `false` здесь являются частью
синтаксиса, а не именами.

## Soft keywords

Soft keyword имеет специальный смысл только в определенном контексте. В других
местах технически может быть именем, но часто ухудшает читаемость.

Частые soft keywords:

```text
by, catch, constructor, delegate, dynamic, field, file, finally,
get, import, init, param, property, receiver, set, setparam, where,
actual, abstract, annotation, companion, const, crossinline, data,
enum, expect, external, final, infix, inline, inner, internal,
lateinit, noinline, open, operator, out, override, private,
protected, public, reified, sealed, suspend, tailrec, vararg, value,
context
```

```kotlin
class Lesson(
    val title: String
) {
    val description: String
        get() = "Урок: $title"
}
```

`get` имеет специальный смысл в объявлении getter. Как имя переменной оно
допустимо, но плохо читается:

```kotlin
val get = "допустимо, но не стоит"
```

## Модификаторы

Многие soft keywords используются как модификаторы объявления.

```kotlin
public data class Lesson(
    private val id: Long,
    val title: String
)
```

Частые группы:

- видимость: `public`, `private`, `protected`, `internal`;
- наследование: `open`, `final`, `abstract`, `sealed`;
- переопределение: `override`;
- специальные классы: `data`, `enum`, `annotation`, `value`;
- функции: `inline`, `infix`, `operator`, `tailrec`, `suspend`;
- generics: `in`, `out`, `reified`, `where`.

## Литералы

```kotlin
val intValue = 42
val longValue = 42L
val doubleValue = 4.2
val floatValue = 4.2f
val charValue = 'K'
val stringValue = "Kotlin"
val rawString = """
    line 1
    line 2
""".trimIndent()
val booleanValue = true
val nothingHere = null
```

Полезные детали:

- `_` можно использовать в числах для читаемости: `1_000_000`;
- `L` указывает `Long`;
- `f` или `F` указывает `Float`;
- одинарные кавычки используются для `Char`, двойные — для `String`;
- `null` допустим только для nullable-типов.

## Комментарии

Line comment:

```kotlin
// Нормализуем ввод перед проверкой.
val title = rawTitle.trim()
```

Block comment:

```kotlin
/*
 * Используется вложенный формат, потому что во фрагменте уже есть комментарии.
 */
val title = "Лексика"
```

Kotlin поддерживает вложенные block comments:

```kotlin
/*
 * Внешний комментарий.
 *
 * /* Вложенный комментарий. */
 */
```

Комментарий должен объяснять намерение, ограничение или неочевидное решение.
Комментарий, который пересказывает синтаксис, обычно лишний.

## KDoc

KDoc — документационный комментарий Kotlin. Его используют IDE и генераторы
документации.

```kotlin
/**
 * Описывает учебный урок.
 *
 * @property title название урока.
 * @property estimatedMinutes примерная длительность в минутах.
 */
data class Lesson(
    val title: String,
    val estimatedMinutes: Int
)

/**
 * Возвращает строку формата "<название>: <минуты> мин.".
 *
 * @param lesson урок для отображения.
 * @return человекочитаемая подпись.
 */
fun formatLesson(lesson: Lesson): String {
    return "${lesson.title}: ${lesson.estimatedMinutes} мин."
}
```

Частые теги:

- `@param`;
- `@return`;
- `@property`;
- `@receiver`;
- `@throws` или `@exception`;
- `@sample`;
- `@see`;
- `@since`;
- `@suppress`.

KDoc должен описывать контракт: формат, ограничения, nullable-поведение,
исключения и инварианты. Не стоит писать KDoc, который только повторяет имя.

## File annotations

```kotlin
@file:JvmName("LessonFormatting")
@file:Suppress("MagicNumber")

package academy.kotlingo.lexis
```

File annotation действует на весь файл. Используй ее точечно: например, file
level `@OptIn` распространяется на все объявления файла и может скрыть место,
где experimental API действительно нужен.

## Точки с запятой

Точка с запятой допустима, но почти всегда лишняя.

```kotlin
val title = "Лексика"
println(title)
```

Не принято:

```kotlin
val title = "Лексика";
println(title);
```

Точка с запятой нужна только для нескольких инструкций в одной строке:

```kotlin
val title = "Лексика"; println(title)
```

Такой стиль обычно хуже читается, чем две строки.

## Переносы строк

Безопасные места для переноса:

```kotlin
val title = listOf(
    "Лексика",
    "Типы",
    "Числа"
).joinToString(separator = " -> ")
```

Цепочки вызовов часто переносят перед точкой:

```kotlin
fun normalizeTitle(rawTitle: String): String {
    return rawTitle
        .trim()
        .lowercase()
        .replace(Regex("\\s+"), "-")
}
```

Опасный перенос:

```kotlin
fun score(): Int {
    return
        100
}
```

Пиши так:

```kotlin
fun score(): Int {
    return 100
}
```

или так:

```kotlin
fun score(): Int =
    100
```

## Labels

Label — имя с `@`, которым помечают выражение. Labels используются с `return`,
`break`, `continue` и лямбдами.

```kotlin
fun main() {
    val titles = listOf("", "Лексика", "Типы")

    titles.forEach lessonLoop@{ title ->
        if (title.isBlank()) {
            return@lessonLoop
        }

        println(title)
    }
}
```

`return@lessonLoop` выходит из текущей лямбды, а не из `main`.

## Контрольный пример

```kotlin
@file:Suppress("MagicNumber")

package academy.kotlingo.lexis

import kotlin.math.roundToInt

data class LessonProgress(
    val title: String,
    val rawCompletionPercent: Double
) {
    val completionPercent: Int
        get() = rawCompletionPercent
            .roundToInt()
            .coerceIn(0, 100)
}

fun formatProgress(progress: LessonProgress): String {
    val title = progress.title.trim()

    require(title.isNotBlank()) {
        "Название урока не должно быть пустым"
    }

    return "$title: ${progress.completionPercent}%"
}

fun main() {
    val progressItems = listOf(
        LessonProgress(" Лексика языка ", 37.8),
        LessonProgress("Базовые типы", 104.2)
    )

    progressItems.forEach { progress ->
        println(formatProgress(progress))
    }
}
```

Файл лексически аккуратен: file annotation стоит до `package`, imports явные,
имена следуют соглашениям, цепочки вызовов перенесены по шагам, комментарии не
дублируют синтаксис.
