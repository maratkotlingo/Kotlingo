## Function literal с ключевым словом `fun`

Anonymous function — function без имени, объявленная как expression.

```kotlin
val normalize: (String) -> String = fun(raw: String): String {
    return raw.trim().replace(Regex("""\s+"""), " ")
}
```

Вызов обычный:

```kotlin
println(normalize("  Anonymous   functions  "))
```

Она похожа на named function, но хранится/передаётся как value.

## Block body

```kotlin
val grade: (Int) -> String = fun(score: Int): String {
    require(score in 0..100) { "Балл должен быть от 0 до 100" }

    if (score >= 90) return "Отлично"
    if (score >= 60) return "Зачёт"
    return "Нужно повторить"
}
```

У block body с useful result return type указывается явно, как у named function. Все normal paths должны вернуть value.

## Expression body

```kotlin
val double: (Int) -> Int = fun(value: Int): Int = value * 2
```

Return type можно вывести:

```kotlin
val length: (String) -> Int = fun(value: String) = value.length
```

Expression body подходит одному короткому вычислению.

## Вывод types из expected context

```kotlin
val predicate: (String) -> Boolean = fun(value) = value.isNotBlank()
```

Parameter type выводится из expected function type. Без context его нужно указать.

## Главное отличие return

Bare `return` в anonymous function всегда возвращает из anonymous function, а не из enclosing named function.

```kotlin
fun firstPassing(scores: List<Int>): Int? {
    var result: Int? = null

    scores.forEach(fun(score) {
        if (score !in 0..100) return
        if (result == null && score >= 60) {
            result = score
        }
    })

    return result
}
```

`return` завершает текущий callback invocation. `firstPassing` продолжает работу.

Эквивалентная lambda требует label:

```kotlin
scores.forEach { score ->
    if (score !in 0..100) return@forEach
    println(score)
}
```

<CatTip title="Котосовет: anonymous fun удобна при нескольких ранних local returns">
Если block напоминает обычную функцию с guard clauses, `fun(...) { return ... }` часто читается лучше нескольких `return@label`.
</CatTip>

## Anonymous function предотвращает non-local return

Даже если higher-order function inline, bare return anonymous function остаётся local.

```kotlin
fun printUntilInvalid(scores: List<Int>) {
    scores.forEach(fun(score) {
        if (score !in 0..100) return
        println(score)
    })

    println("Внешняя функция продолжается")
}
```

Это делает control flow явнее, если non-local exit нежелателен.

## Нет implicit `it`

Anonymous function всегда объявляет parameter list:

```kotlin
val positive = listOf(-1, 0, 1).filter(fun(value): Boolean {
    return value > 0
})
```

`it` отсутствует. Явное имя полезно в длинной logic.

## Передача только внутри parentheses

Trailing lambda syntax работает только для lambda expressions.

```kotlin
val values = listOf("a", "bb").map(fun(value): Int {
    return value.length
})
```

Нельзя вынести `fun(value) ...` после закрывающей скобки `map` как trailing literal.

Это одна из причин, почему anonymous functions встречаются реже lambdas.

## Несколько parameters

```kotlin
val comparator = Comparator<String>(fun(first, second): Int {
    val byLength = first.length.compareTo(second.length)
    if (byLength != 0) return byLength
    return first.compareTo(second)
})
```

Anonymous function хорошо показывает многоступенчатое сравнение с local returns.

Можно короче создать `Comparator { first, second -> ... }`, если labels/returns не нужны.

## Receiver anonymous function

```kotlin
val decorate: String.(String) -> String = fun String.(prefix: String): String {
    return "$prefix: ${trim()}"
}

println("Anonymous functions".decorate("Урок"))
```

Receiver указывается перед parameter list. Внутри доступны `this` и members String.

```kotlin
val safeLength: String?.() -> Int = fun String?.(): Int {
    if (this == null) return 0
    return trim().length
}
```

Nullable receiver обрабатывается явно.

## Suspend anonymous function

Suspending function literal имеет suspend function type и вызывается только в suspend context. В зависимости от syntax/context можно использовать suspending lambda, которая обычно читается проще:

```kotlin
suspend fun loadTitle(id: Int): String = "Урок $id"

val loader: suspend (Int) -> String = { id ->
    loadTitle(id)
}
```

Если block требует многочисленных local returns, вынеси named `suspend fun`: это лучше поддерживается readers/tooling, чем редкая anonymous syntax.

## Захват variables

Anonymous function создаёт closure так же, как lambda.

```kotlin
fun thresholdPredicate(threshold: Int): (Int) -> Boolean {
    return fun(value: Int): Boolean {
        return value >= threshold
    }
}
```

`threshold` живёт вместе с returned function value.

Mutable capture также возможен и не thread-safe автоматически:

```kotlin
fun accumulator(): (Int) -> Int {
    var total = 0

    return fun(value: Int): Int {
        total += value
        return total
    }
}
```

## Anonymous function и recursion

У function нет имени для прямого self-call. Не создавай fragile recursion через `lateinit` variable. Используй local named function:

```kotlin
fun factorial(value: Int): Long {
    require(value >= 0) { "Число должно быть неотрицательным" }

    fun calculate(current: Int): Long {
        return if (current <= 1) 1L else current * calculate(current - 1)
    }

    return calculate(value)
}
```

## Lambda против anonymous function

Lambda лучше, когда:

- expression короткое;
- trailing syntax улучшает DSL;
- result — последнее expression;
- `it` очевиден;
- нужен non-local return из inline call.

Anonymous function лучше, когда:

- несколько local early returns;
- block похож на обычную function;
- return type/parameters важно видеть рядом;
- non-local return нужно запретить syntax-level;
- label lambda ухудшает чтение.

<MouseTrap title="Мышеловка: return меняет адрес назначения">
В lambda bare return может выйти из enclosing function только в inline context; в anonymous function он всегда local. При замене одной формы другой перепроверь control flow.
</MouseTrap>

## Function reference как третий вариант

```kotlin
fun validScore(score: Int): Boolean = score in 0..100

val filtered = listOf(-1, 40, 101).filter(::validScore)
```

Если logic уже имеет содержательное имя и переиспользуется, reference обычно лучше обоих literals.

## Anonymous function и overload resolution

Explicit parameters/return type могут помочь выбрать overload:

```kotlin
fun consume(transform: (String) -> Int): Int = transform("Kotlin")

val length = consume(fun(value: String): Int {
    return value.length
})
```

Не создавай overloads, различающиеся только function types с одинаковым erased JVM shape без `@JvmName`/разных names.

## Практический пример: parser шагов курса

```kotlin
data class CourseStep(
    val order: Int,
    val title: String,
    val optional: Boolean
)

fun parseSteps(lines: List<String>): List<CourseStep> {
    return lines.mapNotNull(fun(line: String): CourseStep? {
        val normalized = line.trim()
        if (normalized.isEmpty()) return null
        if (normalized.startsWith("#")) return null

        val parts = normalized.split(';')
        if (parts.size != 3) return null

        val order = parts[0].trim().toIntOrNull() ?: return null
        if (order <= 0) return null

        val title = parts[1].trim()
        if (title.isEmpty()) return null

        val optional = when (parts[2].trim().lowercase()) {
            "yes", "true", "1" -> true
            "no", "false", "0" -> false
            else -> return null
        }

        return CourseStep(order, title, optional)
    })
}

fun main() {
    val steps = parseSteps(
        listOf(
            "# order;title;optional",
            "39;Function types;no",
            "40;Lambdas;no",
            "bad line",
            "41;Anonymous functions;yes"
        )
    )

    steps.forEach(::println)
}
```

Anonymous function подходит parser block: каждый invalid condition делает простой local `return null`, а enclosing `parseSteps` продолжает обрабатывать остальные lines.

## Итоги

- anonymous function записывается `fun(parameters): Result { ... }`;
- block body использует явные local `return`;
- expression body записывается через `=`;
- types могут выводиться из expected function type;
- bare return всегда выходит из anonymous function;
- anonymous function не использует implicit `it`;
- trailing syntax доступна lambda, но не anonymous `fun`;
- receiver anonymous function поддерживает `A.() -> B`;
- closure захватывает outer state как lambda;
- local named function лучше для recursion;
- форма особенно полезна для длинного callback с guard clauses.
