## Мутация меняет саму коллекцию

Mutating operation изменяет existing mutable collection. В отличие от `filter`, `map`, `sorted` и других операций, которые возвращают новый результат, mutating operation меняет receiver.

```kotlin
val queue = mutableListOf("введение", "типы")

queue.add("null-safety")

println(queue) // [введение, типы, null-safety]
```

В Kotlin важно различать только для чтения interface и mutable interface:

- `List`, `Set`, `Map` не дают API для изменения через эту ссылку;
- `MutableList`, `MutableSet`, `MutableMap` дают API для изменения;
- только для чтения ссылка не гарантирует immutable object, она только запрещает мутацию через данный тип.

```kotlin
val internal = mutableListOf("a", "b")
val exposed: List<String> = internal

internal.add("c")

println(exposed) // [a, b, c]
```

<CatTip>
Mutable collection — это миска, в которую можно досыпать корм. Только для чтения reference — это прозрачная крышка: через нее видно содержимое, но через эту ссылку нельзя досыпать.
</CatTip>

## `add`

`add(element)` добавляет один element.

```kotlin
val lessons = mutableListOf("01-introduction", "02-types")

val changed = lessons.add("03-null-safety")

println(changed) // true
println(lessons) // [01-introduction, 02-types, 03-null-safety]
```

Для `MutableList` element добавляется в конец. Для `MutableSet` результат зависит от того, был ли element уже внутри set.

```kotlin
val tags = mutableSetOf("kotlin", "basics")

println(tags.add("collections")) // true, set изменился
println(tags.add("kotlin"))      // false, такой element уже был
println(tags)                    // [kotlin, basics, collections]
```

`add` возвращает `Boolean`: изменилась collection или нет. Для list добавление обычно возвращает `true`, для set duplicate может вернуть `false`.

## `add(index, element)` для `MutableList`

List-specific перегрузка вставляет element в конкретную позицию.

```kotlin
val roadmap = mutableListOf("основы", "коллекции", "корутины")

roadmap.add(1, "функции")

println(roadmap) // [основы, функции, коллекции, корутины]
```

Valid positions для вставки: от `0` до `size` включительно. Position `size` означает append в конец.

```kotlin
fun MutableList<String>.insertSafely(index: Int, topic: String) {
    require(index in 0..size) {
        "Позиция вставки должна быть от 0 до $size"
    }

    add(index, topic)
}
```

<MouseTrap>
Для вставки допустим index `size`, а для чтения допустимы только `0..lastIndex`. Это разные границы.
</MouseTrap>

## `addAll`

`addAll(elements)` добавляет несколько elements.

```kotlin
val basics = mutableListOf("переменные", "типы")
val next = listOf("условия", "циклы")

val changed = basics.addAll(next)

println(changed) // true
println(basics) // [переменные, типы, условия, циклы]
```

Argument может быть `Collection`, а в стандартных перегрузки также используются iterable-like inputs. Для list порядок добавления соответствует порядку argument traversal.

```kotlin
val numbers = mutableListOf(1, 2)

numbers.addAll(arrayOf(3, 4))
numbers.addAll(sequenceOf(5, 6))

println(numbers) // [1, 2, 3, 4, 5, 6]
```

Для `MutableList` есть перегрузка `addAll(index, elements)`.

```kotlin
val path = mutableListOf("start", "finish")

path.addAll(1, listOf("practice", "quiz"))

println(path) // [start, practice, quiz, finish]
```

Если добавляемая collection empty, результат обычно `false`, потому что receiver не изменился.

```kotlin
val changedByEmpty = path.addAll(emptyList())

println(changedByEmpty) // false
```

## `+=` и `plusAssign`

Для mutable collections можно использовать `+=` как in-place addition.

```kotlin
val queue = mutableListOf("intro")

queue += "types"
queue += listOf("functions", "collections")

println(queue) // [intro, types, functions, collections]
```

Смысл зависит от типа переменной. Если receiver mutable, `+=` может изменить existing collection. Если переменная только для чтения, compiler может построить новую collection и присвоить ее переменной, если это `var`.

```kotlin
var readOnlyView = listOf("intro")

readOnlyView += "types"

println(readOnlyView) // [intro, types]
```

Чтобы код был очевидным, в разделяемое состояние часто лучше писать явный `add`/`addAll`.

## `remove(element)`

`remove(element)` удаляет один occurrence по value и возвращает `Boolean`.

```kotlin
val attempts = mutableListOf("draft", "review", "draft", "published")

val removed = attempts.remove("draft")

println(removed)  // true
println(attempts) // [review, draft, published]
```

Для list удаляется первый matching element. Если element отсутствует, collection не меняется.

```kotlin
val removedMissing = attempts.remove("archived")

println(removedMissing) // false
```

Для set value уникален, поэтому удаляется сам element, если он есть.

```kotlin
val roles = mutableSetOf("reader", "author")

roles.remove("author")

println(roles) // [reader]
```

## `removeAt(index)` для `MutableList`

`removeAt(index)` удаляет element по index и возвращает удаленное значение.

```kotlin
val lessons = mutableListOf("intro", "types", "collections")

val removedLesson = lessons.removeAt(1)

println(removedLesson) // types
println(lessons)       // [intro, collections]
```

Index должен быть внутри `indices`.

```kotlin
fun MutableList<String>.removeAtOrNull(index: Int): String? {
    if (index !in indices) {
        return null
    }

    return removeAt(index)
}
```

После удаления все elements справа сдвигаются влево. Это важно, если ты удаляешь несколько indices.

```kotlin
fun <T> MutableList<T>.removeIndices(indicesToRemove: Set<Int>) {
    for (index in indicesToRemove.sortedDescending()) {
        // Удаляем справа налево, чтобы ранние удаления не сдвигали еще не обработанные индексы.
        removeAt(index)
    }
}
```

## `removeAll`

`removeAll(elements)` удаляет все elements, которые присутствуют в argument collection.

```kotlin
val visibleTags = mutableListOf("kotlin", "draft", "basics", "internal")

visibleTags.removeAll(listOf("draft", "internal"))

println(visibleTags) // [kotlin, basics]
```

Predicate перегрузка удаляет все elements, для которых predicate возвращает `true`.

```kotlin
val scores = mutableListOf(100, -1, 80, -5, 70)

scores.removeAll { score ->
    // Удаляем испорченные значения.
    score < 0
}

println(scores) // [100, 80, 70]
```

`removeAll` возвращает `Boolean`: была ли collection изменена.

```kotlin
val changed = scores.removeAll { score ->
    score > 100
}

println(changed) // false
```

## `retainAll`

`retainAll(elements)` оставляет только values, которые присутствуют в argument collection. Это opposite of `removeAll`.

```kotlin
val enabledFeatures = mutableSetOf("search", "progress", "admin", "debug")
val productionFeatures = setOf("search", "progress")

enabledFeatures.retainAll(productionFeatures)

println(enabledFeatures) // [search, progress]
```

Predicate перегрузка оставляет только matching elements.

```kotlin
val cachedLessons = mutableListOf("01-introduction", "draft-99", "02-types")

cachedLessons.retainAll { slug ->
    // В cache оставляем только опубликованные уроки.
    slug.firstOrNull()?.isDigit() == true
}

println(cachedLessons) // [01-introduction, 02-types]
```

## `clear`

`clear()` удаляет все elements.

```kotlin
val inMemoryEvents = mutableListOf("opened", "clicked", "completed")

inMemoryEvents.clear()

println(inMemoryEvents) // []
```

После `clear` сама collection instance остается той же, но ее содержимое empty.

```kotlin
val reference = inMemoryEvents

inMemoryEvents.clear()

println(reference.isEmpty()) // true
```

Это важно, если на collection есть другие ссылки.

## `set(index, element)` и `list[index] = element`

`MutableList` позволяет заменить element по index.

```kotlin
val statuses = mutableListOf("created", "draft", "published")

val previous = statuses.set(1, "review")

println(previous) // draft
println(statuses) // [created, review, published]
```

Operator syntax выглядит естественнее, когда предыдущий value не нужен.

```kotlin
statuses[0] = "started"

println(statuses) // [started, review, published]
```

`set` не меняет размер list. Он только заменяет existing element. Для вставки нужен `add(index, element)`.

<MouseTrap>
`list[index] = value` не вставляет новый элемент. Если index равен `size`, будет ошибка. Для вставки в конец используй `add(value)`.
</MouseTrap>

## `MutableMap.put`

`put(key, value)` добавляет или заменяет entry.

```kotlin
val progress = mutableMapOf(
    "01-introduction" to 100,
    "02-types" to 20
)

val previous = progress.put("02-types", 80)

println(previous) // 20
println(progress) // {01-introduction=100, 02-types=80}
```

Если key отсутствовал, `put` вернет `null`.

```kotlin
val missingPrevious = progress.put("03-null-safety", 0)

println(missingPrevious) // null
```

Если map допускает nullable values, `null` как return value не отличает missing key от previous null. Проверяй `containsKey`.

```kotlin
val nullableMap = mutableMapOf<String, Int?>("a" to null)

val hadKey = nullableMap.containsKey("a")
val oldValue = nullableMap.put("a", 10)

println(hadKey)   // true
println(oldValue) // null
```

## `map[key] = value`

Operator syntax для map вызывает update.

```kotlin
progress["04-functions"] = 10

println(progress["04-functions"]) // 10
```

Такой синтаксис хорош для simple assignment, но если нужен previous value, используй `put`.

## `putAll`

`putAll` добавляет все entries из другой map.

```kotlin
val remoteProgress = mapOf(
    "02-types" to 100,
    "05-collections" to 15
)

progress.putAll(remoteProgress)

println(progress["02-types"]) // 100
```

При совпадении key новое value заменяет старое.

## `MutableMap.remove`

`remove(key)` удаляет entry по key и возвращает removed value или `null`.

```kotlin
val removedProgress = progress.remove("05-collections")

println(removedProgress) // 15
```

Overload `remove(key, value)` удаляет entry только если key связан именно с этим value.

```kotlin
val removedExact = progress.remove("02-types", 50)

println(removedExact) // false, потому что значение не 50
```

Это удобно для осторожного удаления state, который мог измениться между чтением и записью.

```kotlin
fun MutableMap<String, Int>.resetIfStillZero(slug: String): Boolean {
    return remove(slug, 0)
}
```

## Views mutable map

`keys`, `values` и `entries` у mutable map связаны с самой map.

```kotlin
val settings = mutableMapOf(
    "theme" to "dark",
    "language" to "ru"
)

settings.keys.remove("language")

println(settings) // {theme=dark}
```

Через `entries` можно менять value существующей entry.

```kotlin
for (entry in settings.entries) {
    if (entry.key == "theme") {
        // Меняем значение entry без удаления ключа.
        entry.setValue("cat-dark")
    }
}
```

Не все реализации поддерживают все mutation-возможности одинаково; если collection пришла из внешнего API, учитывай возможные exceptions.

## Безопасная мутация во время итерации

Не изменяй mutable collection напрямую внутри `for (item in collection)`.

```kotlin
val numbers = mutableListOf(1, 2, 3, 4)

// Такой стиль может привести к ошибке на JVM-реализациях вроде ArrayList.
for (number in numbers) {
    if (number % 2 == 0) {
        numbers.remove(number)
    }
}
```

Используй `removeAll(predicate)` или iterator.

```kotlin
numbers.removeAll { number ->
    // Удаляем все четные значения безопасной bulk-операцией.
    number % 2 == 0
}
```

Iterator нужен, когда логика сложнее.

```kotlin
val iterator = numbers.iterator()

while (iterator.hasNext()) {
    val number = iterator.next()

    if (number < 0) {
        // Удаляем текущий element через iterator, который контролирует traversal.
        iterator.remove()
    }
}
```

## Боевой-пример: mutable cache с явными правилами

```kotlin
data class CachedLesson(
    val slug: String,
    val title: String,
    val updatedAtMillis: Long
)

class LessonCache(
    private val maxSize: Int
) {
    private val lessonsBySlug = linkedMapOf<String, CachedLesson>()

    fun put(lesson: CachedLesson) {
        require(lesson.slug.isNotBlank()) {
            "Slug урока не должен быть пустым"
        }
        require(maxSize > 0) {
            "Размер cache должен быть положительным"
        }

        lessonsBySlug[lesson.slug] = lesson

        while (lessonsBySlug.size > maxSize) {
            val oldestKey = lessonsBySlug.keys.first()

            // Удаляем самый старый entry, потому что linkedMap сохраняет порядок вставки.
            lessonsBySlug.remove(oldestKey)
        }
    }

    fun removeStale(minAllowedTimestampMillis: Long): Int {
        val before = lessonsBySlug.size

        lessonsBySlug.entries.removeAll { (_, lesson) ->
            // Устаревшие уроки больше не нужны в памяти.
            lesson.updatedAtMillis < minAllowedTimestampMillis
        }

        return before - lessonsBySlug.size
    }

    fun snapshot(): Map<String, CachedLesson> {
        // Наружу отдаем read-only copy, чтобы caller не мог мутировать внутреннее состояние.
        return lessonsBySlug.toMap()
    }
}
```

Этот пример показывает основной принцип: изменяемое состояние должен иметь владельца, а наружу лучше отдавать snapshot или только для чтения view с понятной жизнью.

## Когда избегать мутации

Мутация уместна для cache, builders, parsers, adapters и performance-critical участков. Но в доменной логике часто проще вернуть новую value.

```kotlin
data class CourseState(
    val completedLessons: Set<String>
)

fun CourseState.complete(slug: String): CourseState {
    return copy(
        completedLessons = completedLessons + slug
    )
}
```

Такой код проще тестировать и безопаснее передавать между слоями.

<CatTip>
Мутируй там, где есть ясный хозяин состояния. Если непонятно, кто владеет collection, сделай immutable copy и не выпускай когти наружу.
</CatTip>
