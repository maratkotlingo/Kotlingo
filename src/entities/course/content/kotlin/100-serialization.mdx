## Что такое serialization

Serialization — преобразование Kotlin object graph в переносимый формат: JSON, ProtoBuf, CBOR, properties или другой representation.

Deserialization — обратный процесс.

Типичные места:

- HTTP request/response DTO;
- configuration files;
- local cache;
- event payloads;
- messages in queues;
- persistence snapshots;
- interop with JavaScript/Swift/backend services;
- test fixtures.

```kotlin
import kotlinx.serialization.Serializable
import kotlinx.serialization.encodeToString
import kotlinx.serialization.json.Json

@Serializable
data class LessonDto(
    val slug: String,
    val title: String
)

fun encodeLesson(): String {
    return Json.encodeToString(
        LessonDto(
            slug = "100-serialization",
            title = "Serialization"
        )
    )
}
```

<CatTip>
Serializer — это кошачий переносчик: внутри может быть богатая Kotlin-модель, но наружу она выходит в форме, понятной другой системе.
</CatTip>

## Gradle setup

`kotlinx.serialization` требует compiler plugin.

```kotlin
plugins {
    kotlin("jvm") version "2.4.0"
    kotlin("plugin.serialization") version "2.4.0"
}

dependencies {
    implementation("org.jetbrains.kotlinx:kotlinx-serialization-json:1.9.0")
}
```

KMP:

```kotlin
plugins {
    kotlin("multiplatform") version "2.4.0"
    kotlin("plugin.serialization") version "2.4.0"
}

kotlin {
    sourceSets {
        commonMain.dependencies {
            implementation("org.jetbrains.kotlinx:kotlinx-serialization-json:1.9.0")
        }
    }
}
```

The plugin generates serializers at compile time. Reflection is not required for normal generated serializers.

## `@Serializable`

Annotate class with `@Serializable`.

```kotlin
import kotlinx.serialization.Serializable

@Serializable
data class ProgressDto(
    val userId: String,
    val completedLessons: Int,
    val totalLessons: Int
)
```

Supported declaration shapes include:

- data classes;
- regular classes with serializable primary constructor properties;
- objects;
- enums;
- sealed classes and sealed interfaces;
- value classes with supported underlying value;
- collections of serializable values;
- maps with supported key/value types.

```kotlin
import kotlinx.serialization.Serializable

@Serializable
@JvmInline
value class LessonSlugDto(
    val value: String
)

@Serializable
enum class LessonDifficultyDto {
    BEGINNER,
    INTERMEDIATE,
    ADVANCED
}
```

Properties in primary constructor are serialized by default.

```kotlin
@Serializable
data class LessonCardDto(
    val slug: String,
    val title: String,
    val difficulty: LessonDifficultyDto
)
```

Non-constructor property with backing field can be serialized too, but domain DTOs are usually clearer with constructor properties.

## `@SerialName`

`@SerialName` changes external field name.

```kotlin
import kotlinx.serialization.SerialName
import kotlinx.serialization.Serializable

@Serializable
data class LessonApiDto(
    @SerialName("lesson_slug")
    val slug: String,

    @SerialName("display_title")
    val title: String
)
```

Use it when:

- API uses snake_case;
- legacy field names cannot change;
- Kotlin name should stay idiomatic;
- enum external names differ from Kotlin enum entries.

```kotlin
@Serializable
enum class PublishStateDto {
    @SerialName("draft")
    DRAFT,

    @SerialName("published")
    PUBLISHED
}
```

<MouseTrap>
Не переименовывай Kotlin property только ради JSON. Kotlin code должен оставаться idiomatic; wire format настраивается через `@SerialName`.
</MouseTrap>

## Defaults and missing fields

Default values help backward compatibility.

```kotlin
import kotlinx.serialization.Serializable
import kotlinx.serialization.json.Json
import kotlinx.serialization.decodeFromString

@Serializable
data class LessonSettingsDto(
    val theme: String = "dark",
    val fontSize: Int = 16,
    val showHints: Boolean = true
)

fun decodeOldSettings(json: String): LessonSettingsDto {
    return Json.decodeFromString(json)
}
```

If input JSON misses `fontSize`, default `16` is used.

Required fields without default must be present.

```kotlin
@Serializable
data class RequiredLessonDto(
    val slug: String,
    val title: String
)
```

Missing `slug` fails deserialization. This is good for contract-critical data.

## Unknown keys

By default, unknown JSON keys fail deserialization. For forward-compatible clients, configure `ignoreUnknownKeys`.

```kotlin
import kotlinx.serialization.json.Json

val apiJson = Json {
    ignoreUnknownKeys = true
}
```

Usage:

```kotlin
import kotlinx.serialization.Serializable
import kotlinx.serialization.decodeFromString

@Serializable
data class LessonSummaryDto(
    val slug: String,
    val title: String
)

fun decodeSummary(payload: String): LessonSummaryDto {
    return apiJson.decodeFromString(payload)
}
```

Use `ignoreUnknownKeys = true` for external APIs where server may add fields. Keep strict mode for internal config if unknown keys should be treated as mistakes.

## Encoding defaults and explicit nulls

`encodeDefaults` controls whether default-valued properties are emitted.

```kotlin
import kotlinx.serialization.Serializable
import kotlinx.serialization.encodeToString
import kotlinx.serialization.json.Json

@Serializable
data class ReaderOptionsDto(
    val theme: String = "dark",
    val showProgress: Boolean = true
)

fun encodeWithDefaults(): String {
    val json = Json {
        encodeDefaults = true
    }

    return json.encodeToString(ReaderOptionsDto())
}
```

`explicitNulls` controls null emission/handling in JSON.

```kotlin
import kotlinx.serialization.Serializable
import kotlinx.serialization.encodeToString
import kotlinx.serialization.json.Json

@Serializable
data class UserProfileDto(
    val displayName: String?,
    val avatarUrl: String?
)

fun encodeWithoutExplicitNulls(): String {
    val json = Json {
        explicitNulls = false
    }

    return json.encodeToString(
        UserProfileDto(
            displayName = null,
            avatarUrl = null
        )
    )
}
```

Be careful: omitting null and sending null are different contract decisions in many APIs.

## DTOs vs domain models

Do not automatically annotate every domain model. Serialization is an external contract; domain is business contract.

DTO:

```kotlin
import kotlinx.serialization.Serializable

@Serializable
data class ProgressResponseDto(
    val user_id: String,
    val completed: Int,
    val total: Int
)
```

Domain:

```kotlin
data class ProgressSnapshot(
    val userId: UserId,
    val completedLessons: Int,
    val totalLessons: Int
) {
    val percent: Int
        get() = if (totalLessons == 0) 0 else completedLessons * 100 / totalLessons
}

@JvmInline
value class UserId(
    val value: String
)
```

Mapper:

```kotlin
fun ProgressSnapshot.toDto(): ProgressResponseDto {
    return ProgressResponseDto(
        user_id = userId.value,
        completed = completedLessons,
        total = totalLessons
    )
}
```

This separates wire compatibility from domain design.

## Formats

`kotlinx.serialization` supports multiple formats.

JSON:

```kotlin
import kotlinx.serialization.encodeToString
import kotlinx.serialization.json.Json

fun toJson(dto: LessonDto): String {
    return Json.encodeToString(dto)
}
```

ProtoBuf:

```kotlin
import kotlinx.serialization.ExperimentalSerializationApi
import kotlinx.serialization.Serializable
import kotlinx.serialization.protobuf.ProtoBuf

@OptIn(ExperimentalSerializationApi::class)
fun toProtoBytes(dto: LessonDto): ByteArray {
    return ProtoBuf.encodeToByteArray(LessonDto.serializer(), dto)
}
```

CBOR:

```kotlin
import kotlinx.serialization.ExperimentalSerializationApi
import kotlinx.serialization.cbor.Cbor

@OptIn(ExperimentalSerializationApi::class)
fun toCborBytes(dto: LessonDto): ByteArray {
    return Cbor.encodeToByteArray(LessonDto.serializer(), dto)
}
```

Properties:

```kotlin
import kotlinx.serialization.ExperimentalSerializationApi
import kotlinx.serialization.properties.Properties

@OptIn(ExperimentalSerializationApi::class)
fun toProperties(dto: LessonDto): Map<String, Any> {
    return Properties.encodeToMap(LessonDto.serializer(), dto)
}
```

Format choice is part of API design. JSON is inspectable, ProtoBuf is compact, CBOR is binary, properties are useful for flat configuration-like structures.

## Custom serializers

Use custom serializer when external representation differs from Kotlin representation.

```kotlin
import kotlinx.serialization.KSerializer
import kotlinx.serialization.Serializable
import kotlinx.serialization.descriptors.PrimitiveKind
import kotlinx.serialization.descriptors.PrimitiveSerialDescriptor
import kotlinx.serialization.descriptors.SerialDescriptor
import kotlinx.serialization.encoding.Decoder
import kotlinx.serialization.encoding.Encoder

@Serializable(with = LessonSlugSerializer::class)
@JvmInline
value class LessonSlug(
    val value: String
)

object LessonSlugSerializer : KSerializer<LessonSlug> {
    override val descriptor: SerialDescriptor =
        PrimitiveSerialDescriptor("LessonSlug", PrimitiveKind.STRING)

    override fun serialize(encoder: Encoder, value: LessonSlug) {
        // Во внешний формат уходит только строковое значение.
        encoder.encodeString(value.value)
    }

    override fun deserialize(decoder: Decoder): LessonSlug {
        val raw = decoder.decodeString()

        require(raw.matches(Regex("[a-z0-9-]+"))) {
            "Некорректный slug: $raw"
        }

        return LessonSlug(raw)
    }
}
```

Prefer generated serializers for simple DTOs. Custom serializer is for real representation mismatch or validation boundary.

## Polymorphism

Sealed hierarchies serialize with known subclasses.

```kotlin
import kotlinx.serialization.SerialName
import kotlinx.serialization.Serializable
import kotlinx.serialization.encodeToString
import kotlinx.serialization.json.Json

@Serializable
sealed interface LessonEventDto {
    @Serializable
    @SerialName("opened")
    data class Opened(
        val slug: String
    ) : LessonEventDto

    @Serializable
    @SerialName("completed")
    data class Completed(
        val slug: String,
        val score: Int
    ) : LessonEventDto
}

fun encodeEvent(event: LessonEventDto): String {
    val json = Json {
        classDiscriminator = "type"
    }

    return json.encodeToString(event)
}
```

Open polymorphism requires a `SerializersModule`.

```kotlin
import kotlinx.serialization.Serializable
import kotlinx.serialization.modules.SerializersModule
import kotlinx.serialization.modules.polymorphic
import kotlinx.serialization.modules.subclass
import kotlinx.serialization.json.Json

@Serializable
abstract class NotificationDto {
    abstract val id: String
}

@Serializable
data class EmailNotificationDto(
    override val id: String,
    val address: String
) : NotificationDto()

val notificationModule = SerializersModule {
    polymorphic(NotificationDto::class) {
        subclass(EmailNotificationDto::class)
    }
}

val notificationJson = Json {
    serializersModule = notificationModule
    classDiscriminator = "kind"
}
```

<MouseTrap>
Polymorphic serialization is part of wire contract. Changing discriminator names or subclass names can break old clients even if Kotlin code still compiles.
</MouseTrap>

## Contextual serialization

Contextual serialization lets you provide serializer through `SerializersModule`.

```kotlin
import kotlinx.serialization.Contextual
import kotlinx.serialization.Serializable
import java.time.Instant

@Serializable
data class JvmAuditDto(
    val id: String,
    @Contextual
    val createdAt: Instant
)
```

Use this for types you do not own, like `Instant`, when a project-wide serializer is appropriate.

## Versioning and compatibility

Wire format changes need compatibility rules.

Backward-compatible changes:

- add optional field with default;
- add nullable field if old producers omit it;
- ignore unknown keys on readers that receive server responses;
- keep old `@SerialName`;
- introduce new enum value only if readers have fallback strategy;
- add new sealed subtype only if receivers can handle unknown type.

Breaking changes:

- rename field without alias strategy;
- remove required field;
- change field type;
- change discriminator;
- turn optional field into required;
- change semantic meaning while keeping same name.

Versioned DTO:

```kotlin
import kotlinx.serialization.Serializable

@Serializable
data class LessonPayloadV2(
    val schemaVersion: Int = 2,
    val slug: String,
    val title: String,
    val body: String,
    val tags: List<String> = emptyList()
)
```

Migration helper:

```kotlin
import kotlinx.serialization.json.Json
import kotlinx.serialization.json.JsonObject
import kotlinx.serialization.json.intOrNull
import kotlinx.serialization.json.jsonPrimitive

fun decodeLessonPayload(raw: String): LessonPayloadV2 {
    val element = Json.parseToJsonElement(raw) as JsonObject
    val version = element["schemaVersion"]?.jsonPrimitive?.intOrNull ?: 1

    return when (version) {
        1 -> migrateV1ToV2(raw)
        2 -> Json.decodeFromString(LessonPayloadV2.serializer(), raw)
        else -> error("Неподдерживаемая версия payload: $version")
    }
}

fun migrateV1ToV2(raw: String): LessonPayloadV2 {
    val old = Json.decodeFromString(LessonPayloadV1.serializer(), raw)

    return LessonPayloadV2(
        slug = old.slug,
        title = old.title,
        body = old.body,
        tags = emptyList()
    )
}

@Serializable
data class LessonPayloadV1(
    val slug: String,
    val title: String,
    val body: String
)
```

## Production checklist

Before shipping serialization contract:

- separate DTOs from domain where external compatibility matters;
- define JSON options explicitly;
- use defaults for additive compatibility;
- decide strict vs unknown-key tolerant decoding;
- keep `@SerialName` stable;
- test old payloads against new readers;
- test new payloads against old readers if needed;
- avoid leaking internal sealed hierarchies as public wire contracts;
- document schema version strategy for persisted or cross-service data.

<CatTip>
Serialization в production — это не просто `encodeToString`. Это договор: какие поля живут, какие могут появиться, какие нельзя переименовывать и как старые payloads проходят через новые лапы кода.
</CatTip>
