## Что такое сериализация

Сериализация — преобразование графа Kotlin-объектов в переносимый формат: JSON, ProtoBuf, CBOR, properties или другое представление.

Десериализация — обратный процесс.

Типичные места:

- DTO для HTTP-запросов и ответов;
- файлы конфигурации;
- локальный кэш;
- payload событий;
- сообщения в очередях;
- снимки постоянного состояния;
- интеграция с JavaScript, Swift и серверными сервисами;
- тестовые фикстуры.

```kotlin
import kotlinx.serialization.Serializable
import kotlinx.serialization.encodeToString
import kotlinx.serialization.json.Json

@Serializable
data class LessonDto(
    val slug: String,
    val title: String
)

fun encodeLesson(): String {
    return Json.encodeToString(
        LessonDto(
            slug = "100-serialization",
            title = "Serialization"
        )
    )
}
```

<CatTip>
Serializer — это кошачий переносчик: внутри может быть богатая Kotlin-модель, но наружу она выходит в форме, понятной другой системе.
</CatTip>

## Настройка Gradle

`kotlinx.serialization` требует плагин компилятора.

```kotlin
plugins {
    kotlin("jvm") version "2.4.0"
    kotlin("plugin.serialization") version "2.4.0"
}

dependencies {
    implementation("org.jetbrains.kotlinx:kotlinx-serialization-json:1.9.0")
}
```

KMP:

```kotlin
plugins {
    kotlin("multiplatform") version "2.4.0"
    kotlin("plugin.serialization") version "2.4.0"
}

kotlin {
    sourceSets {
        commonMain.dependencies {
            implementation("org.jetbrains.kotlinx:kotlinx-serialization-json:1.9.0")
        }
    }
}
```

Плагин генерирует сериализаторы во время компиляции. Для обычных сгенерированных сериализаторов reflection не нужна.

## `@Serializable`

Пометь класс аннотацией `@Serializable`.

```kotlin
import kotlinx.serialization.Serializable

@Serializable
data class ProgressDto(
    val userId: String,
    val completedLessons: Int,
    val totalLessons: Int
)
```

Поддерживаемые формы объявлений:

- data class;
- обычные классы с сериализуемыми свойствами в основном конструкторе;
- object-объявления;
- enum-классы;
- sealed-классы и sealed-интерфейсы;
- value-классы с поддерживаемым базовым значением;
- коллекции сериализуемых значений;
- словари с поддерживаемыми типами ключей и значений.

```kotlin
import kotlinx.serialization.Serializable

@Serializable
@JvmInline
value class LessonSlugDto(
    val value: String
)

@Serializable
enum class LessonDifficultyDto {
    BEGINNER,
    INTERMEDIATE,
    ADVANCED
}
```

Свойства в основном конструкторе сериализуются по умолчанию.

```kotlin
@Serializable
data class LessonCardDto(
    val slug: String,
    val title: String,
    val difficulty: LessonDifficultyDto
)
```

Свойство вне конструктора с backing field тоже может сериализоваться, но доменные DTO обычно понятнее, когда свойства объявлены в конструкторе.

## `@SerialName`

`@SerialName` меняет внешнее имя поля.

```kotlin
import kotlinx.serialization.SerialName
import kotlinx.serialization.Serializable

@Serializable
data class LessonApiDto(
    @SerialName("lesson_slug")
    val slug: String,

    @SerialName("display_title")
    val title: String
)
```

Используй ее, когда:

- API использует snake_case;
- старые имена полей нельзя изменить;
- Kotlin-имя должно оставаться идиоматичным;
- внешние имена enum отличаются от Kotlin-элементов enum.

```kotlin
@Serializable
enum class PublishStateDto {
    @SerialName("draft")
    DRAFT,

    @SerialName("published")
    PUBLISHED
}
```

<MouseTrap>
Не переименовывай Kotlin-свойство только ради JSON. Kotlin-код должен оставаться идиоматичным; формат обмена настраивается через `@SerialName`.
</MouseTrap>

## Значения по умолчанию и отсутствующие поля

Значения по умолчанию помогают сохранять обратную совместимость.

```kotlin
import kotlinx.serialization.Serializable
import kotlinx.serialization.json.Json
import kotlinx.serialization.decodeFromString

@Serializable
data class LessonSettingsDto(
    val theme: String = "dark",
    val fontSize: Int = 16,
    val showHints: Boolean = true
)

fun decodeOldSettings(json: String): LessonSettingsDto {
    return Json.decodeFromString(json)
}
```

Если во входном JSON нет `fontSize`, используется значение по умолчанию `16`.

Обязательные поля без значения по умолчанию должны присутствовать.

```kotlin
@Serializable
data class RequiredLessonDto(
    val slug: String,
    val title: String
)
```

Отсутствие `slug` ломает десериализацию. Для данных, критичных к контракту, это правильное поведение.

## Неизвестные ключи

По умолчанию неизвестные ключи JSON ломают десериализацию. Для клиентов с прямой совместимостью настрой `ignoreUnknownKeys`.

```kotlin
import kotlinx.serialization.json.Json

val apiJson = Json {
    ignoreUnknownKeys = true
}
```

Использование:

```kotlin
import kotlinx.serialization.Serializable
import kotlinx.serialization.decodeFromString

@Serializable
data class LessonSummaryDto(
    val slug: String,
    val title: String
)

fun decodeSummary(payload: String): LessonSummaryDto {
    return apiJson.decodeFromString(payload)
}
```

Используй `ignoreUnknownKeys = true` для внешних API, где сервер может добавлять поля. Для внутренней конфигурации оставляй строгий режим, если неизвестные ключи должны считаться ошибками.

## Кодирование значений по умолчанию и явных null

`encodeDefaults` управляет тем, попадут ли свойства со значениями по умолчанию во внешний формат.

```kotlin
import kotlinx.serialization.Serializable
import kotlinx.serialization.encodeToString
import kotlinx.serialization.json.Json

@Serializable
data class ReaderOptionsDto(
    val theme: String = "dark",
    val showProgress: Boolean = true
)

fun encodeWithDefaults(): String {
    val json = Json {
        encodeDefaults = true
    }

    return json.encodeToString(ReaderOptionsDto())
}
```

`explicitNulls` управляет записью и обработкой `null` в JSON.

```kotlin
import kotlinx.serialization.Serializable
import kotlinx.serialization.encodeToString
import kotlinx.serialization.json.Json

@Serializable
data class UserProfileDto(
    val displayName: String?,
    val avatarUrl: String?
)

fun encodeWithoutExplicitNulls(): String {
    val json = Json {
        explicitNulls = false
    }

    return json.encodeToString(
        UserProfileDto(
            displayName = null,
            avatarUrl = null
        )
    )
}
```

Будь осторожен: пропустить `null` и отправить `null` — разные контрактные решения во многих API.

## DTO против доменных моделей

Не размечай каждую доменную модель автоматически. Сериализация — внешний контракт, домен — бизнес-контракт.

DTO:

```kotlin
import kotlinx.serialization.Serializable

@Serializable
data class ProgressResponseDto(
    val user_id: String,
    val completed: Int,
    val total: Int
)
```

Домен:

```kotlin
data class ProgressSnapshot(
    val userId: UserId,
    val completedLessons: Int,
    val totalLessons: Int
) {
    val percent: Int
        get() = if (totalLessons == 0) 0 else completedLessons * 100 / totalLessons
}

@JvmInline
value class UserId(
    val value: String
)
```

Маппер:

```kotlin
fun ProgressSnapshot.toDto(): ProgressResponseDto {
    return ProgressResponseDto(
        user_id = userId.value,
        completed = completedLessons,
        total = totalLessons
    )
}
```

Так совместимость формата обмена отделяется от доменного дизайна.

## Форматы

`kotlinx.serialization` поддерживает несколько форматов.

JSON:

```kotlin
import kotlinx.serialization.encodeToString
import kotlinx.serialization.json.Json

fun toJson(dto: LessonDto): String {
    return Json.encodeToString(dto)
}
```

ProtoBuf:

```kotlin
import kotlinx.serialization.ExperimentalSerializationApi
import kotlinx.serialization.Serializable
import kotlinx.serialization.protobuf.ProtoBuf

@OptIn(ExperimentalSerializationApi::class)
fun toProtoBytes(dto: LessonDto): ByteArray {
    return ProtoBuf.encodeToByteArray(LessonDto.serializer(), dto)
}
```

CBOR:

```kotlin
import kotlinx.serialization.ExperimentalSerializationApi
import kotlinx.serialization.cbor.Cbor

@OptIn(ExperimentalSerializationApi::class)
fun toCborBytes(dto: LessonDto): ByteArray {
    return Cbor.encodeToByteArray(LessonDto.serializer(), dto)
}
```

Properties:

```kotlin
import kotlinx.serialization.ExperimentalSerializationApi
import kotlinx.serialization.properties.Properties

@OptIn(ExperimentalSerializationApi::class)
fun toProperties(dto: LessonDto): Map<String, Any> {
    return Properties.encodeToMap(LessonDto.serializer(), dto)
}
```

Выбор формата — часть дизайна API. JSON удобно читать, ProtoBuf компактен, CBOR бинарный, а properties полезны для плоских конфигурационных структур.

## Пользовательские сериализаторы

Используй пользовательский сериализатор, когда внешнее представление отличается от Kotlin-представления.

```kotlin
import kotlinx.serialization.KSerializer
import kotlinx.serialization.Serializable
import kotlinx.serialization.descriptors.PrimitiveKind
import kotlinx.serialization.descriptors.PrimitiveSerialDescriptor
import kotlinx.serialization.descriptors.SerialDescriptor
import kotlinx.serialization.encoding.Decoder
import kotlinx.serialization.encoding.Encoder

@Serializable(with = LessonSlugSerializer::class)
@JvmInline
value class LessonSlug(
    val value: String
)

object LessonSlugSerializer : KSerializer<LessonSlug> {
    override val descriptor: SerialDescriptor =
        PrimitiveSerialDescriptor("LessonSlug", PrimitiveKind.STRING)

    override fun serialize(encoder: Encoder, value: LessonSlug) {
        // Во внешний формат уходит только строковое значение.
        encoder.encodeString(value.value)
    }

    override fun deserialize(decoder: Decoder): LessonSlug {
        val raw = decoder.decodeString()

        require(raw.matches(Regex("[a-z0-9-]+"))) {
            "Некорректный slug: $raw"
        }

        return LessonSlug(raw)
    }
}
```

Для простых DTO предпочитай сгенерированные сериализаторы. Пользовательский сериализатор нужен при реальном расхождении представлений или на границе валидации.

## Полиморфизм

Sealed-иерархии сериализуются с известными подклассами.

```kotlin
import kotlinx.serialization.SerialName
import kotlinx.serialization.Serializable
import kotlinx.serialization.encodeToString
import kotlinx.serialization.json.Json

@Serializable
sealed interface LessonEventDto {
    @Serializable
    @SerialName("opened")
    data class Opened(
        val slug: String
    ) : LessonEventDto

    @Serializable
    @SerialName("completed")
    data class Completed(
        val slug: String,
        val score: Int
    ) : LessonEventDto
}

fun encodeEvent(event: LessonEventDto): String {
    val json = Json {
        classDiscriminator = "type"
    }

    return json.encodeToString(event)
}
```

Открытый полиморфизм требует `SerializersModule`.

```kotlin
import kotlinx.serialization.Serializable
import kotlinx.serialization.modules.SerializersModule
import kotlinx.serialization.modules.polymorphic
import kotlinx.serialization.modules.subclass
import kotlinx.serialization.json.Json

@Serializable
abstract class NotificationDto {
    abstract val id: String
}

@Serializable
data class EmailNotificationDto(
    override val id: String,
    val address: String
) : NotificationDto()

val notificationModule = SerializersModule {
    polymorphic(NotificationDto::class) {
        subclass(EmailNotificationDto::class)
    }
}

val notificationJson = Json {
    serializersModule = notificationModule
    classDiscriminator = "kind"
}
```

<MouseTrap>
Полиморфная сериализация — часть контракта формата обмена. Изменение имен дискриминаторов или подклассов может сломать старых клиентов, даже если Kotlin-код все еще компилируется.
</MouseTrap>

## Контекстная сериализация

Контекстная сериализация позволяет передать сериализатор через `SerializersModule`.

```kotlin
import kotlinx.serialization.Contextual
import kotlinx.serialization.Serializable
import java.time.Instant

@Serializable
data class JvmAuditDto(
    val id: String,
    @Contextual
    val createdAt: Instant
)
```

Используй это для типов, которыми ты не владеешь, например `Instant`, когда проектный сериализатор уместен.

## Версионирование и совместимость

Изменения формата обмена требуют правил совместимости.

Обратно совместимые изменения:

- добавить необязательное поле со значением по умолчанию;
- добавить nullable-поле, если старые producer-ы его пропускают;
- игнорировать неизвестные ключи на reader-ах, которые получают ответы сервера;
- сохранять старый `@SerialName`;
- вводить новое значение enum только если у reader-ов есть fallback-стратегия;
- добавлять новый sealed-подтип только если получатели умеют обрабатывать неизвестный тип.

Ломающие изменения:

- переименовать поле без стратегии алиасов;
- удалить обязательное поле;
- изменить тип поля;
- изменить дискриминатор;
- превратить необязательное поле в обязательное;
- изменить семантический смысл, сохранив прежнее имя.

Версионированный DTO:

```kotlin
import kotlinx.serialization.Serializable

@Serializable
data class LessonPayloadV2(
    val schemaVersion: Int = 2,
    val slug: String,
    val title: String,
    val body: String,
    val tags: List<String> = emptyList()
)
```

Помощник миграции:

```kotlin
import kotlinx.serialization.json.Json
import kotlinx.serialization.json.JsonObject
import kotlinx.serialization.json.intOrNull
import kotlinx.serialization.json.jsonPrimitive

fun decodeLessonPayload(raw: String): LessonPayloadV2 {
    val element = Json.parseToJsonElement(raw) as JsonObject
    val version = element["schemaVersion"]?.jsonPrimitive?.intOrNull ?: 1

    return when (version) {
        1 -> migrateV1ToV2(raw)
        2 -> Json.decodeFromString(LessonPayloadV2.serializer(), raw)
        else -> error("Неподдерживаемая версия payload: $version")
    }
}

fun migrateV1ToV2(raw: String): LessonPayloadV2 {
    val old = Json.decodeFromString(LessonPayloadV1.serializer(), raw)

    return LessonPayloadV2(
        slug = old.slug,
        title = old.title,
        body = old.body,
        tags = emptyList()
    )
}

@Serializable
data class LessonPayloadV1(
    val slug: String,
    val title: String,
    val body: String
)
```

## Проверочный список для боевого кода

Перед выпуском контракта сериализации:

- отделить DTO от домена там, где важна внешняя совместимость;
- явно задать JSON-настройки;
- использовать значения по умолчанию для добавочной совместимости;
- выбрать строгую декодировку или режим, терпимый к неизвестным ключам;
- сохранять `@SerialName` стабильным;
- проверять старые payload-ы на новых reader-ах;
- при необходимости проверять новые payload-ы на старых reader-ах;
- не выпускать внутренние sealed-иерархии как публичные wire-контракты;
- документировать стратегию версий схемы для сохраняемых или межсервисных данных.

<CatTip>
Сериализация в боевой среде — это не просто `encodeToString`. Это договор: какие поля живут, какие могут появиться, какие нельзя переименовывать и как старые payload-ы проходят через новые лапы кода.
</CatTip>
