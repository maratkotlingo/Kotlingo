## Класс, объект и экземпляр

**Класс** описывает структуру данных и поведение. **Экземпляр**, или объект класса, содержит конкретные значения этой структуры во время выполнения.

```kotlin
class Lesson(
    val title: String,
    val order: Int
) {
    fun displayName(): String {
        return "$order. $title"
    }
}

fun main() {
    val first = Lesson("Классы и экземпляры", 22)
    val second = Lesson("Свойства", 23)

    println(first.displayName())
    println(second.displayName())
}
```

`Lesson` — класс. `first` и `second` — два независимых экземпляра. У каждого собственные значения `title` и `order`, а реализация `displayName` общая.

## Минимальное объявление класса

```kotlin
class Marker
```

Фигурные скобки можно опустить, если тело пустое. Экземпляр создаётся без `new`:

```kotlin
val marker = Marker()
```

Kotlin использует вызов constructor как обычное выражение. Ключевого слова `new` нет.

## Primary constructor

Primary constructor объявляется в заголовке класса после имени.

```kotlin
class Student(name: String, level: Int)
```

`name` и `level` здесь являются constructor parameters. Без `val` или `var` они не становятся properties автоматически и доступны только во время инициализации тела класса.

```kotlin
class Student(name: String) {
    val normalizedName: String = name.trim()
}
```

После завершения инициализации обратиться к `student.name` нельзя, потому что property `name` не объявлена. Доступна только `normalizedName`.

### Properties в primary constructor

Добавление `val` или `var` одновременно объявляет parameter и property.

```kotlin
class Student(
    val name: String,
    var level: Int
)
```

Теперь `student.name` можно читать, а `student.level` — читать и изменять.

```kotlin
fun main() {
    val student = Student("Анна", 1)
    student.level = 2

    println("${student.name}: уровень ${student.level}")
}
```

Свойства и управление мутацией подробно рассматриваются в следующем уроке.

## Ключевое слово `constructor`

Если у primary constructor нет annotations и visibility modifier, слово `constructor` обычно опускают:

```kotlin
class Lesson(val title: String)
```

Оно необходимо, когда constructor имеет modifier или annotation:

```kotlin
class Lesson private constructor(val title: String)
```

```kotlin
class AuditedLesson @Deprecated("Используй фабрику create") constructor(
    val title: String
)
```

Private constructor контролирует создание объектов через members класса, companion factory или разрешённые secondary constructors.

## Тело класса

Внутри класса можно объявлять:

- properties;
- functions;
- `init` blocks;
- secondary constructors;
- nested и inner classes;
- object/companion object;
- type aliases нельзя объявлять как обычные nested aliases в стабильном общем случае без соответствующей language feature.

```kotlin
class Course(
    val title: String,
    private val lessonTitles: List<String>
) {
    val lessonCount: Int = lessonTitles.size

    fun containsLesson(title: String): Boolean {
        return lessonTitles.any { it.equals(title, ignoreCase = true) }
    }
}
```

## `init` blocks

Primary constructor не имеет отдельного block body. Код первичной инициализации размещают в `init`.

```kotlin
class Lesson(
    val title: String,
    val order: Int
) {
    init {
        require(title.isNotBlank()) { "Название урока не должно быть пустым" }
        require(order > 0) { "Порядок урока должен быть положительным" }
    }
}
```

`init` выполняется при каждом создании экземпляра. Если проверка выбросит исключение, объект не будет успешно создан.

### Несколько init-блоков

В классе может быть несколько `init`. Они выполняются сверху вниз, вперемешку с property initializers в текстовом порядке.

```kotlin
class InitializationTrace(rawTitle: String) {
    val title: String = rawTitle.trim().also {
        println("1. Инициализировано свойство title")
    }

    init {
        println("2. Первый init: $title")
    }

    val slug: String = title.lowercase().replace(' ', '-').also {
        println("3. Инициализировано свойство slug")
    }

    init {
        println("4. Второй init: $slug")
    }
}
```

Порядок определяется расположением в теле, а не тем, что все properties якобы инициализируются до всех `init`.

<MouseTrap title="Мышеловка: текстовый порядок имеет значение">
Не обращайся из раннего initializer или `init` к property, объявленной ниже, через open/member-вызовы или обходные пути. Даже если компилятор разрешит часть сценариев, значение может ещё не быть установлено. Организуй зависимости сверху вниз.
</MouseTrap>

## Порядок инициализации экземпляра

Для обычного класса без наследования модель такова:

1. вычисляются аргументы вызова constructor;
2. параметры primary constructor становятся доступны initializers;
3. property initializers и `init` blocks выполняются сверху вниз;
4. выполняется тело secondary constructor, если объект создавался через него;
5. вызывающий код получает полностью созданный экземпляр.

```kotlin
class Lesson(rawTitle: String, order: Int) {
    val title: String = rawTitle.trim()
    val order: Int = order

    init {
        require(title.isNotEmpty()) { "Название обязательно" }
        require(this.order > 0) { "Порядок должен быть положительным" }
    }
}
```

Аргумент `rawTitle` нормализуется один раз, затем проверяется сохранённое значение `title`.

## Инвариант класса

Инвариант — условие, которое должно быть истинно для каждого корректно созданного объекта на протяжении его доступного состояния.

```kotlin
class Score private constructor(val value: Int) {
    companion object {
        fun of(value: Int): Score {
            require(value in 0..100) { "Балл должен быть от 0 до 100" }
            return Score(value)
        }
    }
}
```

Private constructor и единственная factory гарантируют диапазон. Если бы `value` был public mutable `var`, инвариант можно было бы нарушить после создания.

<CatTip title="Котосовет: создавай объект корректным сразу">
Лучше отклонить некорректные аргументы в constructor/factory, чем создавать полупустой объект и надеяться на последующий `initialize()`. Полностью валидный экземпляр проще использовать, тестировать и передавать между потоками.
</CatTip>

## Secondary constructors

Secondary constructor объявляется в теле ключевым словом `constructor`.

```kotlin
class Lesson(
    val title: String,
    val order: Int
) {
    constructor(title: String) : this(
        title = title,
        order = 1
    )
}
```

Если у класса есть primary constructor, каждый secondary constructor должен прямо или через другой secondary constructor делегировать ему через `this(...)`.

```kotlin
class Lesson(
    val title: String,
    val order: Int,
    val published: Boolean
) {
    constructor(title: String) : this(
        title = title,
        order = 1,
        published = false
    )

    constructor(title: String, order: Int) : this(
        title = title,
        order = order,
        published = false
    )
}
```

Default arguments часто заменяют такие перегрузки:

```kotlin
class Lesson(
    val title: String,
    val order: Int = 1,
    val published: Boolean = false
)
```

Secondary constructors полезны, когда:

- нужна особая Java-friendly форма;
- разные формы входа требуют distinct parsing;
- framework создаёт объект по конкретной constructor-сигнатуре;
- у класса нет primary constructor и нужны несколько сценариев инициализации.

Для именованных способов создания `Lesson.fromPath(...)` или `Lesson.draft(...)` companion factories обычно выразительнее.

## Тело secondary constructor

```kotlin
class ImportedLesson(
    val title: String,
    val source: String
) {
    constructor(csvLine: String) : this(
        title = csvLine.substringBefore(';').trim(),
        source = "csv"
    ) {
        require(';' in csvLine) { "CSV-строка должна содержать разделитель" }
        println("Урок импортирован из CSV")
    }
}
```

Сначала вычисляется делегирование `this(...)`, выполняются property initializers и `init`, и только затем тело secondary constructor. Поэтому поздняя проверка формата здесь неидеальна: часть parsing уже произошла.

Надёжнее factory, которая сначала проверяет вход:

```kotlin
class SafeImportedLesson private constructor(
    val title: String,
    val source: String
) {
    companion object {
        fun fromCsv(csvLine: String): SafeImportedLesson {
            val parts = csvLine.split(';', limit = 2)
            require(parts.size == 2) { "CSV-строка должна содержать разделитель" }

            val title = parts[0].trim()
            require(title.isNotEmpty()) { "Название не должно быть пустым" }

            return SafeImportedLesson(title = title, source = parts[1].trim())
        }
    }
}
```

Factory имеет имя, может вернуть nullable/result тип и выполняет validation до constructor.

## Класс без primary constructor

```kotlin
class Token {
    val value: String

    constructor(raw: String) {
        val normalized = raw.trim()
        require(normalized.isNotEmpty()) { "Токен не должен быть пустым" }
        value = normalized
    }
}
```

Все non-null `val` properties должны быть определённо инициализированы на каждом constructor path. Primary constructor обычно делает этот контракт короче и видимее.

## `this`: текущий экземпляр

Внутри member-функции `this` ссылается на текущий экземпляр.

```kotlin
class Lesson(val title: String) {
    fun sameTitleAs(other: Lesson): Boolean {
        return this.title == other.title
    }
}
```

Явный `this` часто не нужен. Он полезен при затенении имени:

```kotlin
class Progress(var completed: Int) {
    fun update(completed: Int) {
        require(completed >= 0) { "Прогресс не может быть отрицательным" }
        this.completed = completed
    }
}
```

`completed` справа — parameter, `this.completed` слева — property.

### Qualified `this`

Во вложенных receiver scopes используют label.

```kotlin
class Course(private val title: String) {
    fun render(lessons: List<String>): List<String> {
        return lessons.map { lesson ->
            "${this@Course.title}: $lesson"
        }
    }
}
```

`this@Course` однозначно указывает на экземпляр `Course`.

## Утечка `this` во время инициализации

Передавать `this` наружу из constructor/init опасно: внешний код увидит ещё не полностью инициализированный объект.

```kotlin
object Registry {
    private val lessons = mutableListOf<RegisteredLesson>()

    fun register(lesson: RegisteredLesson) {
        lessons += lesson
    }
}

class RegisteredLesson(val title: String) {
    val slug: String

    init {
        slug = title.trim().lowercase().replace(' ', '-')
        Registry.register(this)
    }
}
```

Здесь регистрация стоит после инициализации `slug`, но будущая property ниже может снова сделать объект частично готовым. Лучше завершить создание, затем явно зарегистрировать через service/factory.

Особенно опасен вызов open member из base constructor: override subclass может читать ещё не инициализированное состояние. Подробно это будет разобрано в наследовании.

## Identity и equality

Обычный класс наследует `equals`, `hashCode`, `toString` от `Any`. Без override два разных экземпляра обычно не равны по `==`.

```kotlin
class Lesson(val title: String)

fun main() {
    val first = Lesson("Классы")
    val second = Lesson("Классы")

    println(first == second)
    println(first === second)
}
```

`===` всегда проверяет identity ссылок. `==` вызывает `equals`; у обычного класса базовая реализация также соответствует identity. Data classes автоматически дают value equality и рассматриваются отдельным уроком.

## Класс и состояние

Поведение класса должно защищать его данные.

```kotlin
class CourseProgress(private val totalLessons: Int) {
    private var completedLessons: Int = 0

    init {
        require(totalLessons > 0) { "Курс должен содержать уроки" }
    }

    fun completeNext(): Boolean {
        if (completedLessons == totalLessons) return false

        completedLessons += 1
        return true
    }

    fun snapshot(): String {
        return "$completedLessons из $totalLessons"
    }
}
```

Клиент не может установить `completedLessons = -10` или значение больше total. Методы представляют допустимые переходы состояния.

## Объекты без состояния

Если класс не хранит состояние и не участвует в dependency injection/polymorphism, top-level функция может быть проще. Для единственного singleton-экземпляра Kotlin предоставляет `object`, который изучается позже.

```kotlin
fun normalizedTitle(raw: String): String = raw.trim()
```

Не создавай класс только ради группировки статических helpers.

## Практический пример: агрегат учебной сессии

```kotlin
class StudySession private constructor(
    val studentName: String,
    private val lessonIds: List<Int>,
    initialIndex: Int
) {
    private var currentIndex: Int = initialIndex

    init {
        require(studentName.isNotBlank()) { "Имя студента обязательно" }
        require(lessonIds.isNotEmpty()) { "Сессия должна содержать уроки" }
        require(lessonIds.all { id -> id > 0 }) {
            "Все идентификаторы уроков должны быть положительными"
        }
        require(lessonIds.distinct().size == lessonIds.size) {
            "Список уроков не должен содержать дубликаты"
        }
        require(initialIndex in lessonIds.indices) {
            "Начальная позиция находится вне списка уроков"
        }
    }

    val currentLessonId: Int
        get() = lessonIds[currentIndex]

    val isLastLesson: Boolean
        get() = currentIndex == lessonIds.lastIndex

    fun moveNext(): Boolean {
        if (isLastLesson) return false

        currentIndex += 1
        return true
    }

    fun position(): String {
        return "${currentIndex + 1} из ${lessonIds.size}"
    }

    companion object {
        fun start(studentName: String, lessonIds: List<Int>): StudySession {
            return StudySession(
                studentName = studentName.trim(),
                lessonIds = lessonIds.toList(),
                initialIndex = 0
            )
        }

        fun restore(
            studentName: String,
            lessonIds: List<Int>,
            currentLessonId: Int
        ): StudySession {
            val index = lessonIds.indexOf(currentLessonId)
            require(index >= 0) { "Текущий урок отсутствует в сессии" }

            return StudySession(
                studentName = studentName.trim(),
                lessonIds = lessonIds.toList(),
                initialIndex = index
            )
        }
    }
}

fun main() {
    val session = StudySession.start(
        studentName = "  Мария  ",
        lessonIds = listOf(21, 22, 23)
    )

    println("${session.studentName}: урок ${session.currentLessonId}")
    println(session.position())

    while (session.moveNext()) {
        println("Следующий урок: ${session.currentLessonId}")
    }
}
```

Архитектурные свойства примера:

- private primary constructor не позволяет обойти factories;
- factories имеют предметные имена `start` и `restore`;
- defensive copy `toList()` изолирует внутренний список;
- `init` устанавливает все инварианты;
- private mutable index меняется только допустимым методом;
- computed properties не дублируют состояние;
- объект полностью готов сразу после создания.

## Итоги

- class описывает структуру и поведение, instance содержит конкретное состояние;
- экземпляр создаётся вызовом constructor без `new`;
- primary constructor находится в заголовке класса;
- parameter становится property только с `val` или `var`;
- `init` выполняет проверки и первичную логику;
- initializers и `init` blocks выполняются в текстовом порядке;
- secondary constructor обязан делегировать primary через `this(...)`;
- defaults или named factories часто яснее множества secondary constructors;
- `this` обозначает текущий экземпляр и разрешает затенение;
- нельзя безопасно публиковать `this` до завершения инициализации;
- private constructor и methods помогают сохранять инварианты;
- хороший объект создаётся валидным и разрешает только допустимые изменения состояния.
