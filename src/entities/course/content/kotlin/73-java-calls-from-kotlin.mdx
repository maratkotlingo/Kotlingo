## Kotlin естественно вызывает Java

Kotlin спроектирован для JVM interop: Java classes, methods, fields, collections, exceptions и generics можно использовать из Kotlin почти напрямую.

```kotlin
import java.util.ArrayList

val lessons = ArrayList<String>()

lessons.add("01-introduction")
lessons.add("02-types")

println(lessons[0])
```

Но interop имеет нюансы: synthetic properties, platform types, checked exceptions, SAM conversions, raw types и static members.

<CatTip>
Java interop — это переход между двумя кошачьими комнатами: дверь широкая, но на пороге есть перепады высоты. Их нужно знать, чтобы не споткнуться.
</CatTip>

## Getters и setters как properties

JavaBean getters/setters в Kotlin видны как properties.

Java:

```java
public class JavaLesson {
    private String title;
    private boolean published;

    public String getTitle() {
        return title;
    }

    public void setTitle(String title) {
        this.title = title;
    }

    public boolean isPublished() {
        return published;
    }

    public void setPublished(boolean published) {
        this.published = published;
    }
}
```

Kotlin:

```kotlin
val lesson = JavaLesson()

lesson.title = "Введение"
lesson.isPublished = true

println(lesson.title)
println(lesson.isPublished)
```

Если Java class имеет только setter, Kotlin не показывает set-only property, потому что в Kotlin нет set-only properties.

## Static members

Java static members вызываются через class name.

Java:

```java
public final class LessonIds {
    public static final String INTRO = "01-introduction";

    public static String normalize(String slug) {
        return slug.trim().toLowerCase();
    }
}
```

Kotlin:

```kotlin
val intro = LessonIds.INTRO
val normalized = LessonIds.normalize("  02-TYPES  ")

println(intro)
println(normalized)
```

Java static imports тоже доступны обычным Kotlin imports, если это улучшает читаемость.

```kotlin
import java.lang.Integer.max

val larger = max(10, 20)
```

## SAM conversions

SAM, single abstract method interface, можно реализовать лямбда-выражением.

Java:

```java
public interface LessonCallback {
    void onLesson(String slug);
}

public class LessonRunner {
    public void run(LessonCallback callback) {
        callback.onLesson("01-introduction");
    }
}
```

Kotlin:

```kotlin
val runner = LessonRunner()

runner.run { slug ->
    // Lambda превращается в реализацию Java SAM interface.
    println("Открыт урок $slug")
}
```

Если перегрузки делают лямбда ambiguous, укажи SAM type явно.

```kotlin
runner.run(
    LessonCallback { slug ->
        println("Явный SAM conversion для $slug")
    }
)
```

SAM conversion особенно часто встречается в Java APIs: listeners, обратные вызовы, executors.

## Checked exceptions

Kotlin не имеет checked exceptions. Java methods, объявляющие `throws IOException`, не заставляют Kotlin вызывающий код писать `try/catch`.

Java:

```java
public class JavaFiles {
    public static String readConfig() throws java.io.IOException {
        return "config";
    }
}
```

Kotlin:

```kotlin
val config = JavaFiles.readConfig()
```

Но exception все равно может быть брошен runtime.

```kotlin
fun readConfigSafely(): Result<String> {
    return runCatching {
        // IOException не checked для Kotlin, но может прилететь.
        JavaFiles.readConfig()
    }
}
```

<MouseTrap>
Отсутствие checked exceptions в Kotlin не означает отсутствие ошибок I/O. Оно означает, что compiler не заставляет их ловить.
</MouseTrap>

## Platform types

Java types без nullability annotations приходят в Kotlin как platform types: например `String!`. Kotlin позволяет обращаться с ними как nullable или non-null.

Java:

```java
public class JavaUser {
    public String getName() {
        return null;
    }
}
```

Kotlin:

```kotlin
val name = JavaUser().name

// Это скомпилируется, но может упасть NullPointerException.
println(name.length)
```

Безопаснее нормализовать platform type на границе.

```kotlin
fun JavaUser.safeName(): String {
    return name ?: "anonymous"
}
```

Если Java code имеет nullability annotations, Kotlin учитывает их точнее.

```java
public class AnnotatedUser {
    @org.jetbrains.annotations.NotNull
    public String getName() {
        return "Mila";
    }

    @org.jetbrains.annotations.Nullable
    public String getNickname() {
        return null;
    }
}
```

## Java collections

Java collections становятся Kotlin collection views с platform/nullability нюансами.

```kotlin
fun normalizeJavaList(input: java.util.List<String>): List<String> {
    return input.map { value ->
        // value может быть platform type, если Java не аннотирован.
        value.trim()
    }
}
```

Если Java list может содержать null, не доверяй generic type без annotations.

```kotlin
fun safeStrings(input: java.util.List<String>): List<String> {
    return input.mapNotNull { value ->
        value?.trim()
    }.filter { value ->
        value.isNotEmpty()
    }
}
```

## Raw types

Java raw type в Kotlin обычно становится star projection вроде `List<*>!`.

Java:

```java
public class RawBox {
    public java.util.List rawItems() {
        return java.util.Arrays.asList("kotlin", 42, null);
    }
}
```

Kotlin:

```kotlin
val rawItems = RawBox().rawItems()

for (item in rawItems) {
    // item имеет тип Any? в безопасной модели.
    println(item)
}
```

Чтобы получить typed data, фильтруй или валидируй.

```kotlin
val strings = rawItems.filterIsInstance<String>()

println(strings) // [kotlin]
```

Не делай unchecked cast без проверки.

```kotlin
@Suppress("UNCHECKED_CAST")
fun unsafeCast(raw: java.util.List<*>): List<String> {
    // Такой cast может спрятать ошибку до позднего runtime.
    return raw as List<String>
}
```

## Java generics и wildcards

Java wildcards отображаются в Kotlin type projections.

```java
public interface JavaSource {
    java.util.List<? extends Number> numbers();
    void accept(java.util.List<? super Integer> values);
}
```

Kotlin видит это примерно как producer/consumer types.

```kotlin
fun useSource(source: JavaSource) {
    val numbers = source.numbers()

    for (number in numbers) {
        // Можно читать Number values.
        println(number.toDouble())
    }

    source.accept(mutableListOf<Number>())
}
```

Если Java API слишком raw/unchecked, создай adapter с нормальными Kotlin types.

## Adapter pattern на границе

```kotlin
data class Lesson(
    val slug: String,
    val title: String
)

class JavaLessonApiAdapter(
    private val api: JavaLessonApi
) {
    fun loadLessons(): Result<List<Lesson>> {
        return runCatching {
            api.loadLessons().mapNotNull { javaLesson ->
                val slug = javaLesson.slug?.trim()
                val title = javaLesson.title?.trim()

                if (slug.isNullOrEmpty() || title.isNullOrEmpty()) {
                    null
                } else {
                    Lesson(slug = slug, title = title)
                }
            }
        }
    }
}
```

Здесь adapter превращает platform/null-prone Java model в строгую Kotlin model.

## Java arrays и varargs

Java varargs вызываются естественно.

```java
public class JavaPrinter {
    public static void printAll(String... values) {
        for (String value : values) {
            System.out.println(value);
        }
    }
}
```

Kotlin:

```kotlin
JavaPrinter.printAll("intro", "types", "collections")

val values = arrayOf("functions", "classes")
JavaPrinter.printAll(*values)
```

Spread operator `*` разворачивает array в vararg.

## Практические правила

Нормализуй platform types на границе.

```kotlin
val title = javaLesson.title ?: return null
```

Лови Java exceptions там, где они являются ожидаемым I/O или integration failure.

```kotlin
val result = runCatching { JavaFiles.readConfig() }
```

Не протаскивай raw Java collections глубоко в доменную модель.

```kotlin
val typed = raw.filterIsInstance<String>()
```

Используй SAM conversions, но указывай type явно при перегрузка ambiguity.

```kotlin
runner.run(LessonCallback { slug -> println(slug) })
```

<CatTip>
Хороший Java interop-код похож на прихожую: там снимают грязную обувь platform types, raw types и checked exceptions, а дальше в Kotlin-дом входят чистые typed models.
</CatTip>
