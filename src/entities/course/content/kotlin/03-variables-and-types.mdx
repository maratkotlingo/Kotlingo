## `val` и `var`

```kotlin
val name: Type = expression
var name: Type = expression
```

- `val` — ссылку нельзя переназначить после инициализации.
- `var` — переменную можно переназначать, но ее тип остается тем же.
- В Kotlin обычно начинают с `val` и переходят на `var` только при реальной
  необходимости менять значение.

```kotlin
fun main() {
    val courseName = "kotlingo"
    var completedLessons = 0

    completedLessons += 1

    println("Курс: $courseName")
    println("Пройдено уроков: $completedLessons")
}
```

Нельзя переназначить `val`:

```kotlin
val title = "Переменные"
title = "Типы" // ошибка компиляции
```

Нельзя изменить тип `var`:

```kotlin
var score = 10
score = 15
score = "пятнадцать" // ошибка компиляции
```

## Вывод типов

Kotlin выводит тип из правой части, если он однозначен.

```kotlin
val title = "Переменные" // String
val minutes = 55        // Int
val rating = 4.9        // Double
val published = true    // Boolean
```

Вывод типов не делает переменную динамической. Тип фиксируется на этапе
компиляции.

## Когда писать тип явно

Явный тип полезен, когда:

- правая часть не раскрывает намерение;
- нужен более общий тип, чем конкретная реализация;
- важно избежать неожиданного числового типа;
- переменная объявляется без немедленной инициализации;
- это публичное API;
- выражение сложное и тип ускоряет чтение.

```kotlin
val lessonTitles: List<String> = listOf(
    "Введение",
    "Первая программа",
    "Переменные"
)

val averageMinutes: Double = 42.5
val maxRetries: Long = 3L
```

`List<String>` сообщает контракт: нужен список строк только для чтения, а не
конкретная mutable-реализация.

## Неизменяемая ссылка и изменяемый объект

`val` фиксирует ссылку, но не обязательно делает объект неизменяемым.

```kotlin
fun main() {
    val completedLessons = mutableListOf<String>()

    completedLessons.add("Введение")
    completedLessons.add("Первая программа")

    completedLessons = mutableListOf("Новый список") // ошибка
}
```

`add` разрешен, потому что объект `MutableList` изменяем. Переназначение
запрещено, потому что ссылка объявлена через `val`.

Для read-only API используй `List<T>`:

```kotlin
val completedLessons: List<String> = listOf(
    "Введение",
    "Первая программа"
)

completedLessons.add("Переменные") // ошибка: у List нет add
```

`List` — read-only view, а не глубокая иммутабельность. На границе API лучше
копировать входные mutable-коллекции.

```kotlin
class ProgressSnapshot(completedLessons: List<String>) {
    private val lessons = completedLessons.toList()

    fun allLessons(): List<String> = lessons
}
```

## Инициализация

Локальный `val` можно объявить без значения, если компилятор видит, что значение
будет присвоено ровно один раз до чтения.

```kotlin
fun main() {
    val status: String
    val score = 86

    if (score >= 70) {
        status = "урок усвоен"
    } else {
        status = "нужно повторить"
    }

    println(status)
}
```

Часто короче и безопаснее использовать выражение:

```kotlin
val status = if (score >= 70) {
    "урок усвоен"
} else {
    "нужно повторить"
}
```

## Область видимости

Имя доступно только внутри блока, где объявлено.

```kotlin
fun main() {
    if (true) {
        val message = "Готово"
        println(message)
    }

    println(message) // ошибка
}
```

Если значение нужно после условия, объявляй его снаружи или вычисляй через
expression:

```kotlin
val message = if (courseName.isNotBlank()) {
    "Курс найден: $courseName"
} else {
    "Курс не найден"
}
```

## Shadowing

Shadowing — внутреннее имя скрывает внешнее.

```kotlin
fun main() {
    val title = "Переменные"

    run {
        val title = "Типы"
        println(title)
    }

    println(title)
}
```

Используй осторожно. Допустимый случай — нормализация входного значения:

```kotlin
fun printTitle(rawTitle: String) {
    val title = rawTitle.trim()

    if (title.isBlank()) return

    println(title)
}
```

Если одно имя в разных блоках означает разные сущности, лучше переименовать.

## Top-level свойства

Свойства можно объявлять на уровне файла.

```kotlin
package academy.kotlingo.variables

val defaultCourseTitle = "kotlingo"
```

Top-level `val` инициализируется при загрузке соответствующего файла на
платформе. Для значений, известных во время компиляции, используй `const val`.

## `const val`

`const val` — compile-time constant.

Где разрешен:

- top-level;
- внутри `object`;
- внутри `companion object`.

Типы: примитивные типы, `String` и другие типы, разрешенные для compile-time
constants. Значение должно быть известно компилятору.

```kotlin
const val COURSE_SLUG = "kotlingo"
const val MAX_LESSONS_IN_BATCH = 3

object CourseLimits {
    const val MAX_TITLE_LENGTH = 80
}
```

Нельзя:

```kotlin
fun main() {
    const val localLimit = 10 // ошибка
}

const val NORMALIZED_TITLE = " Kotlin ".trim() // ошибка
```

Используй `const val` для annotation arguments, стабильных ключей, лимитов,
compile-time флагов и Java interop.

## `lateinit var`

`lateinit` откладывает инициализацию свойства.

Ограничения:

- только `var`;
- только свойства, не local variables;
- только non-null reference types;
- нельзя для `Int`, `Long`, `Boolean` и nullable-типов.

```kotlin
class LessonController {
    lateinit var currentTitle: String

    fun open(title: String) {
        currentTitle = title
    }

    fun printCurrentTitle() {
        if (::currentTitle.isInitialized) {
            println(currentTitle)
        } else {
            println("Урок еще не открыт")
        }
    }
}
```

`lateinit` уместен, когда значение гарантированно установит framework, DI или
test lifecycle. Если отсутствие значения является нормальным состоянием, лучше
nullable-свойство.

```kotlin
class LessonController {
    private var currentTitle: String? = null

    fun open(title: String) {
        currentTitle = title
    }
}
```

Если зависимость обязательна, лучше constructor injection:

```kotlin
class LessonService(
    private val repository: LessonRepository
)
```

## Nullable-типы

По умолчанию переменная не принимает `null`.

```kotlin
var title: String = "Переменные"
title = null // ошибка
```

Если значение может отсутствовать, укажи nullable-тип:

```kotlin
var title: String? = "Переменные"
title = null

val visibleTitle = title ?: "урок не выбран"
```

Перед вызовом методов у nullable-значения нужна проверка, safe call или Elvis.

```kotlin
val rawInput: String? = readlnOrNull()
val length = rawInput?.length ?: 0
```

## Частые базовые типы

| Тип | Назначение |
| --- | --- |
| `String` | текст |
| `Int` | 32-битное целое число |
| `Long` | 64-битное целое число |
| `Double` | 64-битное число с плавающей точкой |
| `Boolean` | `true` или `false` |
| `List<T>` | список только для чтения |
| `MutableList<T>` | изменяемый список |

Пример:

```kotlin
fun main() {
    val rawTitle = "  Kotlin Basics  "
    val title = rawTitle.trim()
    val slug = title.lowercase().replace(" ", "-")
    val visibleLessons = 12.coerceIn(1, 10)
    val canOpenLesson = title.isNotBlank() && visibleLessons > 0

    println(slug)
    println(canOpenLesson)
}
```

## Практический шаблон

```kotlin
const val DEFAULT_LANGUAGE = "ru"
const val MAX_BATCH_SIZE = 3

data class LessonConfig(
    val title: String,
    val language: String,
    val estimatedMinutes: Int
)

class CoursePlan {
    private val lessons = mutableListOf<LessonConfig>()

    fun addLesson(
        title: String,
        estimatedMinutes: Int,
        language: String = DEFAULT_LANGUAGE
    ) {
        require(title.isNotBlank())
        require(estimatedMinutes > 0)

        lessons.add(
            LessonConfig(
                title = title.trim(),
                language = language,
                estimatedMinutes = estimatedMinutes.coerceAtMost(180)
            )
        )
    }

    fun nextBatch(): List<LessonConfig> {
        return lessons
            .take(MAX_BATCH_SIZE)
            .toList()
    }
}
```

Здесь:

- константы объявлены через `const val`;
- публичная модель неизменяема через `val`;
- изменяемый список скрыт через `private`;
- наружу возвращается `List`, а не `MutableList`;
- `toList()` создает копию результата.

## Выбор конструкции

| Конструкция | Когда использовать |
| --- | --- |
| `val` | значение известно и ссылка не должна меняться |
| `var` | значение действительно меняется во времени |
| `const val` | compile-time константа на top-level или в object |
| `lateinit var` | обязательное свойство заполнит framework/lifecycle |
| `T?` | отсутствие значения является нормальным состоянием |
| `List<T>` | внешний код должен только читать коллекцию |
| `MutableList<T>` | владелец коллекции должен менять ее содержимое |
