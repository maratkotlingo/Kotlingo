## Что такое Flow

`Flow<T>` — asynchronous stream values, который работает поверх coroutines.

Он нужен, когда result не один, а несколько значений во времени:

- прогресс загрузки;
- events от UI;
- изменения состояния;
- stream строк из файла;
- updates из database;
- polling external service;
- search suggestions;
- notifications;
- telemetry.

```kotlin
import kotlinx.coroutines.delay
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.flow

fun lessonProgress(): Flow<Int> = flow {
    emit(0)
    delay(100)
    emit(50)
    delay(100)
    emit(100)
}
```

`Flow` is cold by default. Это значит: code внутри `flow { }` не запускается, пока кто-то не вызовет terminal operator вроде `collect`.

<CatTip>
Cold flow похож на кошачий след в снегу, который появляется только когда кот реально пошел. Объявление flow само по себе еще ничего не выполняет.
</CatTip>

## Cold streams

Каждый collector запускает cold flow заново.

```kotlin
import kotlinx.coroutines.flow.flow
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

fun coldFlowExample() = runBlocking {
    val flow = flow {
        println("Flow started")
        emit("01-introduction")
        emit("84-flow")
    }

    val first = flow.toList()
    val second = flow.toList()

    println(first)
    println(second)
}
```

Вывод `Flow started` будет дважды, потому что каждый terminal operator starts collection separately.

Это полезно для query-like APIs:

```kotlin
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.flow

class LessonQueryRepository {
    fun lessonsByCategory(category: String): Flow<LessonSummary> = flow {
        val rows = loadRows(category)

        for (row in rows) {
            emit(
                LessonSummary(
                    slug = row.slug,
                    title = row.title
                )
            )
        }
    }

    private fun loadRows(category: String): List<LessonRow> {
        return listOf(
            LessonRow("84-flow", "Flow"),
            LessonRow("85-flow-operators", "Flow operators")
        ).filter { row -> category == "Coroutines" || row.slug.isNotBlank() }
    }
}

data class LessonSummary(
    val slug: String,
    val title: String
)

data class LessonRow(
    val slug: String,
    val title: String
)
```

## `flow { }` builder

`flow { }` создает cold flow. Внутри block есть `FlowCollector`, который умеет `emit`.

```kotlin
import kotlinx.coroutines.delay
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.flow

fun downloadLesson(slug: String): Flow<DownloadState> = flow {
    emit(DownloadState.Started(slug))

    delay(100)
    emit(DownloadState.Progress(percent = 40))

    delay(100)
    emit(DownloadState.Progress(percent = 80))

    delay(100)
    emit(DownloadState.Completed(slug))
}

sealed interface DownloadState {
    data class Started(
        val slug: String
    ) : DownloadState

    data class Progress(
        val percent: Int
    ) : DownloadState

    data class Completed(
        val slug: String
    ) : DownloadState
}
```

`emit` is suspending. Если downstream медленный, producer can suspend. Это базовая форма backpressure: fast producer не обязан бесконечно складывать значения в память.

## `collect`

`collect` — terminal operator, который запускает flow и получает каждое значение.

```kotlin
import kotlinx.coroutines.flow.collect
import kotlinx.coroutines.runBlocking

fun collectExample() = runBlocking {
    downloadLesson("84-flow").collect { state ->
        when (state) {
            is DownloadState.Started -> println("Начали ${state.slug}")
            is DownloadState.Progress -> println("Прогресс ${state.percent}%")
            is DownloadState.Completed -> println("Готово ${state.slug}")
        }
    }
}
```

Terminal operators:

- `collect` — обработать каждое value;
- `toList` — собрать values в list;
- `first` — взять первое value и cancel collection;
- `single` — ожидать ровно одно value;
- `reduce` — свернуть values без начального accumulator;
- `fold` — свернуть values с начальным accumulator;
- `count` — посчитать values;
- `launchIn` — collect in scope as job.

```kotlin
import kotlinx.coroutines.flow.count
import kotlinx.coroutines.flow.first
import kotlinx.coroutines.flow.fold
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

fun terminalOperatorsExample() = runBlocking {
    val states = downloadLesson("84-flow")

    val firstState = states.first()
    val allStates = states.toList()
    val progressEvents = states.count { state -> state is DownloadState.Progress }
    val text = states.fold("states:") { accumulator, state ->
        "$accumulator ${state::class.simpleName}"
    }

    println(firstState)
    println(allStates.size)
    println(progressEvents)
    println(text)
}
```

Помни: для cold flow каждый terminal operator starts collection again.

<MouseTrap>
Если flow делает network request или expensive query, несколько terminal operators могут выполнить эту работу несколько раз. Сохраняй результат, используй один collection pipeline или превращай cold flow в shared/state flow там, где это действительно нужно.
</MouseTrap>

## Intermediate operators

Intermediate operators возвращают новый flow and do not start collection.

```kotlin
import kotlinx.coroutines.flow.filter
import kotlinx.coroutines.flow.map
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

fun intermediateOperatorsExample() = runBlocking {
    val percentValues = downloadLesson("84-flow")
        .filter { state -> state is DownloadState.Progress }
        .map { state -> (state as DownloadState.Progress).percent }
        .toList()

    println(percentValues)
}
```

До `toList` pipeline only describes work. После terminal operator он starts.

## Sequential by default

По умолчанию flow operations sequential and execute in the same coroutine.

```kotlin
import kotlinx.coroutines.delay
import kotlinx.coroutines.flow.flow
import kotlinx.coroutines.flow.map
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

fun sequentialFlowExample() = runBlocking {
    val result = flow {
        emit("01-introduction")
        emit("84-flow")
    }
        .map { slug ->
            delay(100)
            slug.uppercase()
        }
        .toList()

    println(result)
}
```

Concurrency появляется только через специальные operators вроде `buffer`, `flatMapMerge`, `channelFlow`, `flowOn` boundary или явные coroutines в producer.

## `flowOf` и `asFlow`

`flowOf` creates flow from fixed values.

```kotlin
import kotlinx.coroutines.flow.flowOf
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

fun flowOfExample() = runBlocking {
    val slugs = flowOf("84-flow", "85-flow-operators", "86-stateflow-and-sharedflow")
        .toList()

    println(slugs)
}
```

`asFlow` converts Iterable, Sequence, array or range into flow.

```kotlin
import kotlinx.coroutines.flow.asFlow
import kotlinx.coroutines.flow.map
import kotlinx.coroutines.flow.toList
import kotlinx.coroutines.runBlocking

fun asFlowExample() = runBlocking {
    val titles = listOf("Flow", "Operators", "StateFlow")
        .asFlow()
        .map { title -> title.uppercase() }
        .toList()

    println(titles)
}
```

## Flow cancellation

Flow collection follows coroutine cancellation.

```kotlin
import kotlinx.coroutines.cancelAndJoin
import kotlinx.coroutines.delay
import kotlinx.coroutines.flow.collect
import kotlinx.coroutines.flow.flow
import kotlinx.coroutines.launch
import kotlinx.coroutines.runBlocking

fun flowCancellationExample() = runBlocking {
    val job = launch {
        flow {
            var value = 0

            while (true) {
                delay(100)
                emit(value++)
            }
        }.collect { value ->
            println("Получили $value")
        }
    }

    delay(350)
    job.cancelAndJoin()
}
```

`flow { }` checks cancellation on emission. For CPU-heavy loops use `ensureActive`, `yield` or `cancellable`.

```kotlin
import kotlinx.coroutines.currentCoroutineContext
import kotlinx.coroutines.ensureActive
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.flow

fun tokenizeAsFlow(text: String): Flow<String> = flow {
    for (token in text.splitToSequence(" ")) {
        currentCoroutineContext().ensureActive()

        if (token.isNotBlank()) {
            emit(token.lowercase())
        }
    }
}
```

## Exception transparency

Flow follows exception transparency:

- upstream exceptions can be handled by `catch`;
- downstream exceptions are not swallowed by upstream `catch`;
- `emit` after downstream failure is not allowed;
- `CancellationException` should not be converted into data accidentally;
- `onCompletion` is declarative replacement for `finally`-like observation.

```kotlin
import kotlinx.coroutines.flow.catch
import kotlinx.coroutines.flow.collect
import kotlinx.coroutines.flow.flow
import kotlinx.coroutines.flow.map
import kotlinx.coroutines.runBlocking

fun flowExceptionExample() = runBlocking {
    flow {
        emit("84-flow")
        error("Источник упал")
    }
        .map { slug -> slug.uppercase() }
        .catch { exception ->
            // catch видит upstream exception и может emit fallback.
            emit("FALLBACK: ${exception.message}")
        }
        .collect { value ->
            println(value)
        }
}
```

Downstream exception:

```kotlin
import kotlinx.coroutines.flow.catch
import kotlinx.coroutines.flow.flowOf
import kotlinx.coroutines.flow.map
import kotlinx.coroutines.runBlocking

fun downstreamExceptionExample() = runBlocking {
    flowOf("84-flow")
        .catch {
            // Эта ветка не поймает exception из последующего map.
            emit("fallback")
        }
        .map {
            error("Downstream упал")
        }
        .collect {
            println(it)
        }
}
```

## Backpressure conceptually

Backpressure — ситуация, когда producer быстрее consumer-а. Flow by default suspends producer until consumer is ready.

```kotlin
import kotlinx.coroutines.delay
import kotlinx.coroutines.flow.flow
import kotlinx.coroutines.flow.collect
import kotlinx.coroutines.runBlocking

fun backpressureExample() = runBlocking {
    flow {
        repeat(3) { index ->
            println("Emit $index")
            emit(index)
        }
    }.collect { value ->
        delay(200)
        println("Collect $value")
    }
}
```

Без `buffer` следующий `emit` обычно не продолжит pipeline, пока collector не обработает current value.

Operators for pressure:

- `buffer` — разрешает producer и consumer работать concurrent with buffer;
- `conflate` — пропускает intermediate values, оставляя latest;
- `collectLatest` — отменяет обработку previous value when new value arrives;
- `sample` — берет latest value с периодом;
- `debounce` — ждет quiet period;
- `flatMapMerge` — параллелит inner flows with concurrency limit.

Подробно operators будут в следующей теме.

## `flowOn`

`flowOn` changes context for upstream part of pipeline.

```kotlin
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.flow
import kotlinx.coroutines.flow.flowOn
import kotlinx.coroutines.flow.map

fun lessonTitles(paths: List<String>): Flow<String> =
    flow {
        for (path in paths) {
            // Источник может читать blocking filesystem в реальном repository.
            emit(path)
        }
    }
        .map { path -> path.substringAfterLast("/") }
        .flowOn(Dispatchers.IO)
```

Важно: `flowOn` affects only upstream operators before it.

```kotlin
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.flow.flowOf
import kotlinx.coroutines.flow.flowOn
import kotlinx.coroutines.flow.map

fun flowOnBoundaryExample() =
    flowOf("84-flow")
        .map { slug ->
            // Этот map upstream от flowOn, значит выполняется в Default.
            slug.uppercase()
        }
        .flowOn(Dispatchers.Default)
        .map { title ->
            // Этот map downstream от flowOn, значит выполняется в collector context.
            "Title: $title"
        }
```

<MouseTrap>
Не ставь `withContext` прямо внутри `flow { }` для emission из другого context. Для смены контекста upstream используй `flowOn`; для concurrent producer cases есть `channelFlow`.
</MouseTrap>

## `channelFlow`

`channelFlow` позволяет safely emit из child coroutines. Это advanced builder for concurrent producers.

```kotlin
import kotlinx.coroutines.async
import kotlinx.coroutines.awaitAll
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.channelFlow

fun loadLessonCards(slugs: List<String>): Flow<LessonCard> = channelFlow {
    slugs.map { slug ->
        async {
            send(loadCard(slug))
        }
    }.awaitAll()
}

data class LessonCard(
    val slug: String,
    val title: String
)

suspend fun loadCard(slug: String): LessonCard {
    return LessonCard(slug, "Урок $slug")
}
```

Для ordinary sequential stream предпочитай `flow { }`. `channelFlow` adds channel, concurrency and more moving parts.

## Production pattern: progress stream

```kotlin
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.currentCoroutineContext
import kotlinx.coroutines.ensureActive
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.flow
import kotlinx.coroutines.flow.flowOn

class LessonImportProgressService(
    private val parser: LessonParser
) {
    fun import(files: List<String>): Flow<ImportEvent> =
        flow {
            emit(ImportEvent.Started(total = files.size))

            files.forEachIndexed { index, file ->
                currentCoroutineContext().ensureActive()

                val lesson = parser.parse(file)

                emit(
                    ImportEvent.FileImported(
                        slug = lesson.slug,
                        completed = index + 1,
                        total = files.size
                    )
                )
            }

            emit(ImportEvent.Completed)
        }.flowOn(Dispatchers.Default)
}

sealed interface ImportEvent {
    data class Started(
        val total: Int
    ) : ImportEvent

    data class FileImported(
        val slug: String,
        val completed: Int,
        val total: Int
    ) : ImportEvent

    data object Completed : ImportEvent
}

class LessonParser {
    fun parse(file: String): ParsedLesson {
        return ParsedLesson(
            slug = file.substringBeforeLast(".").lowercase(),
            title = file.substringBeforeLast(".")
        )
    }
}

data class ParsedLesson(
    val slug: String,
    val title: String
)
```

Такой API хорош для UI and CLI:

- caller starts collection when ready;
- progress values arrive over time;
- cancellation caller-а cancels import;
- CPU work runs upstream on `Dispatchers.Default`;
- no hidden global coroutine exists.
