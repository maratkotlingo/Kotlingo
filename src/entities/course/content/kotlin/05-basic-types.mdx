## Что описывает тип

Тип задает:

- какие значения допустимы;
- какие операции можно выполнить;
- может ли значение быть `null`;
- что увидит вызывающий код;
- какие ошибки компилятор поймает до запуска.

```kotlin
fun main() {
    val title: String = "Базовые типы"
    val lessonNumber: Int = 5
    val completed: Boolean = false

    println(title.uppercase())
    println(lessonNumber + 1)
    println(!completed)
}
```

Kotlin статически типизирован: тип каждого выражения известен компилятору.

## Non-null и nullable

Почти каждый тип имеет non-null и nullable форму.

```kotlin
val requiredTitle: String = "Типы"
val optionalTitle: String? = null

println(requiredTitle.length)
println(optionalTitle?.length ?: 0)
```

`String` и `String?` — разные типы. Nullable-значение нельзя использовать как
обычное без проверки.

```kotlin
val score: Int = 100
val optionalScore: Int? = null

println(score + 1)
println((optionalScore ?: 0) + 1)
```

На JVM nullable primitive-like типы, например `Int?`, требуют объектного
представления, потому что должны хранить `null`.

## Целые числа со знаком

| Тип | Размер | Основное применение |
| --- | --- | --- |
| `Byte` | 8 бит | байтовые протоколы, файлы, interop |
| `Short` | 16 бит | бинарные форматы, interop |
| `Int` | 32 бита | счетчики, индексы, размеры, большинство целых чисел |
| `Long` | 64 бита | ID, timestamps, большие счетчики, размеры файлов |

```kotlin
val byteValue: Byte = 120
val shortValue: Short = 32_000
val intValue: Int = 42
val longValue: Long = 9_223_372_036_854_775_000L
```

`Int` — основной целочисленный тип. Литерал без суффикса обычно выводится как
`Int`, если значение помещается в диапазон.

```kotlin
fun main() {
    val rawProgress = 137
    val visibleProgress = rawProgress.coerceIn(0, 100)

    println(visibleProgress)
    println(Int.MAX_VALUE)
}
```

Частые операции:

- `+`, `-`, `*`, `/`, `%`;
- `MIN_VALUE`, `MAX_VALUE`;
- `toByte()`, `toShort()`, `toInt()`, `toLong()`;
- `toFloat()`, `toDouble()`;
- `coerceAtLeast`, `coerceAtMost`, `coerceIn`;
- `compareTo`, `toString`.

## Нет неявного widening

Kotlin не расширяет числовые типы неявно.

```kotlin
fun printLong(value: Long) {
    println(value)
}

fun main() {
    val value: Int = 42

    printLong(value.toLong())
    printLong(42L)
}
```

Это снижает риск скрытых преобразований и неоднозначных перегрузок.

## Unsigned-типы

| Тип | Аналог со знаком | Применение |
| --- | --- | --- |
| `UByte` | `Byte` | байтовые маски, бинарные форматы |
| `UShort` | `Short` | низкоуровневые форматы |
| `UInt` | `Int` | unsigned-семантика, bit masks |
| `ULong` | `Long` | большие unsigned-счетчики |

```kotlin
fun main() {
    val byteMask: UByte = 255u
    val lessonCode: UInt = 42u
    val hugeCounter: ULong = 18_446_744_073_709_551_615uL

    println(byteMask)
    println(lessonCode)
    println(hugeCounter)
}
```

Используй unsigned-типы, когда unsigned-семантика действительно часть модели:
протоколы, bit masks, низкоуровневые алгоритмы. Не выбирай их только потому, что
значение "не бывает отрицательным": для бизнес-кода чаще подходит `Int` или
`Long` с валидацией.

## Дробные числа

| Тип | Размер | Применение |
| --- | --- | --- |
| `Float` | 32 бита | графика, аудио, некоторые ML/API, экономия памяти |
| `Double` | 64 бита | средние, коэффициенты, измерения, статистика |

```kotlin
val rating: Float = 4.8f
val averageMinutes: Double = 42.5
```

`Double` — основной дробный тип в обычных вычислениях.

```kotlin
fun main() {
    val totalMinutes = 125.0
    val lessons = 3.0
    val average = totalMinutes / lessons

    println(average)
    println(average.toInt())
}
```

Частые свойства и операции:

- `Float.NaN`, `Double.NaN`;
- `POSITIVE_INFINITY`, `NEGATIVE_INFINITY`;
- `isNaN()`, `isInfinite()`, `isFinite()`;
- `toInt()`, `toLong()`, `toFloat()`, `toDouble()`;
- арифметические операторы.

`Float` и `Double` не подходят для денег без специальных правил округления и
представления. Для денежных значений обычно используют `BigDecimal`, minor units
в `Long` или доменный тип.

## `Boolean`

`Boolean` хранит `true` или `false`.

```kotlin
fun main() {
    val hasAccount = true
    val completedIntro = false

    val canOpenSecondLesson = hasAccount && completedIntro
    println(canOpenSecondLesson)
}
```

Операции:

- `&&` — логическое "и" с short-circuit evaluation;
- `||` — логическое "или" с short-circuit evaluation;
- `!` — отрицание;
- `and`, `or`, `xor` — infix-функции без short-circuit.

```kotlin
fun expensiveCheck(): Boolean {
    println("Проверка выполнена")
    return true
}

fun main() {
    val result = false && expensiveCheck()
    println(result)
}
```

`expensiveCheck()` не вызовется из-за short-circuit.

## `Char`

`Char` хранит один UTF-16 code unit.

```kotlin
val shortcut: Char = 'K'
```

Частые операции:

- `isDigit()`, `isLetter()`, `isLetterOrDigit()`;
- `isUpperCase()`, `isLowerCase()`;
- `lowercaseChar()`, `uppercaseChar()`;
- `digitToInt()`, `digitToIntOrNull()`;
- `code`;
- `toString()`.

```kotlin
fun main() {
    val input = '7'
    val digit = input.digitToIntOrNull()

    if (digit == null) {
        println("Это не цифра")
        return
    }

    println(digit)
}
```

Не каждый видимый пользователю символ помещается в один `Char`: некоторые emoji
и редкие символы представлены несколькими code units.

## `String`

`String` хранит неизменяемую последовательность символов. Методы вроде `trim`
и `replace` возвращают новую строку.

```kotlin
fun main() {
    val rawTitle = "  Базовые типы  "
    val title = rawTitle.trim()

    println(rawTitle)
    println(title)
}
```

Частые свойства и методы:

- `length`, `indices`, `lastIndex`;
- `isEmpty()`, `isNotEmpty()`;
- `isBlank()`, `isNotBlank()`;
- `trim()`, `trimStart()`, `trimEnd()`;
- `lowercase()`, `uppercase()`;
- `replace(oldValue, newValue)`;
- `contains`, `startsWith`, `endsWith`;
- `substring`, `split`, `lines`;
- `toInt()`, `toIntOrNull()`, `toLongOrNull()`, `toDoubleOrNull()`.

```kotlin
fun main() {
    val rawInput = "  Kotlin, Типы, Числа  "

    val topics = rawInput
        .trim()
        .split(",")
        .map { topic -> topic.trim() }
        .filter { topic -> topic.isNotBlank() }

    println(topics)
}
```

## `Any`

`Any` — корень non-null типов Kotlin. Любое non-null значение можно рассматривать
как `Any`.

```kotlin
fun printValue(value: Any) {
    println(value.toString())
}

fun main() {
    printValue("Kotlin")
    printValue(42)
    printValue(true)
}
```

Базовые методы:

- `equals(other: Any?)`;
- `hashCode()`;
- `toString()`.

`Any` полезен для логирования, generic-free utility API и временного хранения
разных типов с последующей проверкой. Не используй `Any`, когда можно выразить
точный тип, sealed hierarchy или generic.

```kotlin
fun describe(value: Any): String {
    return when (value) {
        is String -> "Строка длиной ${value.length}"
        is Int -> "Целое число: $value"
        is Boolean -> "Логическое значение: $value"
        else -> "Неизвестное значение: $value"
    }
}
```

## `Unit`

`Unit` означает отсутствие полезного возвращаемого значения. Это реальное
значение с единственным экземпляром, а не `null`.

```kotlin
fun printLesson(title: String): Unit {
    println(title)
}
```

Обычно `Unit` в возвращаемом типе не пишут:

```kotlin
fun printLesson(title: String) {
    println(title)
}
```

Где `Unit` важен явно:

- function types: `(String) -> Unit`;
- callbacks;
- generic API;
- документация намеренного side effect.

```kotlin
fun runForEachLesson(
    lessons: List<String>,
    action: (String) -> Unit
) {
    lessons.forEach(action)
}
```

## `Nothing`

`Nothing` — тип без значений. Функция с типом `Nothing` не возвращает нормальный
результат: бросает исключение, завершает процесс или работает бесконечно.

```kotlin
fun fail(message: String): Nothing {
    throw IllegalStateException(message)
}

fun main() {
    val title: String? = null
    val safeTitle = title ?: fail("Название урока отсутствует")

    println(safeTitle)
}
```

`Nothing` совместим с любым ожидаемым типом, потому что значения там никогда не
будет. Поэтому он хорошо работает в правой части Elvis `?:`.

## JVM primitives и boxing

В исходном Kotlin `Int`, `Boolean`, `Double` и другие типы выглядят как обычные
типы. На JVM компилятор может представить non-null числовые и boolean-значения
как primitives:

- `Int` как `int`;
- `Long` как `long`;
- `Boolean` как `boolean`;
- `Double` как `double`.

Boxing нужен, когда значение должно стать объектом:

- nullable primitive-like тип: `Int?`;
- generic-контейнер: `List<Int>`;
- передача как `Any`;
- Java object API.

```kotlin
fun printAny(value: Any) {
    println(value)
}

fun main() {
    val score: Int = 100
    val optionalScore: Int? = score
    val scores: List<Int> = listOf(score)

    printAny(score)
    println(optionalScore)
    println(scores)
}
```

Сначала выбирай точный и понятный типовой контракт. Оптимизируй представление
только после измерений или в очевидных hot paths.

## Практическая модель

```kotlin
data class LessonMetrics(
    val id: Long,
    val title: String,
    val estimatedMinutes: Int,
    val averageScore: Double,
    val completed: Boolean,
    val shortcut: Char
)

fun validateMetrics(metrics: LessonMetrics) {
    require(metrics.id > 0L)
    require(metrics.title.isNotBlank())
    require(metrics.estimatedMinutes in 1..240)
    require(metrics.averageScore.isFinite())
    require(metrics.averageScore in 0.0..100.0)
    require(metrics.shortcut.isLetterOrDigit())
}

fun formatMetrics(metrics: LessonMetrics): String {
    validateMetrics(metrics)

    val status = if (metrics.completed) "пройден" else "в процессе"

    return buildString {
        append("Урок #")
        append(metrics.id)
        append(": ")
        append(metrics.title.trim())
        append(", длительность: ")
        append(metrics.estimatedMinutes)
        append(" мин., средний балл: ")
        append(metrics.averageScore)
        append(", статус: ")
        append(status)
        append(", клавиша: ")
        append(metrics.shortcut.uppercaseChar())
    }
}
```

Здесь:

- `Long` хранит идентификатор;
- `String` хранит текст;
- `Int` хранит длительность;
- `Double` хранит средний балл;
- `Boolean` хранит состояние;
- `Char` хранит клавишу;
- `require` проверяет доменные ограничения, которые типы сами не выражают.

## Быстрый выбор типа

| Задача | Тип |
| --- | --- |
| текст | `String` |
| номер, счетчик, индекс | `Int` |
| ID, timestamp, большой счетчик | `Long` |
| среднее, коэффициент, измерение | `Double` |
| графика/аудио/interop с 32-битным float | `Float` |
| флаг состояния | `Boolean` |
| одиночная клавиша или UTF-16 code unit | `Char` |
| значение может отсутствовать | `T?` |
| функция ничего полезного не возвращает | `Unit` |
| функция не возвращается нормально | `Nothing` |
| нужен произвольный non-null тип | `Any` |
