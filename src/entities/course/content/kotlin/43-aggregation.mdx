## Aggregation сворачивает много значений в одно

Aggregation operation берет collection и получает summary: count, sum, min, max, average, пользовательский state, running totals.

```kotlin
val minutes = listOf(14, 28, 35)

val total = minutes.sum()

println(total) // 77
```

После aggregation обычно остается одно значение или одно значение на шаге накопления.

<CatTip>
Aggregation — это как пересчитать кошачьи лакомства в миске: элементов много, а результат один — сколько всего, сколько осталось, какое самое большое.
</CatTip>

## `count`

`count()` возвращает количество элементов.

```kotlin
val lessons = listOf("Введение", "Типы", "Null-safety")

println(lessons.count()) // 3
```

Для `Collection` свойство `size` обычно прямее.

```kotlin
val amount = lessons.size
```

`count(predicate)` считает только matching elements.

```kotlin
data class Lesson(
    val slug: String,
    val title: String,
    val minutes: Int,
    val completed: Boolean
)

val course = listOf(
    Lesson("01-introduction", "Введение", 14, completed = true),
    Lesson("02-types", "Типы", 28, completed = false),
    Lesson("03-null-safety", "Null-safety", 35, completed = false)
)

val completedCount = course.count { lesson ->
    // Считаем только завершенные уроки.
    lesson.completed
}

println(completedCount) // 1
```

`count(predicate)` может short-circuit только в смысле отсутствия дополнительной collection: ему все равно нужно проверить каждый element, чтобы получить точное количество.

## `sum`

`sum` складывает numeric collection.

```kotlin
val totalMinutes = course
    .map { lesson ->
        lesson.minutes
    }
    .sum()

println(totalMinutes) // 77
```

Для primitive numeric lists доступны specialized перегрузки. Для nullable values сначала убери `null` или задай default.

```kotlin
val optionalScores: List<Int?> = listOf(10, null, 20)

val totalScore = optionalScores
    .filterNotNull()
    .sum()

println(totalScore) // 30
```

## `sumOf`

`sumOf` объединяет mapping и summing.

```kotlin
val totalMinutesDirect = course.sumOf { lesson ->
    // Берем числовое поле и сразу складываем.
    lesson.minutes
}

println(totalMinutesDirect) // 77
```

`sumOf` обычно лучше, чем `map { ... }.sum()`, потому что не создает intermediate list.

```kotlin
data class InvoiceLine(
    val unitPriceCents: Long,
    val quantity: Int
)

val lines = listOf(
    InvoiceLine(unitPriceCents = 1_500, quantity = 2),
    InvoiceLine(unitPriceCents = 2_000, quantity = 1)
)

val totalCents = lines.sumOf { line ->
    // Long защищает денежные суммы от переполнения Int в типичных сценариях.
    line.unitPriceCents * line.quantity
}

println(totalCents) // 5000
```

<MouseTrap>
Не используй `Double` для денег. Для учебных примеров это кажется удобным, но боевой-суммы лучше хранить в минимальных единицах валюты или в `BigDecimal`.
</MouseTrap>

## `min`, `max`, `minOrNull`, `maxOrNull`

Для collection comparable values можно получить minimum и maximum.

```kotlin
val shortest = minutes.min()
val longest = minutes.max()

println(shortest) // 14
println(longest)  // 35
```

На пустой collection `min()` и `max()` бросают exception. Безопасные варианты возвращают nullable result.

```kotlin
val empty = emptyList<Int>()

val safeMin: Int? = empty.minOrNull()
val safeMax: Int? = empty.maxOrNull()

println(safeMin) // null
println(safeMax) // null
```

Используй non-null variants только когда invariant гарантирует непустой input.

```kotlin
fun requireShortestDuration(minutes: List<Int>): Int {
    require(minutes.isNotEmpty()) {
        "Список длительностей не должен быть пустым"
    }

    return minutes.min()
}
```

## `minBy`, `maxBy`, `minByOrNull`, `maxByOrNull`

`minBy` и `maxBy` выбирают element по selector.

```kotlin
val shortestLesson = course.minBy { lesson ->
    lesson.minutes
}

val longestLesson = course.maxBy { lesson ->
    lesson.minutes
}

println(shortestLesson.title) // Введение
println(longestLesson.title)  // Null-safety
```

Для possibly empty collection используй nullable variants.

```kotlin
val maybeShortest = emptyList<Lesson>().minByOrNull { lesson ->
    lesson.minutes
}

println(maybeShortest) // null
```

Если несколько elements имеют одинаковый minimum или maximum selector value, возвращается один из них согласно порядку traversal. Для предсказуемости можно отсортировать с tie-breaker или использовать `reduce`.

```kotlin
val bestForStart = course.minWithOrNull(
    compareBy<Lesson> { lesson ->
        lesson.minutes
    }.thenBy { lesson ->
        lesson.slug
    }
)
```

## `fold`

`fold` сворачивает collection в accumulator, начиная с explicit initial value.

```kotlin
val totalWithFold = minutes.fold(initial = 0) { accumulator, value ->
    // accumulator хранит уже накопленную сумму.
    accumulator + value
}

println(totalWithFold) // 77
```

Тип accumulator может отличаться от type element.

```kotlin
data class CourseStats(
    val lessons: Int,
    val completed: Int,
    val totalMinutes: Int
)

val stats = course.fold(
    initial = CourseStats(lessons = 0, completed = 0, totalMinutes = 0)
) { accumulator, lesson ->
    accumulator.copy(
        lessons = accumulator.lessons + 1,
        completed = accumulator.completed + if (lesson.completed) 1 else 0,
        totalMinutes = accumulator.totalMinutes + lesson.minutes
    )
}

println(stats)
```

`fold` безопасен для empty collection, потому что initial value уже есть.

```kotlin
val emptyStats = emptyList<Lesson>().fold(
    initial = CourseStats(lessons = 0, completed = 0, totalMinutes = 0)
) { accumulator, lesson ->
    accumulator.copy(
        lessons = accumulator.lessons + 1,
        totalMinutes = accumulator.totalMinutes + lesson.minutes
    )
}
```

## `reduce`

`reduce` использует первый element как initial accumulator.

```kotlin
val totalWithReduce = minutes.reduce { accumulator, value ->
    accumulator + value
}

println(totalWithReduce) // 77
```

`reduce` требует non-empty collection. На empty collection будет exception.

```kotlin
fun sumNonEmpty(values: List<Int>): Int {
    require(values.isNotEmpty()) {
        "Для reduce нужен непустой список"
    }

    return values.reduce { accumulator, value ->
        accumulator + value
    }
}
```

Тип accumulator совпадает с element type. Если нужен другой type, используй `fold`.

```kotlin
val csv = lessons.reduce { accumulator, title ->
    // Здесь accumulator и title оба String.
    "$accumulator, $title"
}
```

<MouseTrap>
Не используй `reduce` просто потому, что он выглядит короче. Если пустой input допустим, `fold` обычно надежнее.
</MouseTrap>

## `runningFold`

`runningFold` возвращает list промежуточных accumulator values, включая initial value.

```kotlin
val cumulativeMinutes = minutes.runningFold(initial = 0) { accumulator, value ->
    accumulator + value
}

println(cumulativeMinutes) // [0, 14, 42, 77]
```

Это полезно для графиков, прогресс bars, audit trail и пошаговых explainers.

```kotlin
data class ProgressPoint(
    val lessonNumber: Int,
    val totalMinutes: Int
)

val progressPoints = minutes
    .runningFold(initial = 0) { accumulator, value ->
        accumulator + value
    }
    .drop(1)
    .mapIndexed { index, total ->
        ProgressPoint(
            lessonNumber = index + 1,
            totalMinutes = total
        )
    }
```

## `runningReduce`

`runningReduce` похож на `runningFold`, но initial value берется из первого element и не добавляется отдельно.

```kotlin
val runningTotals = minutes.runningReduce { accumulator, value ->
    accumulator + value
}

println(runningTotals) // [14, 42, 77]
```

Для empty collection результат будет empty list.

```kotlin
val emptyRunning = emptyList<Int>().runningReduce { accumulator, value ->
    accumulator + value
}

println(emptyRunning) // []
```

## Right variants

У folding/reducing есть right-to-left variants: `foldRight`, `reduceRight`, `runningFold`, `runningReduce` имеют свои особенности порядка. `foldRight` и `reduceRight` важны, когда operation не associative.

```kotlin
val words = listOf("кот", "пьет", "молоко")

val leftSentence = words.fold("") { accumulator, word ->
    // Идем слева направо.
    if (accumulator.isEmpty()) word else "$accumulator $word"
}

val rightSentence = words.foldRight("") { word, accumulator ->
    // Идем справа налево, порядок параметров отличается.
    if (accumulator.isEmpty()) word else "$word $accumulator"
}

println(leftSentence)  // кот пьет молоко
println(rightSentence) // кот пьет молоко
```

Для сложения порядок не важен, но для строк, деревьев и parsers порядок может менять смысл.

## Боевой-пример: отчет по платежам

```kotlin
enum class PaymentStatus {
    CREATED,
    PAID,
    REFUNDED,
    FAILED
}

data class Payment(
    val id: String,
    val userId: String,
    val amountCents: Long,
    val status: PaymentStatus
)

data class PaymentReport(
    val totalPayments: Int,
    val paidPayments: Int,
    val failedPayments: Int,
    val grossCents: Long,
    val refundedCents: Long
) {
    val netCents: Long
        get() {
            // Возвраты уменьшают итоговую выручку.
            return grossCents - refundedCents
        }
}

fun buildPaymentReport(payments: List<Payment>): PaymentReport {
    return payments.fold(
        initial = PaymentReport(
            totalPayments = 0,
            paidPayments = 0,
            failedPayments = 0,
            grossCents = 0,
            refundedCents = 0
        )
    ) { report, payment ->
        val grossDelta = if (payment.status == PaymentStatus.PAID) {
            payment.amountCents
        } else {
            0L
        }

        val refundDelta = if (payment.status == PaymentStatus.REFUNDED) {
            payment.amountCents
        } else {
            0L
        }

        report.copy(
            totalPayments = report.totalPayments + 1,
            paidPayments = report.paidPayments + if (payment.status == PaymentStatus.PAID) 1 else 0,
            failedPayments = report.failedPayments + if (payment.status == PaymentStatus.FAILED) 1 else 0,
            grossCents = report.grossCents + grossDelta,
            refundedCents = report.refundedCents + refundDelta
        )
    }
}
```

Такой `fold` хорошо тестируется: на вход даешь список payments, на выходе ожидаемый immutable report.

## Aggregation проверочный список

Для суммы поля используй `sumOf`.

```kotlin
val total = course.sumOf { lesson -> lesson.minutes }
```

Для count по условию используй `count(predicate)`.

```kotlin
val completed = course.count { lesson -> lesson.completed }
```

Для пользовательский неизменяемое состояние используй `fold`.

```kotlin
val titles = course.fold(initial = emptyList<String>()) { accumulator, lesson ->
    accumulator + lesson.title
}
```

Для non-empty mathematical reduction можно использовать `reduce`, но сначала проверь invariant.

```kotlin
val product = values.reduce { accumulator, value ->
    accumulator * value
}
```

Для графика накопления используй `runningFold` или `runningReduce`.

```kotlin
val cumulative = values.runningFold(0) { accumulator, value ->
    accumulator + value
}
```

<CatTip>
Хорошая aggregation-функция не прячет побочные эффекты. Пусть она спокойно считает итог, а запись результата в базу или отправка события будет отдельным шагом.
</CatTip>
